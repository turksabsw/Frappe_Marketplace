# Kategori SVG İkon Atama Planı (Category SVG Icon Assignment Plan)

## 1. Bağlam (Context)

### Mevcut Durum

TradeHub platformunda kategori DocType'larında görsel alan olarak yalnızca `Attach Image` tipinde `image` alanı bulunmaktadır. Bu alan raster görseller (PNG, JPG, WebP) için tasarlanmıştır ve SVG ikon yönetimi için optimize değildir.

#### 1.1 Category DocType — Mevcut `image` Alanı

- **Konum:** `tradehub_catalog/doctype/category/category.json`
- **Modül:** TradeHub Catalog
- **Bölüm:** Details Section

| fieldname | fieldtype | label | options | reqd | default | description |
|-----------|-----------|-------|---------|------|---------|-------------|
| image | Attach Image | Image | — | 0 | — | Kategori görseli (raster format) |

**Frappe `Attach Image` Davranışı:**
- Dosya yükleme dialog'u açar (Drag & Drop veya File Manager)
- Yüklenen dosya `/files/` dizinine kaydedilir (`/api/method/upload_file`)
- Veritabanında dosya yolu saklanır (örn: `/files/category-elektronik.png`)
- Thumbnail otomatik oluşturulur (küçük resim)
- Desteklenen formatlar: PNG, JPG, JPEG, GIF, WebP, SVG (ama SVG'ye özel işlem yok)
- SVG yüklendiğinde thumbnail oluşturulamaz, doğrudan dosya yolu saklanır

#### 1.2 Product Category DocType — Mevcut `image` Alanı

- **Konum:** `tradehub_catalog/doctype/product_category/product_category.json`
- **Modül:** TradeHub Catalog
- **Bölüm:** Description Section

| fieldname | fieldtype | label | options | reqd | default | description |
|-----------|-----------|-------|---------|------|---------|-------------|
| image | Attach Image | Category Image | — | 0 | — | Category image for display |

**Her iki DocType'ta Ortak Eksiklikler:**
- SVG ikon için ayrılmış bir alan yok
- SVG içeriğini inline olarak saklama imkanı yok
- İkon kütüphanesi referansı (ör: `fa-laptop`, `mdi-phone`) için alan yok
- Farklı boyut/renk varyantları için yapılandırma yok

### Neden SVG İkon Gerekli?

1. **Navigasyon Menüsü:** Kategori ağacında her kategorinin yanında küçük, ölçeklenebilir ikonlar gösterilmeli
2. **Mobil Uyumluluk:** SVG'ler vektörel olduğu için tüm ekran çözünürlüklerinde net görünür
3. **Performans:** SVG ikonlar raster görsellere göre çok daha küçük boyutludur (tipik: 1-5 KB vs 50-200 KB)
4. **Tema Uyumu:** SVG ikonların renkleri CSS ile değiştirilebilir (dark mode, hover durumları)
5. **Tutarlılık:** Tüm kategorilerde aynı stil ve boyutta ikon kullanımı sağlanır

## 2. Analiz (Analysis)

### 2.1 Mevcut `image` Alanı vs. SVG İkon İhtiyacı

| Özellik | Mevcut `image` (Attach Image) | İhtiyaç: SVG İkon |
|---------|-------------------------------|-------------------|
| Dosya Türü | PNG, JPG, WebP, GIF, (SVG) | Yalnızca SVG |
| Kullanım Amacı | Kategori banner/kapak görseli | Navigasyon ikonu (küçük, ölçeklenebilir) |
| Tipik Boyut | 200-500 KB (raster) | 1-5 KB (vektörel) |
| Ölçeklenebilirlik | Piksel bazlı, büyütünce bozulur | Sonsuz ölçeklenebilir |
| Renk Değiştirme | Mümkün değil | CSS `fill` / `stroke` ile mümkün |
| Thumbnail | Otomatik oluşturulur | Gerekli değil (zaten küçük) |
| Inline Render | `<img src="...">` | `<svg>...</svg>` veya `<img src="...">` |
| Dark Mode | Farklı görsel yüklemek gerekir | CSS ile otomatik renk değişimi |

**Sonuç:** Mevcut `image` alanı korunmalı (banner/kapak görseli olarak), SVG ikon için **ayrı bir alan** eklenmelidir.

### 2.2 SVG İkon Saklama Yaklaşımları Karşılaştırması

| Yaklaşım | Avantajlar | Dezavantajlar | Uygunluk |
|----------|------------|---------------|----------|
| **A) Attach Image (dosya yükleme)** | Frappe native, tanıdık UX, dosya yönetimi kolay | SVG validasyonu yok, XSS riski, inline render için extra fetch gerekli | ⭐⭐⭐ |
| **B) Code (inline SVG)** | Tam kontrol, doğrudan inline render, CDN gereksiz | Büyük SVG'ler veritabanını şişirir, editör UX zor | ⭐⭐ |
| **C) Data (ikon kütüphanesi referansı)** | En hafif, tutarlı stil, kolay yönetim | Özel ikonlar için esneklik yok, kütüphane bağımlılığı | ⭐⭐⭐ |
| **D) Hibrit: Attach + Data** | Hem özel SVG yükleme hem de kütüphane referansı desteği | Daha karmaşık frontend mantığı | ⭐⭐⭐⭐ |

### 2.3 Önerilen Yaklaşım: Hibrit (D)

**Birincil alan:** `icon` — `Attach Image` tipinde, yalnızca SVG dosyası yüklemek için
**İkincil alan:** `icon_class` — `Data` tipinde, ikon kütüphanesi sınıf adı için (ör: `fa-laptop`, `mdi-phone`)

**Öncelik Kuralı:** `icon` (yüklenen SVG) > `icon_class` (kütüphane referansı) > varsayılan ikon

Bu yaklaşım:
- Özel SVG ikonlar yüklenebilir (marka/kurumsal ikonlar)
- Hızlı kullanım için ikon kütüphanesi referansı verilebilir
- Her ikisi de tanımlıysa yüklenen SVG öncelikli olur
- Frontend tek bir `renderCategoryIcon()` fonksiyonu ile her iki durumu yönetir

## 5. Validasyon Kuralları (Validation Rules)

### 5.1 SVG Dosya Yükleme Validasyonu

| Kural | Alan | Koşul | Hata Mesajı |
|-------|------|-------|-------------|
| Dosya uzantısı | icon | Yalnızca `.svg` uzantılı dosyalar | "Only SVG files are allowed for the icon field. Please upload a .svg file." |
| Dosya boyutu | icon | Maksimum 50 KB | "SVG icon file size must not exceed 50 KB." |
| XSS sanitizasyonu | icon | `<script>`, `on*` event handler'ları, `javascript:` URI'leri temizlenmeli | "The uploaded SVG contains potentially unsafe content and has been sanitized." |
| MIME tipi kontrolü | icon | Content-Type: `image/svg+xml` olmalı | "Invalid file type. Expected SVG (image/svg+xml)." |
| İkon sınıfı formatı | icon_class | Yalnızca harf, rakam, tire ve alt çizgi | "Icon class name can only contain letters, numbers, hyphens and underscores." |
| Renk formatı | icon_color | Geçerli CSS renk kodu (#hex, rgb(), hsl()) | "Please enter a valid CSS color value." |

**Python Validasyon Mantığı (Planlama Notu — Kod Oluşturulmayacak):**

```python
# category.py - validate() metodu içine eklenecek kurallar (planlama)
import re
import os

def validate_icon(self):
    """SVG ikon alanı validasyonu"""
    if self.icon:
        # 1. Uzantı kontrolü
        file_ext = os.path.splitext(self.icon)[1].lower()
        if file_ext != '.svg':
            frappe.throw(_("Only SVG files are allowed for the icon field. Please upload a .svg file."))

        # 2. Boyut kontrolü (50 KB = 51200 bytes)
        file_doc = frappe.get_doc("File", {"file_url": self.icon})
        if file_doc.file_size and file_doc.file_size > 51200:
            frappe.throw(_("SVG icon file size must not exceed 50 KB."))

        # 3. XSS sanitizasyonu
        svg_content = file_doc.get_content()
        if isinstance(svg_content, bytes):
            svg_content = svg_content.decode('utf-8')
        sanitized = sanitize_svg(svg_content)
        if sanitized != svg_content:
            file_doc.save_file(content=sanitized.encode('utf-8'))
            frappe.msgprint(_("The uploaded SVG has been sanitized for security."))

        # 4. icon_name otomatik doldurma
        self.icon_name = os.path.basename(self.icon)

    if self.icon_class:
        # 5. İkon sınıfı formatı
        if not re.match(r'^[a-zA-Z0-9_-]+$', self.icon_class):
            frappe.throw(_("Icon class name can only contain letters, numbers, hyphens and underscores."))
```

### 5.2 SVG XSS Sanitizasyon Detayları

⚠️ **KRİTİK GÜVENLİK NOTU:** SVG dosyaları XML tabanlıdır ve gömülü JavaScript içerebilir. Bu, Cross-Site Scripting (XSS) saldırı vektörü oluşturur.

**Tehlikeli SVG İçerik Örnekleri:**

```xml
<!-- Tehlikeli: Gömülü script -->
<svg xmlns="http://www.w3.org/2000/svg">
  <script>alert('XSS')</script>
  <circle r="50"/>
</svg>

<!-- Tehlikeli: Event handler -->
<svg xmlns="http://www.w3.org/2000/svg">
  <circle r="50" onmouseover="alert('XSS')"/>
</svg>

<!-- Tehlikeli: JavaScript URI -->
<svg xmlns="http://www.w3.org/2000/svg">
  <a xlink:href="javascript:alert('XSS')">
    <circle r="50"/>
  </a>
</svg>

<!-- Tehlikeli: foreignObject ile HTML enjeksiyonu -->
<svg xmlns="http://www.w3.org/2000/svg">
  <foreignObject>
    <body xmlns="http://www.w3.org/1999/xhtml">
      <script>alert('XSS')</script>
    </body>
  </foreignObject>
</svg>
```

**Sanitizasyon Kuralları:**

| Kaldırılacak Öğe | Açıklama | Örnek |
|-------------------|----------|-------|
| `<script>` etiketleri | Tüm script blokları | `<script>...</script>` |
| `on*` event handler'ları | Tüm olay işleyicileri | `onclick`, `onmouseover`, `onload`, `onerror` |
| `javascript:` URI'leri | href/xlink:href içinde | `<a href="javascript:...">` |
| `data:` URI'leri | href/xlink:href içinde (base64 payload) | `<a href="data:text/html,...">` |
| `<foreignObject>` | HTML enjeksiyon vektörü | `<foreignObject><body>...</body></foreignObject>` |
| `<iframe>` | Dış kaynak yükleme | `<iframe src="...">` |
| `<embed>` / `<object>` | Plugin enjeksiyonu | `<embed src="malicious.swf">` |
| `xlink:href` (harici) | Dış kaynak referansı | `<use xlink:href="http://evil.com/sprite.svg#icon">` |
| `<set>` / `<animate>` attributeName="href" | Animasyonla değer değiştirme | `<set attributeName="href" to="javascript:...">` |

**Önerilen Sanitizasyon Kütüphanesi:**

```python
# Python: bleach veya defusedxml kullanımı (planlama notu)
# pip install defusedxml

from defusedxml import ElementTree as ET
import re

def sanitize_svg(svg_content):
    """SVG içeriğini XSS saldırılarına karşı temizle"""
    # Tehlikeli etiketleri kaldır
    dangerous_tags = ['script', 'foreignObject', 'iframe', 'embed', 'object']
    for tag in dangerous_tags:
        svg_content = re.sub(
            rf'<{tag}[^>]*>.*?</{tag}>',
            '', svg_content, flags=re.DOTALL | re.IGNORECASE
        )

    # Event handler attribute'larını kaldır
    svg_content = re.sub(
        r'\s+on\w+\s*=\s*["\'][^"\']*["\']',
        '', svg_content, flags=re.IGNORECASE
    )

    # javascript: ve data: URI'lerini kaldır
    svg_content = re.sub(
        r'(href|xlink:href)\s*=\s*["\']javascript:[^"\']*["\']',
        '', svg_content, flags=re.IGNORECASE
    )
    svg_content = re.sub(
        r'(href|xlink:href)\s*=\s*["\']data:[^"\']*["\']',
        '', svg_content, flags=re.IGNORECASE
    )

    return svg_content
```

## 6. Frontend Planı (Frontend Plan)

### 6.1 İkon Render Stratejisi

Kategorilerin ikonlarını göstermek için birleşik bir render fonksiyonu kullanılacaktır:

**Öncelik Sırası:**
1. `icon` alanı doluysa → SVG dosyasını `<img>` etiketi ile render et
2. `icon_class` alanı doluysa → CSS ikon sınıfını `<i>` etiketi ile render et
3. Her ikisi de boşsa → Varsayılan kategori ikonu göster

**JavaScript Render Fonksiyonu (Planlama Notu):**

```javascript
// tradehub_catalog/public/js/category_icon.js (planlama notu — kod oluşturulmayacak)

/**
 * Kategori ikonunu render et
 * @param {Object} category - Kategori verisi {icon, icon_class, icon_color, category_name}
 * @param {Object} options - Render seçenekleri {size: '24px', className: ''}
 * @returns {string} HTML string
 */
function renderCategoryIcon(category, options = {}) {
    const size = options.size || '24px';
    const className = options.className || 'category-icon';
    const color = category.icon_color || '#333333';

    // 1. Yüklenen SVG ikon (öncelikli)
    if (category.icon) {
        return `<img
            src="${category.icon}"
            alt="${category.category_name} icon"
            class="${className}"
            style="width: ${size}; height: ${size};"
            loading="lazy"
        />`;
    }

    // 2. İkon kütüphanesi sınıfı
    if (category.icon_class) {
        return `<i
            class="${category.icon_class} ${className}"
            style="font-size: ${size}; color: ${color};"
            aria-label="${category.category_name}"
        ></i>`;
    }

    // 3. Varsayılan ikon (SVG inline)
    return `<svg
        class="${className} category-icon-default"
        width="${size}" height="${size}"
        viewBox="0 0 24 24"
        fill="${color}"
        aria-label="${category.category_name}"
    >
        <path d="M10 4H4c-1.1 0-2 .9-2 2v12c0 1.1.9 2 2 2h16c1.1 0 2-.9 2-2V8c0-1.1-.9-2-2-2h-8l-2-2z"/>
    </svg>`;
}
```

### 7.4 Mevcut Kategori Verileri İçin Toplu İkon Atama

Migration sonrası, mevcut kategorilere varsayılan ikon sınıfları atanabilir:

```python
# Bench console'da çalıştırılacak script (planlama notu)
# bench --site marketplace.local console

default_icons = {
    "Elektronik": "fas fa-microchip",
    "Bilgisayarlar": "fas fa-laptop",
    "Telefonlar": "fas fa-mobile-alt",
    "Giyim": "fas fa-tshirt",
    "Ev & Yaşam": "fas fa-home",
    "Spor & Outdoor": "fas fa-running",
    "Kozmetik": "fas fa-spa",
    "Otomotiv": "fas fa-car",
    "Kitap & Kırtasiye": "fas fa-book",
    "Gıda": "fas fa-utensils",
}

for category_name, icon_class in default_icons.items():
    if frappe.db.exists("Category", category_name):
        frappe.db.set_value("Category", category_name, "icon_class", icon_class)
        frappe.db.set_value("Category", category_name, "icon_color", "#333333")

frappe.db.commit()
```

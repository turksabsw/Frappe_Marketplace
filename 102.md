**Benzersizlik Kuralı:**
- `custom_category_name` + `seller` + `marketplace_category` kombinasyonu benzersiz olmalıdır
- Bu, aynı satıcının aynı marketplace kategorisi altında aynı isimde iki özel kategori oluşturmasını engeller
- Python `validate()` metodu ile kontrol edilecek:

```python
# seller_custom_category.py (planlama notu - kod oluşturulmayacak)
def validate(self):
    # Aynı satıcı + aynı marketplace kategorisi altında aynı isim kontrolü
    existing = frappe.db.exists("Seller Custom Category", {
        "custom_category_name": self.custom_category_name,
        "seller": self.seller,
        "marketplace_category": self.marketplace_category,
        "name": ("!=", self.name)
    })
    if existing:
        frappe.throw(
            _("You already have a custom category '{0}' under '{1}'").format(
                self.custom_category_name,
                self.marketplace_category_name
            )
        )

def before_insert(self):
    # seller alanını giriş yapan kullanıcının Seller Profile'ından otomatik ata
    if not self.seller:
        seller = frappe.db.get_value("Seller Profile", {"user": frappe.session.user}, "name")
        if seller:
            self.seller = seller
        else:
            frappe.throw(_("You must have a Seller Profile to create custom categories"))
```

def get_permission_query_conditions(user):
    """Satıcılar sadece kendi özel kategorilerini görsün"""
    if "System Manager" in frappe.get_roles(user):
        return ""

    seller = frappe.db.get_value("Seller Profile", {"user": user}, "name")
    if seller:
        return f"(`tabSeller Custom Category`.seller = '{seller}')"
    return "1=0"  # Seller profile yoksa hiçbir şey görme

def has_permission(doc, ptype, user):
    """Kayıt seviyesinde izin kontrolü"""
    if "System Manager" in frappe.get_roles(user):
        return True

    seller = frappe.db.get_value("Seller Profile", {"user": user}, "name")
    if seller and doc.seller == seller:
        return True
    return False

## 6. Frontend Planı (Frontend Plan)

### 6.1 Ağaç Seçici / Cascader UI Yaklaşımı

Kategori seçimi için **üç aşamalı cascader (basamaklı)** bileşen kullanılacaktır:

```
┌──────────────────────────────────────────────────────────────────────┐
│ Kategori Seçimi                                                       │
├──────────────────┬──────────────────┬──────────────────────────────────┤
│ Ana Kategori     │ Alt Kategori     │ Özel Alt Kategori (opsiyonel)    │
├──────────────────┼──────────────────┼──────────────────────────────────┤
│ > Elektronik     │ > Bilgisayarlar  │ > Gaming Laptoplar (benim)       │
│   Giyim          │   Telefonlar     │   İş Laptopları (benim)          │
│   Ev & Yaşam     │   Tabletler      │   + Yeni Özel Kategori Ekle      │
│   Spor & Outdoor │                  │                                  │
│                  │                  │                                  │
└──────────────────┴──────────────────┴──────────────────────────────────┘
```

**Uygulama Stratejisi:**

1. **1. Seviye (Ana Kategori):** Product Category listesi (`level: 0` veya `parent_product_category: null`, `enabled: 1`)
2. **2. Seviye (Alt Kategori):** Seçilen ana kategorinin alt kategorileri (`parent_product_category: {seçilen}`, `enabled: 1`)
3. **3. Seviye (Özel Alt Kategori):** Seçilen alt kategoriye bağlı Seller Custom Category listesi (`marketplace_category: {seçilen}`, `seller: {mevcut_satıcı}`, `is_active: 1`)

**Frappe Frappe Entegrasyonu:**

```javascript
// Listing form'unda cascader davranışı (planlama notu)
frappe.ui.form.on("Listing", {
    category: function(frm) {
        // Ana kategori değiştiğinde subcategory'yi sıfırla
        frm.set_value("subcategory", "");
        frm.set_value("seller_custom_category", "");

        // Subcategory filtresini güncelle
        frm.set_query("subcategory", function() {
            return {
                filters: {
                    parent_product_category: frm.doc.category,
                    enabled: 1
                }
            };
        });

        // Attribute set'i kategoriden otomatik doldur
        if (frm.doc.category) {
            frappe.db.get_value("Product Category", frm.doc.category,
                "attribute_set", function(r) {
                    if (r && r.attribute_set) {
                        frm.set_value("attribute_set", r.attribute_set);
                    }
                }
            );
        }
    },

    subcategory: function(frm) {
        // Alt kategori değiştiğinde özel kategoriyi sıfırla
        frm.set_value("seller_custom_category", "");

        // Seller custom category filtresini güncelle
        frm.set_query("seller_custom_category", function() {
            return {
                filters: {
                    marketplace_category: frm.doc.subcategory || frm.doc.category,
                    seller: frm.doc.seller,
                    is_active: 1
                }
            };
        });
    }
});
```

**Frappe List View Yapılandırması:**
- `in_list_view: 1` alanlar: `custom_category_name`, `seller_name`, `marketplace_category_name`, `is_active`
- `in_standard_filter: 1` alanlar: `seller`, `marketplace_category`, `is_active`, `tenant`
- Varsayılan filtre: `seller = {mevcut_satıcı}` (Seller rolü için)

### 6.3 Ağaç Görünümü (Tree View)

Product Category DocType zaten `is_tree: 1` olduğu için Frappe'nin yerleşik ağaç görünümü mevcut. Ek olarak:

- `/app/product-category?view=Tree` — Platform yöneticileri için tam ağaç görünümü
- Seller Custom Category'ler ağaç görünümünde gösterilmez (ayrı bir DocType olduğu için)
- Bunun yerine satıcı panelinde düz liste olarak yönetilir

## 7. Uygulama Sonrası Adımlar (Post-Implementation Steps)

### 7.1 Zorunlu Komutlar

```bash
# 1. Veritabanı migrasyonu (yeni DocType tablosu oluşturma)
bench --site marketplace.local migrate

# 2. Önbellek temizleme
bench --site marketplace.local clear-cache

# 3. İzin güncelleme
bench --site marketplace.local setup-permissions

# Kısa yol:
bench migrate && bench clear-cache
```

### 7.2 Veri Migrasyonu Adımları

1. **Category → Product Category eşleme tablosu oluştur**
   - Mevcut Category kayıtlarını Product Category ile eşle (category_name bazlı)
   - Eşleşmeyen Category kayıtlarını Product Category'ye kopyala

2. **Listing DocType güncelle**
   - `category` alanının `options` değerini `Product Category` olarak değiştir
   - `subcategory` alanının `options` değerini `Product Category` olarak değiştir
   - Mevcut Listing kayıtlarındaki category referanslarını yeni Product Category'lere güncelle

3. **PIM Product DocType güncelle**
   - `primary_category` alanının `options` değerini `Product Category` olarak değiştir
   - Mevcut kayıtların referanslarını güncelle

### 7.3 Test Kontrol Listesi

- [ ] Seller Custom Category oluşturma (Satici rolü ile)
- [ ] Satıcının sadece kendi özel kategorilerini görmesi (permission query)
- [ ] Aynı satıcı + aynı marketplace kategorisi altında aynı isim engelleme
- [ ] Aktif olmayan marketplace kategorisine bağlama engelleme
- [ ] System Manager'ın tüm özel kategorileri görmesi
- [ ] Listing form'unda cascader kategori seçimi
- [ ] Subcategory'nin ana kategoriye göre filtrelenmesi
- [ ] Seller Custom Category'nin subcategory'ye göre filtrelenmesi
- [ ] is_active = 0 olan özel kategorilerin cascader'da görünmemesi
- [ ] Tenant izolasyonunun doğru çalışması
- [ ] Ürün bağlantılı özel kategorinin silinememesi
- [ ] Satıcı başına 50 özel kategori limitinin çalışması

### 7.4 Performans Dikkat Noktaları

- Product Category ağaç sorgularında `lft`/`rgt` index'lerinin mevcut olduğunu doğrula
- Seller Custom Category için `seller` + `marketplace_category` + `is_active` composite index ekle
- Cascader API çağrılarında cache kullan (Redis/Frappe cache)

## 1. Overview

### 1.1 Purpose

TradeHub Marketplace requires a self-service advertising platform that allows sellers to promote their products through paid placements. The system enables sellers to create ad campaigns, bid on placements, manage budgets through a pre-funded wallet, and track performance - all within the existing Frappe/ERPNext framework.

### 1.2 Scope

The Ad Management System introduces **14 new DocTypes** in the `tradehub_marketing` app:

| # | DocType | Type | Purpose |
|---|---------|------|---------|
| 1 | Ad Campaign | Standalone | Main campaign record with budget, targeting, schedule |
| 2 | Ad Campaign Item | Child Table | Products/listings promoted within a campaign |
| 3 | Ad Keyword | Child Table | Keywords for search-triggered ads |
| 4 | Ad Creative | Standalone | Creative assets (images, text) with approval workflow |
| 5 | Ad Placement | Standalone | Available ad slots on the marketplace |
| 6 | Banner Booking | Standalone | Fixed-time banner reservations (CPT model) |
| 7 | Ad Event Log | Standalone | Impression/click/conversion event tracking |
| 8 | Ad Wallet | Standalone | Prepaid advertising wallet per seller |
| 9 | Ad Wallet Transaction | Child Table | Transaction history for wallet operations |
| 10 | Ad Invoice | Standalone | Monthly billing summaries |
| 11 | Ad Invoice Item | Child Table | Line items on invoices |
| 12 | Ad Quality Score | Standalone | Quality score calculation per product/keyword |
| 13 | Ad Fraud Rule | Standalone | Configurable fraud detection rules |
| 14 | Ad Fraud Alert | Standalone | Fraud detection alert records |

### 1.3 Design Principles

1. **Mirror existing patterns**: Ad Wallet mirrors `Seller Balance`; Ad Campaign extends the `Campaign` DocType pattern
2. **Wallet-first billing**: Sellers pre-fund wallets; costs deducted per-click/impression in real time
3. **Second-price auction**: Winners pay $0.01 above the second-highest bid (fair pricing)
4. **Admin approval gates**: All creatives and new campaigns require admin review before going live
5. **Fraud-first design**: Click fraud detection is built into the core event processing pipeline
6. **Tenant isolation**: All DocTypes include `tenant` field with proper `fetch_from` chains

### 1.4 Current State Analysis

The existing `tradehub_marketing` app provides:

- **Campaign** DocType: Discount/promotional campaigns with budget tracking, status workflow (Draft -> Scheduled -> Active -> Paused -> Ended -> Cancelled), seller/tenant association, analytics counters. This is the baseline pattern for Ad Campaign but lacks bidding, auction, and approval mechanisms.
- **Coupon** / **Coupon Product Item** / **Coupon Category Item**: Child table pattern with Link + fetch_from for product/category associations.
- **Seller Balance** + **Seller Balance Transaction** (in `tradehub_seller`): Wallet pattern with available/pending/held/reserved balance tiers, transaction logging via child table, db_set() atomic mutations. This is the reference architecture for Ad Wallet but reversed (spending not earning).

### 1.5 Integration Points

| System | Integration |
|--------|-------------|
| `Seller Profile` | Seller identification, eligibility checks |
| `Seller Balance` | Cross-reference for seller financial health |
| `Listing` | Products being promoted in ad campaigns |
| `Product` / `Product Category` | Catalog references for targeting |
| `Campaign` | Potential cross-promotion with discount campaigns |
| `Moderation Case` | Creative approval follows moderation queue pattern |
| `Tenant` | Multi-tenant isolation for all ad operations |
| `Sales Channel` | Channel-specific ad placements |

## 2. Reference Platforms

### 2.1 Amazon Advertising (Sponsored Products)

| Feature | Amazon Implementation | TradeHub Adaptation |
|---------|----------------------|---------------------|
| Campaign Types | Sponsored Products, Sponsored Brands, Sponsored Display | Sponsored Listing (CPC), Banner Ad (CPM/CPT), Featured Placement (CPC) |
| Bidding | Dynamic bids (up/down), fixed bids, suggested bids | Second-price auction with quality score, min bid per placement |
| Targeting | Keyword targeting, product targeting, category targeting | Keyword targeting + category targeting + placement targeting |
| Budget | Daily budget, lifetime budget, budget caps | Daily budget + total budget, auto-pause on wallet depletion |
| Reporting | CTR, ACoS, ROAS, impressions, clicks, conversions | Same metrics + fraud-adjusted stats |
| Billing | Post-pay (invoice monthly) | Pre-pay wallet + monthly invoice summary |

### 2.2 Alibaba P4P (Pay-for-Performance)

| Feature | Alibaba Implementation | TradeHub Adaptation |
|---------|------------------------|---------------------|
| Ad Types | P4P keyword ads, banner ads, featured supplier | Keyword CPC ads, banner booking (CPT), featured listing |
| Wallet | Pre-funded wallet, minimum deposit required | Ad Wallet with minimum balance threshold |
| Quality Score | Click-through rate, relevance, landing page quality | CTR history + creative quality + seller score + relevance |
| Fraud Protection | Click protection, refund for invalid clicks | Same-IP dedup, bot filtering, coordinated fraud detection |
| Dashboard | Real-time spend tracking, keyword performance | Seller dashboard with CTR, ACoS, ROAS, spend tracking |

### 2.3 eBay Promoted Listings

| Feature | eBay Implementation | TradeHub Adaptation |
|---------|---------------------|---------------------|
| Model | Cost-per-sale (ad fee on completed sale) | CPC primary, CPA optional for high-value categories |
| Ad Rate | Seller sets ad rate (% of sale price) | Seller sets CPC bid per keyword/placement |
| Placement | Search results, browse pages, item pages | Search results, category pages, homepage, product detail pages |
| Reporting | Impressions, clicks, sold items, ad fees | Impressions, clicks, conversions, revenue, costs |

### 2.4 Key Decisions for TradeHub

Based on reference platform analysis:

1. **Primary model**: CPC (cost-per-click) for keyword and listing ads - proven, simple, seller-friendly
2. **Secondary model**: CPT (cost-per-time) for banner bookings - fixed-price, predictable for display ads
3. **Tertiary model**: CPM (cost-per-mille) for awareness campaigns - available for brand placements
4. **Billing**: Pre-funded wallet (Alibaba model) - reduces credit risk for the marketplace
5. **Auction**: Second-price auction with quality score (Google/Amazon model) - fair and optimizes for relevance
6. **Approval**: All creatives require admin approval before serving (quality control)

## 3. System Architecture

### 3.1 High-Level Flow

```
Seller funds Ad Wallet
        |
        v
Seller creates Ad Campaign (status: Draft)
        |
        v
Seller adds Campaign Items (products + bids)
        |
        v
Seller uploads/selects Ad Creatives
        |
        v
Seller submits campaign for review (status: Pending Approval)
        |
        v
Admin reviews & approves (status: Approved -> Scheduled/Active)
        |
        v
Auction Engine matches ads to placements on page load
        |
        v
Ad served -> Impression logged (Ad Event Log)
        |
        v
User clicks ad -> Click logged -> Fraud check -> Wallet deducted
        |
        v
User completes purchase -> Conversion logged
        |
        v
Monthly: Ad Invoice generated from Ad Event Logs
```

All new DocTypes belong to `tradehub_marketing`:
- **Module**: `TradeHub Marketing`
- **App directory**: `frappe-bench/apps/tradehub_marketing/tradehub_marketing/tradehub_marketing/doctype/`

The core advertising campaign record. Extends the existing `Campaign` DocType pattern with bidding, auction, and approval mechanisms.

### 4.1 DocType Metadata

```json
  "name": "Ad Campaign",
  "module": "TradeHub Marketing",

  "naming_rule": "By naming series",
  "allow_rename": 0,
  "engine": "InnoDB",
  "track_changes": 1,
  "quick_entry": 0,
  "sort_field": "modified",
  "sort_order": "DESC",
  "title_field": "campaign_title",
  "search_fields": "campaign_title, campaign_type, status, seller, seller_name",
  "show_name_in_global_search": 1
}
```

### 4.2 Status Workflow

```
Draft --> Pending Approval --> Approved --> Scheduled --> Active --> Paused --> Ended
  |            |                                          |          |
  |            v                                          |          v
  |         Rejected --> Draft (resubmit)                 +-------> Cancelled
  |                                                       |
  v                                                       v
Cancelled                                               Ended (auto on end_date or budget depletion)
```

**State Colors** (following Campaign pattern):

| Status | Color | Description |
|--------|-------|-------------|
| Draft | gray | Campaign being edited by seller |
| Pending Approval | orange | Submitted for admin review |
| Rejected | red | Admin rejected - needs revision |
| Approved | blue | Admin approved - awaiting schedule |
| Scheduled | blue | Approved and waiting for start_date |
| Active | green | Currently serving ads |
| Paused | yellow | Temporarily paused by seller or system |
| Budget Depleted | red | Auto-paused due to zero wallet/budget |
| Ended | gray | Reached end_date or manually ended |
| Cancelled | red | Permanently cancelled |

```python
# ad_campaign.py

class AdCampaign(Document):
    def validate(self):
        self.validate_dates()
        self.validate_budget()
        self.validate_wallet_balance()
        self.validate_campaign_items()
        self.calculate_remaining_budget()

    def validate_dates(self):
        """Ensure start_date < end_date and end_date is in the future."""
        if self.end_date and self.start_date >= self.end_date:
            frappe.throw(_("End Date must be after Start Date"))
        if self.end_date and getdate(self.end_date) < getdate(nowdate()):
            frappe.throw(_("End Date cannot be in the past"))

    def validate_budget(self):
        """Ensure daily_budget <= total_budget and minimums are met."""
        if self.daily_budget > self.total_budget:
            frappe.throw(_("Daily Budget cannot exceed Total Budget"))
        if self.daily_budget < 10:  # Minimum 10 TRY daily budget
            frappe.throw(_("Minimum daily budget is 10 {0}").format(self.currency))

    def validate_wallet_balance(self):
        """Check Ad Wallet has sufficient balance for at least 1 day."""
        if self.ad_wallet:
            wallet = frappe.get_doc("Ad Wallet", self.ad_wallet)
            if wallet.available_balance < self.daily_budget:
                frappe.throw(_("Insufficient Ad Wallet balance. Minimum {0} {1} required.").format(
                    self.daily_budget, self.currency
                ))

    def submit_for_review(self):
        """Seller submits campaign for admin approval."""
        self.status = "Pending Approval"
        self.submitted_at = now_datetime()
        self.save()

    def approve(self, notes=None):
        """Admin approves the campaign."""
        self.status = "Approved"
        self.reviewed_by = frappe.session.user
        self.reviewed_at = now_datetime()
        self.approval_notes = notes
        if self.auto_activate and getdate(self.start_date) <= getdate(nowdate()):
            self.status = "Active"
        elif self.start_date:
            self.status = "Scheduled"
        self.save()

    def reject(self, reason):
        """Admin rejects the campaign."""
        self.status = "Rejected"
        self.reviewed_by = frappe.session.user
        self.reviewed_at = now_datetime()
        self.rejection_reason = reason
        self.save()

    def pause(self, reason="Manual"):
        """Pause the campaign."""
        self.status = "Paused"
        self.pause_reason = reason
        self.save()

    def check_daily_budget(self):
        """Check if daily budget has been reached."""
        if self.today_spent >= self.daily_budget:
            self.pause("Daily Budget Reached")
            return False
        return True

    def check_total_budget(self):
        """Check if total budget has been reached."""
        if self.spent_amount >= self.total_budget:
            self.status = "Budget Depleted"
            self.pause_reason = "Budget Depleted"
            self.save()
            return False
        return True
```

| Match Type | Behavior | Example Keyword | Triggers On |
|-----------|----------|-----------------|-------------|

### 6.4 Negative Keyword Logic

When `is_negative = 1`:
- The keyword acts as an exclusion filter
- If a user's search matches a negative keyword, the campaign's ads are NOT served for that search
- Negative keywords support **Phrase** and **Exact** match types only (Broad negative is too aggressive)
- Negative keywords do not accumulate metrics (impressions, clicks, etc.)

### 6.5 Keyword Validation Rules

```python
def validate_keywords(self):
    """Validate keyword list in Ad Campaign."""
    seen_keywords = set()
    for row in self.campaign_keywords:
        # Normalize keyword
        row.keyword = row.keyword.strip().lower()

        # Check for duplicates (same keyword + match_type combination)
        key = f"{row.keyword}|{row.match_type}|{row.is_negative}"
        if key in seen_keywords:
            frappe.throw(_(
                "Duplicate keyword '{0}' with match type '{1}' at row {2}"
            ).format(row.keyword, row.match_type, row.idx))
        seen_keywords.add(key)

        # Validate keyword length
        if len(row.keyword) < 2:
            frappe.throw(_("Keyword must be at least 2 characters at row {0}").format(row.idx))
        if len(row.keyword) > 100:
            frappe.throw(_("Keyword must be at most 100 characters at row {0}").format(row.idx))

        # Validate bid amount
        if not row.is_negative and row.bid_amount <= 0:
            frappe.throw(_("Bid amount must be positive for active keyword at row {0}").format(row.idx))

        # Negative keywords cannot have Broad match
        if row.is_negative and row.match_type == "Broad":
            frappe.throw(_(
                "Negative keywords cannot use Broad match type at row {0}. Use Phrase or Exact."
            ).format(row.idx))
```

```python
def validate_media(self):
    """Validate creative media assets against rules."""
    rules = {
        "Product Image": {"max_size_kb": 2048, "formats": ["jpg", "jpeg", "png", "webp"]},
        "Banner": {"max_size_kb": 5120, "formats": ["jpg", "jpeg", "png", "webp", "gif"]},
        "Video": {"max_size_kb": 51200, "formats": ["mp4", "webm"]},
        "Native": {"max_size_kb": 2048, "formats": ["jpg", "jpeg", "png", "webp"]},
        "Logo": {"max_size_kb": 1024, "formats": ["png", "svg"]}
    }

    if self.creative_type == "Text Ad":
        # Text ads only need headline validation
        if not self.headline:
            frappe.throw(_("Headline is required for Text Ads"))
        if len(self.headline) > 60:
            frappe.throw(_("Headline must be 60 characters or less"))
        if self.body_text and len(self.body_text) > 150:
            frappe.throw(_("Body text must be 150 characters or less"))
        return

    rule = rules.get(self.creative_type)
    if not rule:
        return

    # Check file format
    if self.file_format and self.file_format.lower() not in rule["formats"]:
        frappe.throw(_("File format '{0}' is not allowed for {1}. Allowed: {2}").format(
            self.file_format, self.creative_type, ", ".join(rule["formats"])
        ))

    # Check file size
    if self.file_size and self.file_size > rule["max_size_kb"]:
        frappe.throw(_("File size {0} KB exceeds maximum {1} KB for {2}").format(
            self.file_size, rule["max_size_kb"], self.creative_type
        ))
```

### 9.3 Conflict Detection Algorithm

Banner bookings require server-side conflict detection to prevent double-booking the same placement slot during overlapping date ranges.

```python
# ad_management/banner_booking.py

class BannerBooking(Document):
    def validate(self):
        self.validate_dates()
        self.calculate_costs()
        self.check_creative_approval()
        self.check_conflicts()

    def validate_dates(self):
        if self.start_date > self.end_date:
            frappe.throw(_("Start Date cannot be after End Date"))
        if self.start_date < frappe.utils.today():
            frappe.throw(_("Start Date cannot be in the past"))
        self.duration_days = frappe.utils.date_diff(self.end_date, self.start_date) + 1

    def calculate_costs(self):
        self.total_cost = self.daily_rate * self.duration_days
        discount = self.discount_percent or 0
        self.final_cost = self.total_cost * (1 - discount / 100)

    def check_conflicts(self):
        """Detect overlapping bookings for the same placement + slot."""
        if self.status in ("Cancelled", "Rejected", "Draft"):
            return

        conflicts = frappe.db.sql("""
            SELECT name, seller, start_date, end_date
            FROM `tabBanner Booking`
            WHERE placement = %(placement)s
              AND slot_index = %(slot_index)s
              AND name != %(name)s
              AND status NOT IN ('Cancelled', 'Rejected', 'Draft')
              AND start_date <= %(end_date)s
              AND end_date >= %(start_date)s
        """, {
            "placement": self.placement,
            "slot_index": self.slot_index,
            "name": self.name or "",
            "start_date": self.start_date,
            "end_date": self.end_date
        }, as_dict=True)

        if conflicts:
            conflict = conflicts[0]
            frappe.throw(
                _("Conflict: Booking {0} by {1} already occupies slot {2} "
                  "from {3} to {4}").format(
                    conflict.name, conflict.seller,
                    self.slot_index, conflict.start_date, conflict.end_date
                ),
                title=_("Booking Conflict")
            )

    def on_submit(self):
        """Deduct cost from Ad Wallet on approval."""
        if self.status == "Approved":
            self.deduct_from_wallet()

    def deduct_from_wallet(self):
        wallet = frappe.get_doc("Ad Wallet", {"seller": self.seller})
        if wallet.available_balance < self.final_cost:
            frappe.throw(_("Insufficient Ad Wallet balance. Required: {0}, Available: {1}").format(
                self.final_cost, wallet.available_balance
            ))
        wallet.deduct_balance(self.final_cost, "Banner Booking", self.name,
                              f"Banner booking: {self.booking_title}")
```

### 11.3 Python API (mirroring Seller Balance pattern)

```python
# ad_management/ad_wallet.py

class AdWallet(Document):

    @staticmethod
    def get_or_create(seller):
        """Factory method to get existing wallet or create new one."""
        if frappe.db.exists("Ad Wallet", {"seller": seller}):
            return frappe.get_doc("Ad Wallet", {"seller": seller})
        wallet = frappe.new_doc("Ad Wallet")
        wallet.seller = seller
        wallet.status = "Active"
        wallet.insert(ignore_permissions=True)
        return wallet

    def add_funds(self, amount, method, reference_type=None, reference_name=None, description=None):
        """Credit funds to the wallet. Uses db_set() for atomic mutation."""
        if amount <= 0:
            frappe.throw(_("Amount must be positive"))

        new_balance = self.available_balance + amount
        self.db_set("available_balance", new_balance)
        self.db_set("total_balance", self.total_balance + amount)
        self.db_set("lifetime_deposits", self.lifetime_deposits + amount)
        self.db_set("last_updated", frappe.utils.now())
        self.db_set("last_fund_date", frappe.utils.now())
        self.db_set("last_fund_amount", amount)
        self.db_set("last_fund_method", method)

        # Reset low-balance alerts
        if new_balance > self.minimum_balance_threshold:
            self.db_set("low_balance_alert_sent", 0)
        if self.campaigns_auto_paused and new_balance > self.auto_pause_threshold:
            self.db_set("campaigns_auto_paused", 0)
            self.resume_paused_campaigns()

        self._log_transaction("Credit", amount, new_balance,
                              reference_type, reference_name,
                              description or f"Funds added via {method}")

    def deduct_balance(self, amount, reference_type=None, reference_name=None, description=None):
        """Debit funds from available balance for ad spend."""
        if amount <= 0:
            frappe.throw(_("Amount must be positive"))
        if self.available_balance < amount:
            frappe.throw(_("Insufficient balance"))

        new_balance = self.available_balance - amount
        self.db_set("available_balance", new_balance)
        self.db_set("total_balance", self.total_balance - amount)
        self.db_set("lifetime_spend", self.lifetime_spend + amount)
        self.db_set("last_updated", frappe.utils.now())

        self._log_transaction("Debit", amount, new_balance,
                              reference_type, reference_name,
                              description or "Ad spend deduction")

        # Check thresholds after deduction
        self._check_balance_thresholds(new_balance)

    def reserve_for_campaign(self, amount, campaign_name):
        """Reserve daily budget from available into reserved."""
        if self.available_balance < amount:
            frappe.throw(_("Insufficient balance for reservation"))

        self.db_set("available_balance", self.available_balance - amount)
        self.db_set("reserved_balance", self.reserved_balance + amount)
        self.db_set("last_updated", frappe.utils.now())

        self._log_transaction("Hold", amount, self.available_balance,
                              "Ad Campaign", campaign_name,
                              f"Daily budget reservation for {campaign_name}")

    def release_reservation(self, amount, campaign_name):
        """Release unused reserved funds back to available."""
        release_amount = min(amount, self.reserved_balance)
        self.db_set("reserved_balance", self.reserved_balance - release_amount)
        self.db_set("available_balance", self.available_balance + release_amount)
        self.db_set("last_updated", frappe.utils.now())

        self._log_transaction("Release", release_amount, self.available_balance,
                              "Ad Campaign", campaign_name,
                              f"Released unused budget from {campaign_name}")

    def refund(self, amount, reference_type=None, reference_name=None, description=None):
        """Refund for invalid clicks, cancelled bookings, etc."""
        new_balance = self.available_balance + amount
        self.db_set("available_balance", new_balance)
        self.db_set("total_balance", self.total_balance + amount)
        self.db_set("lifetime_refunds", self.lifetime_refunds + amount)
        self.db_set("last_updated", frappe.utils.now())

        self._log_transaction("Credit", amount, new_balance,
                              reference_type, reference_name,
                              description or "Refund")

    def _log_transaction(self, txn_type, amount, running_balance,
                         ref_type=None, ref_name=None, description=None):
        """Append a row to the transaction_records child table."""
        self.append("transaction_records", {
            "transaction_date": frappe.utils.now(),
            "transaction_type": txn_type,
            "amount": amount,
            "running_balance": running_balance,
            "reference_type": ref_type,
            "reference_name": ref_name,
            "description": description
        })
        self.save(ignore_permissions=True)

    def _check_balance_thresholds(self, current_balance):
        """Trigger alerts and auto-pause when balance is low."""
        # Low balance alert
        if (current_balance <= self.minimum_balance_threshold
                and not self.low_balance_alert_sent):
            self.db_set("low_balance_alert_sent", 1)
            frappe.publish_realtime(
                "ad_wallet_low_balance",
                {"seller": self.seller, "balance": current_balance},
                user=frappe.db.get_value("Seller Profile", self.seller, "user")
            )

        # Auto-pause campaigns at zero/threshold
        if current_balance <= self.auto_pause_threshold:
            self.db_set("campaigns_auto_paused", 1)
            self.pause_all_campaigns()

    def pause_all_campaigns(self):
        """Pause all active campaigns for this seller."""
        campaigns = frappe.get_all("Ad Campaign",
            filters={"seller": self.seller, "status": "Active"},
            pluck="name")
        for campaign_name in campaigns:
            frappe.db.set_value("Ad Campaign", campaign_name, "status", "Paused")
            frappe.db.set_value("Ad Campaign", campaign_name,
                                "pause_reason", "Insufficient ad wallet balance")

    def resume_paused_campaigns(self):
        """Resume campaigns that were auto-paused due to balance."""
        campaigns = frappe.get_all("Ad Campaign",
            filters={
                "seller": self.seller,
                "status": "Paused",
                "pause_reason": "Insufficient ad wallet balance"
            }, pluck="name")
        for campaign_name in campaigns:
            frappe.db.set_value("Ad Campaign", campaign_name, "status", "Active")
            frappe.db.set_value("Ad Campaign", campaign_name, "pause_reason", "")
```

```python
# ad_management/tasks.py

def generate_monthly_ad_invoices():
    """Scheduled task: runs on 1st of each month for previous month."""
    today = frappe.utils.today()
    period_end = frappe.utils.add_days(today, -1)  # Last day of prev month
    period_start = frappe.utils.get_first_day(period_end)

    sellers_with_spend = frappe.db.sql("""
        SELECT DISTINCT seller
        FROM `tabAd Event Log`
        WHERE event_timestamp BETWEEN %(start)s AND %(end)s
          AND is_billable = 1
          AND actual_cost > 0
    """, {"start": period_start, "end": period_end}, as_dict=True)

    for row in sellers_with_spend:
        generate_invoice_for_seller(row.seller, period_start, period_end)

def generate_invoice_for_seller(seller, period_start, period_end):
    """Create Ad Invoice with itemized charges per campaign."""
    invoice = frappe.new_doc("Ad Invoice")
    invoice.seller = seller
    invoice.billing_period_start = period_start
    invoice.billing_period_end = period_end
    invoice.invoice_date = frappe.utils.today()
    invoice.billing_month = frappe.utils.formatdate(period_start, "MMMM YYYY")
    invoice.days_in_period = frappe.utils.date_diff(period_end, period_start) + 1

    # Aggregate charges per campaign
    campaign_charges = frappe.db.sql("""
        SELECT
            campaign,
            pricing_model,
            COUNT(CASE WHEN event_type='Impression' THEN 1 END) as impressions,
            COUNT(CASE WHEN event_type='Click' THEN 1 END) as clicks,
            COUNT(CASE WHEN event_type='Conversion' THEN 1 END) as conversions,
            SUM(actual_cost) as total_cost,
            SUM(CASE WHEN is_valid=0 THEN actual_cost ELSE 0 END) as invalid_cost
        FROM `tabAd Event Log`
        WHERE seller = %(seller)s
          AND event_timestamp BETWEEN %(start)s AND %(end)s
          AND is_billable = 1
        GROUP BY campaign, pricing_model
    """, {
        "seller": seller,
        "start": period_start,
        "end": period_end
    }, as_dict=True)

    for charge in campaign_charges:
        invoice.append("items", {
            "campaign": charge.campaign,
            "pricing_model": charge.pricing_model,
            "impressions": charge.impressions,
            "clicks": charge.clicks,
            "conversions": charge.conversions,
            "total_charge": charge.total_cost,
            "fraud_adjustment": -abs(charge.invalid_cost)
        })

    invoice.calculate_totals()
    invoice.insert(ignore_permissions=True)
```

### 15.3 Quality Score Calculation

```python
# ad_management/quality_score.py

# Component weights (must sum to 1.0)
QUALITY_SCORE_WEIGHTS = {
    "ctr_score": 0.35,       # Historical click-through rate (most important)
    "relevance_score": 0.25,  # Keyword-to-product relevance
    "creative_score": 0.15,   # Creative quality and compliance
    "landing_page_score": 0.15, # Listing completeness and quality
    "seller_score": 0.10      # Seller reputation
}

def calculate_quality_score(seller, listing=None, keyword=None, placement=None):
    """Calculate or update quality score for a seller/listing/keyword combo."""

    qs = get_or_create_quality_score(seller, listing, keyword, placement)

    # 1. CTR Score (0-10)
    qs.ctr_score = calculate_ctr_score(qs)

    # 2. Relevance Score (0-10)
    qs.relevance_score = calculate_relevance_score(listing, keyword)

    # 3. Creative Score (0-10)
    qs.creative_score = calculate_creative_score(seller, listing)

    # 4. Landing Page Score (0-10)
    qs.landing_page_score = calculate_landing_page_score(listing)

    # 5. Seller Score (0-10)
    qs.seller_score = calculate_seller_score(seller)

    # Weighted average
    qs.overall_score = sum(
        getattr(qs, component) * weight
        for component, weight in QUALITY_SCORE_WEIGHTS.items()
    )

    # Tier classification
    if qs.overall_score >= 8:
        qs.score_tier = "Excellent (8-10)"
    elif qs.overall_score >= 6:
        qs.score_tier = "Good (6-8)"
    elif qs.overall_score >= 4:
        qs.score_tier = "Average (4-6)"
    elif qs.overall_score >= 2:
        qs.score_tier = "Below Average (2-4)"
    else:
        qs.score_tier = "Poor (0-2)"

    # Ad Rank multiplier (maps 0-10 score to 0.5-1.5 multiplier)
    qs.ad_rank_multiplier = 0.5 + (qs.overall_score / 10.0)

    qs.last_calculated = frappe.utils.now()
    qs.save(ignore_permissions=True)
    return qs

def calculate_ctr_score(qs):
    """CTR Score: compare actual CTR to placement benchmark."""
    if qs.historical_impressions < 100:
        return 5.0  # Default for new ads (insufficient data)

    actual_ctr = qs.historical_ctr or 0
    benchmark = qs.benchmark_ctr or 1.0
    ratio = actual_ctr / benchmark if benchmark > 0 else 1.0
    qs.ctr_vs_benchmark = ratio

    # Score: 5 at benchmark, 10 at 2x benchmark, 0 at 0 CTR
    return min(10.0, max(0.0, ratio * 5.0))

def calculate_relevance_score(listing, keyword):
    """Relevance: keyword presence in listing title, description, category."""
    if not listing or not keyword:
        return 5.0

    listing_doc = frappe.get_cached_doc("Listing", listing)
    score = 0.0
    keyword_lower = keyword.lower()

    # Title match (most important)
    if keyword_lower in (listing_doc.listing_title or "").lower():
        score += 4.0
    # Description match
    if keyword_lower in (listing_doc.description or "").lower():
        score += 3.0
    # Category match
    category_name = frappe.get_value("Product Category",
                                      listing_doc.category, "category_name") or ""
    if keyword_lower in category_name.lower():
        score += 3.0

    return min(10.0, score)

def calculate_landing_page_score(listing):
    """Landing page quality: listing completeness, images, reviews."""
    if not listing:
        return 5.0

    listing_doc = frappe.get_cached_doc("Listing", listing)
    score = 0.0

    # Has title and description
    if listing_doc.listing_title:
        score += 2.0
    if listing_doc.description and len(listing_doc.description) > 50:
        score += 2.0
    # Has images
    image_count = frappe.db.count("Listing Image", {"parent": listing})
    if image_count >= 3:
        score += 3.0
    elif image_count >= 1:
        score += 1.5
    # Has price
    if listing_doc.price and listing_doc.price > 0:
        score += 1.5
    # In stock
    if listing_doc.stock_quantity and listing_doc.stock_quantity > 0:
        score += 1.5

    return min(10.0, score)

def calculate_seller_score(seller):
    """Seller reputation from Seller Score / Seller Metrics."""
    seller_score = frappe.db.get_value("Seller Score", {"seller": seller}, "overall_score")
    if seller_score:
        return min(10.0, seller_score * 2)  # Map 0-5 star to 0-10
    return 5.0  # Default for unscored sellers
```

## 17. Auction Engine Algorithm

The auction engine is the core ad serving mechanism. It runs on every page load where ad placements exist, selecting which ads to show based on a second-price auction with quality score weighting.

### 17.1 Ad Rank Formula

```
Ad Rank = Max CPC Bid x Quality Score Multiplier
```

Where:
- **Max CPC Bid**: The seller's maximum cost-per-click bid for the keyword/placement
- **Quality Score Multiplier**: Derived from `Ad Quality Score.ad_rank_multiplier` (range 0.5 - 1.5)

**Example:**

| Advertiser | Max CPC Bid | Quality Score | QS Multiplier | Ad Rank | Position |
|------------|-------------|---------------|---------------|---------|----------|
| Seller A | 2.00 TRY | 8.0/10 | 1.30 | 2.60 | 1st |
| Seller B | 3.00 TRY | 4.0/10 | 0.70 | 2.10 | 2nd |
| Seller C | 1.50 TRY | 9.0/10 | 1.40 | 2.10 | 3rd (tie-break: higher QS) |
| Seller D | 2.50 TRY | 3.0/10 | 0.65 | 1.63 | 4th (below minimum, not shown) |

### 17.2 Second-Price Auction

Winners pay the minimum amount needed to beat the next-highest bidder, plus a small increment:

```
Actual CPC = (Ad Rank of next bidder / Winner's QS Multiplier) + 0.01 TRY
```

**Example (from above):**
- Seller A wins position 1. Actual CPC = (2.10 / 1.30) + 0.01 = 1.62 + 0.01 = **1.63 TRY** (not 2.00)
- Seller B wins position 2. Actual CPC = (2.10 / 0.70) + 0.01 = 3.00 + 0.01 = **3.00 TRY** (capped at max bid)

**Rules:**
1. Actual CPC is always <= Max CPC Bid (never exceeds seller's maximum)
2. Actual CPC is always >= placement's `min_cpc_bid`
3. If only one eligible bidder, they pay `min_cpc_bid`
4. Tie-breaking: higher quality score wins; if still tied, earlier campaign wins

### 17.3 Auction Flow

```python
# ad_management/auction_engine.py

def run_auction(placement_name, context):
    """
    Run ad auction for a specific placement.

    Args:
        placement_name: Ad Placement name
        context: dict with search_query, category, page_url, user_info

    Returns:
        list of winning ads with actual CPC
    """
    placement = frappe.get_cached_doc("Ad Placement", placement_name)

    if placement.status != "Active":
        return []

    # 1. Check for active banner bookings first (CPT takes priority)
    active_booking = get_active_banner_booking(placement_name)
    if active_booking and placement.rotation_strategy == "Fixed (Booking)":
        return [format_booking_result(active_booking)]

    # 2. Get eligible campaigns
    eligible = get_eligible_campaigns(placement, context)

    if not eligible:
        return []

    # 3. Calculate Ad Rank for each candidate
    candidates = []
    for campaign_item in eligible:
        quality_score = get_quality_score(
            campaign_item.seller,
            campaign_item.listing,
            context.get("search_query"),
            placement_name
        )

        ad_rank = campaign_item.bid_amount * quality_score.ad_rank_multiplier

        candidates.append({
            "campaign": campaign_item.campaign,
            "campaign_item": campaign_item.name,
            "seller": campaign_item.seller,
            "listing": campaign_item.listing,
            "creative": campaign_item.creative,
            "bid_amount": campaign_item.bid_amount,
            "quality_score": quality_score.overall_score,
            "qs_multiplier": quality_score.ad_rank_multiplier,
            "ad_rank": ad_rank
        })

    # 4. Sort by Ad Rank (desc), then quality score (desc) for tie-breaking
    candidates.sort(key=lambda x: (-x["ad_rank"], -x["quality_score"]))

    # 5. Select top N candidates (based on placement slots)
    max_slots = placement.max_ads_per_page or 1
    winners = candidates[:max_slots]

    # 6. Enforce max_ad_per_seller limit
    winners = enforce_seller_limit(winners, placement.max_ad_per_seller)

    # 7. Calculate second-price CPC for each winner
    for i, winner in enumerate(winners):
        next_rank = candidates[i + 1]["ad_rank"] if i + 1 < len(candidates) else 0
        second_price_cpc = (next_rank / winner["qs_multiplier"]) + 0.01

        # Cap at max bid, floor at min CPC
        winner["actual_cpc"] = max(
            placement.min_cpc_bid or 0.01,
            min(second_price_cpc, winner["bid_amount"])
        )
        winner["auction_rank"] = i + 1

    return winners


def get_eligible_campaigns(placement, context):
    """Filter campaigns eligible for this auction."""
    now = frappe.utils.now()
    today = frappe.utils.today()

    candidates = frappe.db.sql("""
        SELECT
            ci.name, ci.parent as campaign, ci.listing, ci.bid_amount,
            ci.creative, ci.seller,
            ac.daily_budget, ac.total_budget, ac.daily_spend, ac.total_spend,
            ac.campaign_type, ac.seller as campaign_seller
        FROM `tabAd Campaign Item` ci
        INNER JOIN `tabAd Campaign` ac ON ac.name = ci.parent
        WHERE ac.status = 'Active'
          AND ac.start_date <= %(today)s
          AND (ac.end_date IS NULL OR ac.end_date >= %(today)s)
          AND ci.bid_amount >= %(min_bid)s
          AND ci.is_active = 1
          -- Daily budget check: still has budget left today
          AND (ac.daily_budget = 0 OR ac.daily_spend < ac.daily_budget)
          -- Total budget check: still has overall budget left
          AND (ac.total_budget = 0 OR ac.total_spend < ac.total_budget)
    """, {
        "today": today,
        "min_bid": placement.min_cpc_bid or 0,
    }, as_dict=True)

    # Additional filters
    filtered = []
    for c in candidates:
        # Check wallet balance
        wallet_balance = frappe.db.get_value(
            "Ad Wallet", {"seller": c.campaign_seller}, "available_balance"
        ) or 0
        if wallet_balance < c.bid_amount:
            continue

        # Keyword matching (for search placements)
        if context.get("search_query") and placement.placement_type == "Search Results":
            if not matches_keyword(c.parent, context["search_query"]):
                continue

        # Category targeting
        if context.get("category"):
            if not matches_category_targeting(c.parent, context["category"]):
                continue

        filtered.append(c)

    return filtered


def matches_keyword(campaign_name, search_query):
    """Check if campaign has keywords matching the search query."""
    keywords = frappe.get_all("Ad Keyword",
        filters={"parent": campaign_name, "is_negative": 0},
        fields=["keyword", "match_type"])

    query_lower = search_query.lower()

    for kw in keywords:
        kw_lower = kw.keyword.lower()

        if kw.match_type == "Exact" and query_lower == kw_lower:
            return True
        elif kw.match_type == "Phrase" and kw_lower in query_lower:
            return True
        elif kw.match_type == "Broad":
            # Check if any word in keyword appears in query
            kw_words = set(kw_lower.split())
            query_words = set(query_lower.split())
            if kw_words & query_words:  # Intersection
                return True

    return False


def get_active_banner_booking(placement_name):
    """Check for active CPT banner booking."""
    today = frappe.utils.today()
    now_hour = frappe.utils.now_datetime().hour

    booking = frappe.db.get_value("Banner Booking", {
        "placement": placement_name,
        "status": "Active",
        "start_date": ["<=", today],
        "end_date": [">=", today]
    }, ["name", "creative", "seller", "destination_url", "start_hour", "end_hour"],
    as_dict=True)

    if booking and booking.start_hour <= now_hour <= booking.end_hour:
        return booking
    return None
```

## 18. Fraud Detection System

The fraud detection pipeline processes every ad click event in real-time, evaluating it against configured `Ad Fraud Rule` records. Invalid events are flagged, billing is blocked, and alerts are generated.

### 18.1 Detection Pipeline

```
Click Event Received
        |
        v
  [Pre-validation]
  - Is event well-formed?
  - Does campaign exist and is active?
        |
        v
  [Rule Engine]
  - Evaluate each Ad Fraud Rule by priority
  - Stop on first Critical/High match
        |
        v
  [Fraud Decision]
  - Valid: proceed to billing
  - Invalid: flag event, block billing, create alert
        |
        v
  [Post-processing]
  - Update rule match statistics
  - Trigger notifications if configured
```

#### Rule 4: Coordinated Fraud Detection

```python
def check_coordinated_fraud(event, rule):
    """
    Detect coordinated click attacks from multiple IPs with similar patterns.
    Looks for unusual clustering of clicks on a single campaign from
    many unique IPs within a short time window.
    """
    window_start = frappe.utils.add_to_date(
        event.event_timestamp, minutes=-rule.correlation_window_minutes
    )

    # Count unique IPs clicking this campaign in the window
    result = frappe.db.sql("""
        SELECT COUNT(DISTINCT ip_address) as unique_ips,
               COUNT(*) as total_clicks
        FROM `tabAd Event Log`
        WHERE campaign = %(campaign)s
          AND event_type = 'Click'
          AND event_timestamp >= %(window_start)s
    """, {
        "campaign": event.campaign,
        "window_start": window_start
    }, as_dict=True)[0]

    if result.unique_ips >= rule.unique_ip_threshold:
        # Check click pattern similarity (timing intervals)
        click_times = frappe.db.sql("""
            SELECT event_timestamp
            FROM `tabAd Event Log`
            WHERE campaign = %(campaign)s
              AND event_type = 'Click'
              AND event_timestamp >= %(window_start)s
            ORDER BY event_timestamp
        """, {"campaign": event.campaign, "window_start": window_start},
        as_dict=True)

        if is_pattern_suspicious(click_times, rule.min_pattern_similarity):
            return {
                "is_fraud": True,
                "reason": f"COORDINATED: {result.unique_ips} unique IPs, "
                          f"{result.total_clicks} clicks in "
                          f"{rule.correlation_window_minutes}min",
                "rule": rule.name
            }

    return {"is_fraud": False}

def is_pattern_suspicious(click_times, similarity_threshold):
    """
    Check if click timestamps show suspiciously regular patterns.
    Coordinated attacks often have evenly-spaced clicks.
    """
    if len(click_times) < 3:
        return False

    intervals = []
    for i in range(1, len(click_times)):
        diff = (click_times[i].event_timestamp -
                click_times[i-1].event_timestamp).total_seconds()
        intervals.append(diff)

    if not intervals:
        return False

    avg_interval = sum(intervals) / len(intervals)
    if avg_interval == 0:
        return True  # All clicks at same time = definitely suspicious

    # Calculate coefficient of variation (CV)
    variance = sum((x - avg_interval) ** 2 for x in intervals) / len(intervals)
    std_dev = variance ** 0.5
    cv = (std_dev / avg_interval) * 100 if avg_interval > 0 else 100

    # Low CV = suspiciously regular pattern
    # Natural clicks have CV > 50%, bots often < 20%
    regularity = 100 - cv
    return regularity >= similarity_threshold
```

### 18.3 Fraud Processing Pipeline

```python
# ad_management/fraud_engine.py

def process_click_event(event_name):
    """
    Main fraud detection entry point. Called after every click event is created.
    Evaluates all active fraud rules in priority order.
    """
    event = frappe.get_doc("Ad Event Log", event_name)

    if event.event_type != "Click":
        return  # Only clicks are fraud-checked

    # Get all active fraud rules, ordered by priority
    rules = frappe.get_all("Ad Fraud Rule",
        filters={"status": "Active"},
        fields=["*"],

    fraud_detected = False

    for rule in rules:
        # Dispatch to appropriate checker
        checker = FRAUD_CHECKERS.get(rule.rule_type)
        if not checker:
            continue

        result = checker(event, rule)

        if result["is_fraud"]:
            fraud_detected = True

            # Apply actions
            if rule.action_invalidate_event:
                event.db_set("is_valid", 0)
                event.db_set("fraud_reason", result["reason"])

            if rule.action_block_billing:
                event.db_set("is_billable", 0)

            if rule.action_create_alert:
                create_fraud_alert(event, rule, result["reason"])

            if rule.action_refund and event.wallet_deducted:
                process_fraud_refund(event)

            if rule.action_suspend_ip:
                suspend_ip(event.ip_address, rule.ip_suspension_hours)

            # Update rule statistics
            frappe.db.set_value("Ad Fraud Rule", rule.name,
                                "total_matches", rule.total_matches + 1)
            frappe.db.set_value("Ad Fraud Rule", rule.name,
                                "last_match_at", frappe.utils.now())

            # Notify if configured
            if rule.notify_admin:
                send_admin_fraud_notification(event, rule, result)

            # Stop on first Critical/High match
            if rule.severity in ("Critical", "High"):
                break

    return fraud_detected


FRAUD_CHECKERS = {
    "IP Deduplication": check_ip_dedup,
    "Bot Detection": check_bot,
    "Click Flooding": check_click_flooding,
    "Coordinated Fraud": check_coordinated_fraud,
}


def create_fraud_alert(event, rule, reason):
    """Create an Ad Fraud Alert record for admin review."""
    alert = frappe.new_doc("Ad Fraud Alert")
    alert.alert_type = rule.rule_type
    alert.severity = rule.severity
    alert.campaign = event.campaign
    alert.seller = event.seller
    alert.event_log = event.name
    alert.fraud_rule = rule.name
    alert.description = reason
    alert.ip_address = event.ip_address
    alert.detected_at = frappe.utils.now()
    alert.status = "Open"
    alert.insert(ignore_permissions=True)


def process_fraud_refund(event):
    """Refund fraudulently charged click to seller's Ad Wallet."""
    if event.actual_cost and event.actual_cost > 0:
        wallet = frappe.get_doc("Ad Wallet", {"seller": event.seller})
        wallet.refund(
            event.actual_cost,
            "Ad Event Log", event.name,
            f"Fraud refund: {event.fraud_reason}"
        )
        # Reverse the campaign spend
        frappe.db.sql("""
            UPDATE `tabAd Campaign`
            SET daily_spend = GREATEST(0, daily_spend - %(cost)s),
                total_spend = GREATEST(0, total_spend - %(cost)s)
            WHERE name = %(campaign)s
        """, {"cost": event.actual_cost, "campaign": event.campaign})
```

### 19.1 Wallet Funding Flow

```python
@frappe.whitelist()
def transfer_to_ad_wallet(seller, amount):
    """Transfer funds from Seller Balance to Ad Wallet."""
    amount = flt(amount)
    if amount <= 0:
        frappe.throw(_("Amount must be positive"))

    # Check Seller Balance
    seller_balance = frappe.get_doc("Seller Balance", {"seller": seller})
    if seller_balance.available_balance < amount:
        frappe.throw(_("Insufficient Seller Balance. Available: {0}").format(
            seller_balance.available_balance
        ))

    # Debit from Seller Balance
    seller_balance.db_set("available_balance",
                          seller_balance.available_balance - amount)
    seller_balance.db_set("total_balance",
                          seller_balance.total_balance - amount)
    seller_balance.append("transaction_records", {
        "transaction_date": frappe.utils.now(),
        "transaction_type": "Debit",
        "amount": amount,
        "running_balance": seller_balance.available_balance,
        "reference_type": "Ad Wallet",
        "reference_name": seller,
        "description": "Transfer to Ad Wallet"
    })
    seller_balance.save(ignore_permissions=True)

    # Credit to Ad Wallet
    wallet = AdWallet.get_or_create(seller)
    wallet.add_funds(amount, "Seller Balance Transfer",
                     "Seller Balance", seller_balance.name,
                     "Transfer from Seller Balance")

    return {"success": True, "new_ad_balance": wallet.available_balance}
```

### 19.2 Per-Click Deduction Flow

```
User clicks ad
        |
        v
Ad Event Log created (event_type: Click)
        |
        v
Fraud detection pipeline runs
        |
        |-- If FRAUD: event.is_valid=0, event.is_billable=0, alert created
        |   (no charge)
        |
        |-- If VALID:
        v
Daily budget check (campaign.daily_spend + cost <= campaign.daily_budget?)
        |
        |-- If OVER BUDGET: event logged but not billed this click
        |
        |-- If WITHIN BUDGET:
        v
Calculate actual CPC (second-price auction result)
        |
        v
Deduct from Ad Wallet:
  wallet.deduct_balance(actual_cpc, "Ad Event Log", event.name)
        |
        v
Update campaign spend counters:
  campaign.daily_spend += actual_cpc
  campaign.total_spend += actual_cpc
        |
        v
Update event record:
  event.actual_cost = actual_cpc
  event.wallet_deducted = 1
  event.wallet_transaction_id = txn_ref
        |
        v
Check wallet balance thresholds:
  - Below minimum_balance_threshold? -> Send low-balance alert
  - Below auto_pause_threshold? -> Pause all active campaigns
```

#### Click Processing Code

```python
# ad_management/event_processor.py

def process_click(event_name):
    """Process a click event: fraud check, budget check, wallet deduction."""
    event = frappe.get_doc("Ad Event Log", event_name)

    # Step 1: Fraud detection
    is_fraud = process_click_event(event_name)
    if is_fraud:
        return {"billed": False, "reason": "fraud_detected"}

    # Step 2: Budget check
    campaign = frappe.get_doc("Ad Campaign", event.campaign)
    if not check_daily_budget(campaign.name, event.actual_cost or event.bid_amount):
        event.db_set("is_billable", 0)
        return {"billed": False, "reason": "daily_budget_exhausted"}

    # Step 3: Calculate actual cost (should already be set by auction engine)
    actual_cost = event.actual_cost or event.second_price or event.bid_amount

    # Step 4: Wallet deduction
    wallet = frappe.get_doc("Ad Wallet", {"seller": event.seller})
    if wallet.available_balance < actual_cost:
        # Insufficient balance - pause campaigns
        wallet._check_balance_thresholds(wallet.available_balance)
        event.db_set("is_billable", 0)
        return {"billed": False, "reason": "insufficient_balance"}

    wallet.deduct_balance(actual_cost, "Ad Event Log", event.name,
                          f"CPC charge: campaign {event.campaign}")

    # Step 5: Update event and campaign
    event.db_set("actual_cost", actual_cost)
    event.db_set("wallet_deducted", 1)
    record_spend(event.campaign, actual_cost)

    return {"billed": True, "cost": actual_cost}
```

### 19.3 Zero-Balance Auto-Pause

When a seller's Ad Wallet balance drops to or below `auto_pause_threshold`:

1. **All active campaigns are paused** with `pause_reason = "Insufficient ad wallet balance"`
2. **Seller receives a notification** via email and in-app realtime event
3. **Campaigns remain paused** until the seller adds funds
4. **When funds are added** above the threshold, campaigns are automatically resumed

```python
# Threshold check (called after every deduction)
def _check_balance_thresholds(self, current_balance):
    # Low balance warning (first threshold)
    if current_balance <= self.minimum_balance_threshold:
        if not self.low_balance_alert_sent:
            self.db_set("low_balance_alert_sent", 1)
            # Send email notification
            frappe.sendmail(
                recipients=[get_seller_email(self.seller)],
                subject=_("Low Ad Wallet Balance Warning"),
                message=_("Your ad wallet balance is {0}. "
                          "Add funds to keep your campaigns running.").format(
                    frappe.format_value(current_balance, {"fieldtype": "Currency"})
                )
            )

    # Auto-pause (critical threshold)
    if current_balance <= self.auto_pause_threshold:
        if not self.campaigns_auto_paused:
            self.db_set("campaigns_auto_paused", 1)
            self.pause_all_campaigns()
            # Critical notification
            frappe.sendmail(
                recipients=[get_seller_email(self.seller)],
                subject=_("Ad Campaigns Paused - Zero Balance"),
                message=_("All your ad campaigns have been paused because "
                          "your ad wallet balance is {0}. "
                          "Add funds to resume advertising.").format(
                    frappe.format_value(current_balance, {"fieldtype": "Currency"})
                )
            )
```

### 19.4 Monthly Invoice Reconciliation

At month end, the system generates an `Ad Invoice` that reconciles:

| Item | Source | Value |
|------|--------|-------|
| **CPC Charges** | Sum of `actual_cost` from Click events with `is_billable=1` | Positive |
| **CPM Charges** | Impression-based charges at CPM rate | Positive |
| **CPT Charges** | Banner Booking `final_cost` values | Positive |
| **Fraud Adjustments** | Sum of refunds for invalid clicks detected post-billing | Negative |
| **Manual Adjustments** | Admin credits/debits | +/- |
| **Tax** | Applicable KDV rate on subtotal | Positive |
| **Wallet Payments** | Already deducted from wallet during the month | Negative |
| **Balance Due** | Typically 0 TRY (pre-paid model) | Zero or Positive |

> **Note**: In the pre-paid wallet model, the `balance_due` should normally be 0 because charges are deducted in real-time. The invoice serves as a summary/receipt. Any positive `balance_due` indicates a system error or retroactive adjustment that needs resolution.

```python
@frappe.whitelist()
def get_seller_ad_dashboard(seller, period="30d"):
    """API endpoint for seller's ad dashboard data."""
    period_map = {"7d": 7, "30d": 30, "90d": 90}
    days = period_map.get(period, 30)
    start_date = frappe.utils.add_days(frappe.utils.today(), -days)

    # Wallet summary
    wallet = frappe.get_value("Ad Wallet", {"seller": seller},
        ["available_balance", "reserved_balance", "lifetime_spend",
         "total_clicks_purchased", "overall_roas"], as_dict=True) or {}

    # Campaign performance
    campaigns = frappe.db.sql("""
        SELECT
            ac.name, ac.campaign_name, ac.status, ac.daily_budget, ac.total_budget,
            ac.daily_spend, ac.total_spend,
            COUNT(CASE WHEN ael.event_type='Impression' THEN 1 END) as impressions,
            COUNT(CASE WHEN ael.event_type='Click' AND ael.is_valid=1 THEN 1 END) as clicks,
            COUNT(CASE WHEN ael.event_type='Conversion' THEN 1 END) as conversions,
            SUM(CASE WHEN ael.is_billable=1 THEN ael.actual_cost ELSE 0 END) as spend,
            SUM(CASE WHEN ael.event_type='Conversion' THEN ael.conversion_value ELSE 0 END) as conv_value
        FROM `tabAd Campaign` ac
        LEFT JOIN `tabAd Event Log` ael ON ael.campaign = ac.name
            AND ael.event_timestamp >= %(start_date)s
        WHERE ac.seller = %(seller)s
        GROUP BY ac.name
        ORDER BY spend DESC
    """, {"seller": seller, "start_date": start_date}, as_dict=True)

    # Calculate derived metrics
    for c in campaigns:
        c.ctr = (c.clicks / c.impressions * 100) if c.impressions else 0
        c.avg_cpc = (c.spend / c.clicks) if c.clicks else 0
        c.acos = (c.spend / c.conv_value * 100) if c.conv_value else 0
        c.roas = (c.conv_value / c.spend) if c.spend else 0

    # Daily spend trend

        SELECT DATE(event_timestamp) as date,
               SUM(actual_cost) as spend,
               COUNT(CASE WHEN event_type='Click' AND is_valid=1 THEN 1 END) as clicks,

        ORDER BY spend DESC
        LIMIT 20
    """, {"seller": seller, "start_date": start_date}, as_dict=True)

    for kw in top_keywords:
        kw.ctr = (kw.clicks / kw.impressions * 100) if kw.impressions else 0
        kw.avg_cpc = (kw.spend / kw.clicks) if kw.clicks else 0
```

```python
    if not reason:
        frappe.throw(_("Rejection reason is required"))

    campaign = frappe.get_doc("Ad Campaign", campaign_name)
    campaign.status = "Rejected"
    campaign.rejection_reason = reason
    campaign.save(ignore_permissions=True)

    frappe.sendmail(
        recipients=[get_seller_email(campaign.seller)],
        subject=_("Ad Campaign Rejected: {0}").format(campaign.campaign_name),
        message=_("Your ad campaign '{0}' has been rejected.\nReason: {1}").format(
            campaign.campaign_name, reason
        )
    )
```

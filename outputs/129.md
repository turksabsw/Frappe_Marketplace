# Task 129 - Automated Customer Grading (A-F) System Plan

## 1. Overview

This document defines the **Automated Customer Grading System** for the TradeHub marketplace platform. The system assigns letter grades (A through F) to buyers based on a weighted composite score derived from their marketplace behavior. It provides:

- **Customer Grade DocType**: A platform-level grade record per buyer, storing the computed score (0-100), letter grade (A-F), calculation date, previous grade, and trend direction.
- **Seller Customer Grade DocType**: A seller-specific view of customer grades with anonymous customer identification and configurable seller-specific criteria weighting.
- **Grading Criteria Child Table**: A child DocType (`Grading Criterion`) that defines individual criteria with customizable weights.
- **Weighted scoring system**: Seven criteria with defined default percentages totaling exactly 100%.
- **Grade thresholds**: Six inclusive grade bands mapping scores to letter grades (A through F).
- **Grade-based actions**: Automatic behaviors triggered by grade changes (VIP perks, incentives, restrictions).
- **Dashboard components**: Grade distribution chart, Pareto analysis, and trend reporting.

### Key Design Decisions

1. **Platform-level grades**: The `Customer Grade` DocType stores a single grade per buyer, calculated by the platform. This is the authoritative grade visible to admins and used for automated actions.
2. **Seller-specific view**: The `Seller Customer Grade` DocType allows sellers to see anonymized customer grades and optionally apply their own criteria weighting. This does **not** override the platform grade.
3. **Inclusive boundaries**: A score of exactly 85 is grade A (not B). All boundaries use `>=` for the lower bound and `<=` for the upper bound within each grade band.
4. **Coexistence with Buyer Level**: Grades (A-F) measure behavioral quality; Buyer Levels (New-Premium) measure spend/loyalty. A buyer can be grade A (excellent behavior) at New level (just started), or grade D (poor behavior) at Premium level (high spender with issues).

```python
import hashlib

def generate_anonymous_customer_id(buyer_name, seller_name):
    """
    Generate a deterministic but anonymized customer ID.
    Same buyer+seller pair always produces the same ID,
    but the seller cannot reverse-engineer the buyer identity.
    """
    salt = frappe.get_conf().get("customer_grade_salt", "tradehub-grade-salt")
    raw = f"{buyer_name}:{seller_name}:{salt}"
    hash_hex = hashlib.sha256(raw.encode()).hexdigest()[:12]
    return f"CUST-{hash_hex.upper()}"
```

## 4. Business Logic & Formulas

### 4.1 Weighted Scoring Criteria

The customer grade is calculated from **7 behavioral criteria**, each with a defined weight. Weights sum to exactly **100%**.

| # | Criterion | Field Source | Weight | Direction | Description |
|---|-----------|-------------|--------|-----------|-------------|
| 1 | **Total Spend** | `Buyer Profile.total_spent` | **25%** | Higher is better | Cumulative purchase amount on the platform |
| 2 | **Order Frequency** | Calculated from `Sales Order` | **20%** | Higher is better | Average orders per month over the evaluation period |
| 3 | **On-Time Payment** | `Buyer Profile.payment_on_time_rate` | **15%** | Higher is better | Percentage of payments made on or before due date |
| 4 | **Return/Cancel Rate** | Combined `return_rate` + `cancellation_rate` | **15%** | Lower is better | Combined percentage of returned and cancelled orders |
| 5 | **Account Age** | `Buyer Profile.joined_at` | **10%** | Higher is better | Days since account creation |
| 6 | **Dispute Rate** | `Buyer Profile.dispute_rate` | **10%** | Lower is better | Percentage of orders with disputes initiated |
| 7 | **Feedback Rate** | `Buyer Profile.feedback_rate` | **5%** | Higher is better | Percentage of completed orders where buyer left feedback |

**Total Weight**: 25 + 20 + 15 + 15 + 10 + 10 + 5 = **100%**

### 4.2 Metric Normalization

Each raw metric is normalized to a 0-100 scale before weighting. The normalization approach depends on the metric direction.

#### Higher-is-Better Metrics (Total Spend, Order Frequency, On-Time Payment, Account Age, Feedback Rate)

```python
def normalize_higher_is_better(value, target_good, target_poor):
    """
    Normalize a metric where higher values are better.
    Returns a score from 0 to 100.

    - value >= target_good  => 100
    - value <= target_poor  => 0
    - In between            => linear interpolation
    """
    if value >= target_good:
        return 100.0
    if value <= target_poor:
        return 0.0
    return ((value - target_poor) / (target_good - target_poor)) * 100.0
```

#### Lower-is-Better Metrics (Return/Cancel Rate, Dispute Rate)

```python
def normalize_lower_is_better(value, target_good, target_poor):
    """
    Normalize a metric where lower values are better.
    Returns a score from 0 to 100.

    - value <= target_good  => 100
    - value >= target_poor  => 0
    - In between            => linear interpolation (inverted)
    """
    if value <= target_good:
        return 100.0
    if value >= target_poor:
        return 0.0
    return ((target_poor - value) / (target_poor - target_good)) * 100.0
```

### 4.3 Normalization Targets

| Criterion | Target (Good) | Target (Poor) | Unit | Notes |
|-----------|--------------|---------------|------|-------|
| Total Spend | 10,000 | 0 | Currency (TRY) | Logarithmic scaling applied for high-value buyers |
| Order Frequency | 4.0 | 0 | Orders/month | Average over last 6 months |
| On-Time Payment | 100 | 50 | Percent | From `payment_on_time_rate` |
| Return/Cancel Rate | 0 | 20 | Percent | Combined `return_rate` + `cancellation_rate` |
| Account Age | 730 | 0 | Days | 2 years = full score; capped at 100 |
| Dispute Rate | 0 | 10 | Percent | From `dispute_rate` |
| Feedback Rate | 100 | 0 | Percent | From `feedback_rate` |

**Note on Total Spend**: Because total spend has a very wide range (0 to millions), logarithmic scaling is applied before linear normalization:

```python
import math

def normalize_total_spend(total_spent, target_good=10000, target_poor=0):
    """
    Logarithmic normalization for total spend.
    Prevents ultra-high spenders from distorting the scale.
    """
    if total_spent <= 0:
        return 0.0
    log_value = math.log10(total_spent + 1)
    log_good = math.log10(target_good + 1)
    log_poor = math.log10(target_poor + 1)
    if log_value >= log_good:
        return 100.0
    if log_value <= log_poor:
        return 0.0
    return ((log_value - log_poor) / (log_good - log_poor)) * 100.0
```

```python
        ),
        "order_frequency": normalize_higher_is_better(
            raw_metrics["order_frequency"], target_good=4.0, target_poor=0
        ),
        "payment_on_time": normalize_higher_is_better(
            raw_metrics["payment_on_time_rate"], target_good=100, target_poor=50
        ),
        "return_cancel": normalize_lower_is_better(
            raw_metrics["return_cancel_rate"], target_good=0, target_poor=20
        ),
        "account_age": normalize_higher_is_better(
            raw_metrics["account_age_days"], target_good=730, target_poor=0
        ),
        "dispute_rate": normalize_lower_is_better(
            raw_metrics["dispute_rate"], target_good=0, target_poor=10
        ),
        "feedback_rate": normalize_higher_is_better(
            raw_metrics["feedback_rate"], target_good=100, target_poor=0
        ),
    }

    # Step 3: Apply weights (default weights)
    weights = {
        "total_spend": 25,
        "order_frequency": 20,
        "payment_on_time": 15,
        "return_cancel": 15,
        "account_age": 10,
        "dispute_rate": 10,
        "feedback_rate": 5,
    }

    # Step 4: Calculate weighted score
    score = sum(
        normalized[key] * (weights[key] / 100.0)
        for key in weights
    )

    # Step 5: Clamp to 0-100
    score = max(0.0, min(100.0, round(score, 2)))

    # Step 6: Map score to grade
    grade = score_to_grade(score)

    return score, grade, normalized
```

### 4.5 Order Frequency Calculation

```python
def calculate_order_frequency(buyer_name, months=6):
    """
    Calculate average orders per month over the last N months.
    Uses Sales Order with status in ('Completed', 'Delivered').
    """
    from_date = frappe.utils.add_months(frappe.utils.today(), -months)

    order_count = frappe.db.count("Sales Order", filters={
        "buyer": buyer_name,
        "status": ("in", ["Completed", "Delivered"]),
        "transaction_date": (">=", from_date),
    })

    return round(order_count / months, 2) if months > 0 else 0.0
```

### 4.6 Grade Thresholds

Scores map to letter grades using **inclusive boundaries**. The lower bound of each grade range uses `>=`.

| Grade | Score Range | Description | Color Code |
|-------|-----------|-------------|------------|
| **A** | **85 - 100** | Excellent customer. Top-tier behavior across all criteria. | `#22c55e` (Green) |
| **B** | **70 - 84** | Good customer. Above average with minor areas for improvement. | `#3b82f6` (Blue) |
| **C** | **55 - 69** | Average customer. Meets basic expectations. | `#eab308` (Yellow) |
| **D** | **40 - 54** | Below average. Multiple areas need improvement. | `#f97316` (Orange) |
| **E** | **25 - 39** | Poor customer. Significant behavioral concerns. | `#ef4444` (Red) |
| **F** | **0 - 24** | Critical. Severe issues; account may face restrictions. | `#991b1b` (Dark Red) |

**Boundary Rule**: The lower bound is **inclusive** (`>=`) and the upper bound is **inclusive** (`<=`).

- A score of exactly **85.00** = Grade **A** (not B)
- A score of exactly **84.99** = Grade **B**
- A score of exactly **70.00** = Grade **B** (not C)
- A score of exactly **0.00** = Grade **F**
- A score of exactly **100.00** = Grade **A**

```python
def score_to_grade(score):
    """
    Map a numeric score (0-100) to a letter grade (A-F).
    Boundaries are inclusive on the lower end.
    """
    if score >= 85:
        return "A"
    elif score >= 70:
        return "B"
    elif score >= 55:
        return "C"
    elif score >= 40:
        return "D"
    elif score >= 25:
        return "E"
    else:
        return "F"
```

### 4.7 Trend Calculation

```python
def calculate_trend(current_score, previous_score, threshold=2.0):
    """
    Determine grade trend based on score change.
    A change of less than `threshold` points is considered Stable.
    """
    if previous_score is None or previous_score == 0:
        return "Stable"

    change = current_score - previous_score

    if change >= threshold:
        return "Improving"
    elif change <= -threshold:
        return "Declining"
    else:
        return "Stable"
```

### 4.8 Edge Cases

#### New Users (No Orders)

Buyers with zero completed orders receive a **default score of 50** (Grade C) with the following rationale:
- Total Spend: 0 (normalized: 0)
- Order Frequency: 0 (normalized: 0)
- On-Time Payment: 100 (no late payments = 100% on-time, normalized: 100)
- Return/Cancel Rate: 0 (no returns = 0%, normalized: 100)
- Account Age: varies (normalized based on actual age)
- Dispute Rate: 0 (no disputes = 0%, normalized: 100)
- Feedback Rate: 0 (no feedback, normalized: 0)

Weighted default calculation: `(0*0.25) + (0*0.20) + (100*0.15) + (100*0.15) + (age_score*0.10) + (100*0.10) + (0*0.05)` = ~40 + age_bonus = approximately Grade D/C for new users.

**Minimum order threshold**: Grades are only finalized after a buyer has **at least 3 completed orders**. Before that threshold, the grade shows as "Provisional" in the `calculation_notes` field.

#### Score of Exactly 85

A score of **85.00** is Grade **A**. The `score_to_grade()` function uses `>=` for boundary comparisons, so 85 is captured by the `score >= 85` condition before reaching the `score >= 70` check.

#### Division by Zero

All division operations use safe division:

```python
def safe_divide(numerator, denominator, default=0):
    """Safe division that returns default on zero denominator."""
    if denominator == 0:
        return default
    return numerator / denominator
```

#### Return/Cancel Rate Combination

The return/cancel rate is the **sum** of `return_rate` and `cancellation_rate`, not the average. This means a buyer with 10% returns and 5% cancellations has a combined rate of 15%.

If the combined rate exceeds 20%, the normalized score is 0 (worst possible for this criterion).

## 5. Scheduled Jobs

### 5.1 Weekly Grade Calculation Job

| Property | Value |
|----------|-------|
| **Job Name** | `tradehub_core.tasks.calculate_customer_grades` |
| **Frequency** | Weekly (cron: `0 5 * * 0` - Sunday at 05:00 AM) |
| **What It Calculates** | Composite score and letter grade for all active buyers |
| **Data Sources** | `Buyer Profile`, `Sales Order`, `Payment Entry`, `Return Request`, `Dispute`, `Review` |
| **Data Targets** | `Customer Grade` (creates new records per period) |
| **Estimated Runtime** | ~2-5 minutes for 10,000 buyers; ~15-30 minutes for 100,000 buyers |
| **Error Handling** | Per-buyer try/except; failed buyers logged and skipped; summary email to System Manager |
| **Locking** | Uses `frappe.cache().set_value("customer_grade_lock", 1, expires_in_sec=3600)` to prevent concurrent runs |

```python
def calculate_customer_grades():
    """
    Weekly scheduled job: Calculate grades for all active buyers.
    Creates one Customer Grade record per buyer per period.
    """
    lock_key = "customer_grade_lock"
    if frappe.cache().get_value(lock_key):
        frappe.log_error("Customer grade calculation already in progress", "Grade Lock")
        return

    frappe.cache().set_value(lock_key, 1, expires_in_sec=3600)

    try:
        period = get_current_grade_period()  # e.g., "2026-W09"
        buyers = frappe.get_all("Buyer Profile", filters={"status": "Active"}, pluck="name")

        success_count = 0
        error_count = 0
        errors = []

        for buyer_name in buyers:
            try:
                score, grade, components = calculate_customer_grade_score(buyer_name)

                # Get previous grade
                prev = get_previous_grade(buyer_name, period)

                # Create new Customer Grade record
                grade_doc = frappe.new_doc("Customer Grade")
                grade_doc.buyer = buyer_name
                grade_doc.grade = grade
                grade_doc.score = score
                grade_doc.status = "Finalized"
                grade_doc.grade_period = period
                grade_doc.calculation_date = frappe.utils.today()

                # Score components
                grade_doc.total_spend_score = components["total_spend"]
                grade_doc.order_frequency_score = components["order_frequency"]
                grade_doc.payment_score = components["payment_on_time"]
                grade_doc.return_cancel_score = components["return_cancel"]
                grade_doc.account_age_score = components["account_age"]
                grade_doc.dispute_score = components["dispute_rate"]
                grade_doc.feedback_score = components["feedback_rate"]

                # History
                if prev:
                    grade_doc.previous_grade = prev.grade
                    grade_doc.previous_score = prev.score
                    grade_doc.score_change = round(score - prev.score, 2)
                    grade_doc.trend = calculate_trend(score, prev.score)
                    grade_doc.grade_changed = 1 if grade != prev.grade else 0
                    grade_doc.consecutive_same_grade = (
                        prev.consecutive_same_grade + 1 if grade == prev.grade else 1
                    )

                # Store raw metrics for audit
                profile = frappe.get_doc("Buyer Profile", buyer_name)
                grade_doc.raw_total_spent = profile.total_spent or 0
                grade_doc.raw_total_orders = profile.total_orders or 0
                grade_doc.raw_order_frequency = calculate_order_frequency(buyer_name)
                grade_doc.raw_payment_on_time_rate = profile.payment_on_time_rate or 0
                grade_doc.raw_return_rate = profile.return_rate or 0
                grade_doc.raw_cancellation_rate = profile.cancellation_rate or 0
                grade_doc.raw_dispute_rate = profile.dispute_rate or 0
                grade_doc.raw_feedback_rate = profile.feedback_rate or 0
                grade_doc.raw_account_age_days = days_since(profile.joined_at)

                grade_doc.insert(ignore_permissions=True)

                # Trigger grade-based actions if grade changed
                if prev and grade != prev.grade:
                    trigger_grade_change_actions(buyer_name, prev.grade, grade, score)

                success_count += 1

                # Commit every 100 records to avoid long transactions
                if success_count % 100 == 0:
                    frappe.db.commit()

            except Exception as e:
                error_count += 1
                errors.append(f"{buyer_name}: {str(e)}")
                frappe.log_error(
                    f"Grade calculation failed for {buyer_name}: {str(e)}",
                    "Customer Grade Error"
                )

        frappe.db.commit()

        # Send summary
        if error_count > 0:
            frappe.sendmail(
                recipients=get_system_managers(),
                subject=f"Customer Grade Calculation: {success_count} OK, {error_count} errors",
                message=f"Period: {period}\n\nErrors:\n" + "\n".join(errors[:50])
            )

    finally:
        frappe.cache().delete_value(lock_key)
```

## Appendix C: Grade Calculation Example

**Example Buyer**: Buyer-00042 (Ali, Premium level, 18 months old account)

| Metric | Raw Value | Normalization | Normalized Score | Weight | Weighted |
|--------|-----------|---------------|-----------------|--------|----------|
| Total Spent | 15,000 TRY | log10(15001) / log10(10001) * 100 | 100.00 | 25% | 25.00 |
| Order Frequency | 3.2 orders/month | (3.2 - 0) / (4.0 - 0) * 100 | 80.00 | 20% | 16.00 |
| Payment On-Time | 92% | (92 - 50) / (100 - 50) * 100 | 84.00 | 15% | 12.60 |
| Return/Cancel Rate | 8% (5% + 3%) | (20 - 8) / (20 - 0) * 100 | 60.00 | 15% | 9.00 |
| Account Age | 548 days | (548 - 0) / (730 - 0) * 100 | 75.07 | 10% | 7.51 |
| Dispute Rate | 2% | (10 - 2) / (10 - 0) * 100 | 80.00 | 10% | 8.00 |
| Feedback Rate | 70% | (70 - 0) / (100 - 0) * 100 | 70.00 | 5% | 3.50 |

**Composite Score**: 25.00 + 16.00 + 12.60 + 9.00 + 7.51 + 8.00 + 3.50 = **81.61**

**Grade**: 81.61 >= 70 and < 85 => **B**

**Previous**: Score 78.3 (Grade B) => Trend: Improving (+3.31), Grade Changed: No

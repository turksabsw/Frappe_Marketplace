# Task 122: UOM Filtering System - Implementation Plan

## 1. Overview

This document defines the complete implementation plan for the UOM (Unit of Measure) Filtering System across the TradeHub Marketplace platform. The goal is to replace hardcoded `Select` UOM fields with dynamic `Link` fields pointing to ERPNext's standard `UOM` DocType, and to implement `set_query` filters that scope UOM dropdowns by category (Weight, Length, Volume, Area, Quantity, Other).

ERPNext already provides the infrastructure:
- **UOM** DocType (in Frappe core) - master list of all units
- **UOM Category** DocType (`erpnext.stock.doctype.uom_category`) - category grouping (e.g., "Mass", "Length")
- **UOM Conversion Factor** DocType (`erpnext.setup.doctype.uom_conversion_factor`) - conversion rules with `category`, `from_uom`, `to_uom`, and `value` fields

The current state has **critical bugs and inconsistencies** that this plan resolves.

## 2. Current State Analysis - Bugs & Issues

### Bug 1: Marketplace Shipment JS references wrong field names (CRITICAL)

The `marketplace_shipment.js` `setup_uom_filters()` function calls:

```javascript
frm.set_query('weight_uom', ...);    // field does NOT exist - actual field is "weight_unit" (Select)
frm.set_query('dimension_uom', ...); // field does NOT exist - actual field is "dimension_unit" (Select)
frm.set_query('volume_uom', ...);    // field does NOT exist on this DocType
```

**Impact:** All three `set_query` calls silently fail. No UOM filtering is applied.

### Bug 2: Shipment `volume_uom` is `Data` not `Link`

In `shipment.json`, `volume_uom` is defined as `fieldtype: "Data"` with default `"cbm"`, yet `shipment.js` calls `frm.set_query('volume_uom', ...)` which only works on Link fields.

**Impact:** The `set_query` call silently fails. Users see a free-text input instead of a filtered dropdown.

## 6. `set_query` Filter Implementation

### 6.1 Category-Based Filtering Strategy

Instead of hardcoding UOM names in JavaScript arrays (current approach), use the `UOM Conversion Factor` DocType's `category` field to filter dynamically:

```javascript
// RECOMMENDED: Category-based filter (dynamic, future-proof)
frm.set_query('weight_uom', function() {
    return {
        query: 'tradehub_logistics.api.get_uoms_by_category',
        filters: {
            'category': 'Mass'
        }
    };
});
```

**Server-side whitelisted method:**

```python
# tradehub_logistics/api.py
import frappe

@frappe.whitelist()
def get_uoms_by_category(doctype, txt, searchfield, start, page_len, filters):
    """Return UOMs filtered by their UOM Category via UOM Conversion Factor."""
    category = filters.get('category')
    if not category:
        return frappe.get_all('UOM', filters={'name': ['like', f'%{txt}%']},
                              as_list=True, limit_start=start, limit_page_len=page_len)

    return frappe.db.sql("""
        SELECT DISTINCT ucf.from_uom as name
        FROM `tabUOM Conversion Factor` ucf
        WHERE ucf.category = %(category)s
          AND ucf.from_uom LIKE %(txt)s
        UNION
        SELECT DISTINCT ucf.to_uom as name
        FROM `tabUOM Conversion Factor` ucf
        WHERE ucf.category = %(category)s
          AND ucf.to_uom LIKE %(txt)s
        ORDER BY name
        LIMIT %(start)s, %(page_len)s
    """, {
        'category': category,
        'txt': f'%{txt}%',
        'start': start,
        'page_len': page_len
    })
```

### 7.2 Conversion Utility Function

```python
# tradehub_logistics/utils/uom.py

import frappe

def convert_uom(value, from_uom, to_uom):
    """
    Convert a value from one UOM to another using UOM Conversion Factor.

    Args:
        value (float): The numeric value to convert
        from_uom (str): Source UOM name (e.g., 'Pound')
        to_uom (str): Target UOM name (e.g., 'Kg')

    Returns:
        float: Converted value

    Raises:
        frappe.ValidationError: If conversion factor not found or cross-category
    """
    if from_uom == to_uom:
        return value

    # Check same category
    from_category = get_uom_category(from_uom)
    to_category = get_uom_category(to_uom)

    if from_category and to_category and from_category != to_category:
        frappe.throw(
            _("Cannot convert between {0} ({1}) and {2} ({3}). "
              "Cross-category conversion is not supported.").format(
                from_uom, from_category, to_uom, to_category
            ),
            title=_("UOM Conversion Error")
        )

    # Try direct conversion
    factor = frappe.db.get_value('UOM Conversion Factor', {
        'from_uom': from_uom,
        'to_uom': to_uom
    }, 'value')

    if factor:
        return value * factor

    # Try reverse conversion
    factor = frappe.db.get_value('UOM Conversion Factor', {
        'from_uom': to_uom,
        'to_uom': from_uom
    }, 'value')

    if factor:
        return value / factor

    frappe.throw(
        _("No conversion factor found between {0} and {1}. "
          "Please add it in UOM Conversion Factor.").format(from_uom, to_uom),
        title=_("UOM Conversion Error")
    )


def get_uom_category(uom_name):
    """Get the UOM Category for a given UOM via UOM Conversion Factor."""
    category = frappe.db.get_value('UOM Conversion Factor', {
        'from_uom': uom_name
    }, 'category')

    if not category:
        category = frappe.db.get_value('UOM Conversion Factor', {
            'to_uom': uom_name
        }, 'category')

    return category
```

### 7.3 Usage in Shipment Context

**Dimensional weight calculation (Shipment):**

```python
def calculate_dimensional_weight(doc):
    """Calculate dimensional weight from L x W x H dimensions."""
    if not (doc.length and doc.width and doc.height):
        return 0

    # Convert dimensions to cm if not already
    length_cm = convert_uom(doc.length, doc.dimension_uom, 'Cm')
    width_cm = convert_uom(doc.width, doc.dimension_uom, 'Cm')
    height_cm = convert_uom(doc.height, doc.dimension_uom, 'Cm')

    # Standard divisor: 5000 for international, 6000 for domestic
    divisor = 5000
    dimensional_weight_kg = (length_cm * width_cm * height_cm) / divisor

    # Convert to the document's weight UOM
    return convert_uom(dimensional_weight_kg, 'Kg', doc.weight_uom)
```

**Weight comparison for shipping cost (Shipping Rule):**

```python
def get_billable_weight(shipment):
    """Return the greater of actual weight and dimensional weight."""
    actual_kg = convert_uom(shipment.total_weight, shipment.weight_uom, 'Kg')
    dim_weight_kg = shipment.dimensional_weight  # Already in weight_uom

    if dim_weight_kg:
        dim_kg = convert_uom(dim_weight_kg, shipment.weight_uom, 'Kg')
        return max(actual_kg, dim_kg)

    return actual_kg
```

## 8. Edge Cases

### 8.1 Custom UOMs Without Category

**Scenario:** A user creates a custom UOM (e.g., "Carton") that is not assigned to any UOM Category or UOM Conversion Factor record.

**Handling:**
- Custom UOMs without a category are treated as "Other" category
- They will NOT appear in filtered dropdowns (weight, length, volume)
- They WILL appear in unfiltered `stock_uom`/`uom` dropdowns (Quantity/Any)
- Admin notification: When a UOM is created without a category assignment, show a warning:
  ```python
  def after_insert(self):
      if not frappe.db.exists('UOM Conversion Factor', {'from_uom': self.name}):
          frappe.msgprint(
              _("UOM '{0}' has no conversion factors defined. "
                "It will only appear in unfiltered UOM fields.").format(self.name),
              alert=True
          )
  ```

### 8.2 Cross-Category Conversion Errors

**Scenario:** Code or user attempts to convert between incompatible categories (e.g., Kg to Centimeter).

**Handling:**
- The `convert_uom()` function checks category membership before attempting conversion
- Raises `frappe.ValidationError` with a clear message:
  `"Cannot convert between Kg (Mass) and Cm (Length). Cross-category conversion is not supported."`
- UI-level prevention: since each field has category-specific `set_query`, users cannot select incompatible UOMs in the same context

### 8.3 Missing Conversion Factor

**Scenario:** A valid within-category conversion is requested but no `UOM Conversion Factor` record exists (e.g., converting between two custom weight UOMs).

**Handling:**
- Try direct lookup: `from_uom -> to_uom`
- Try reverse lookup: `to_uom -> from_uom` with inverse
- Try transitive conversion via base unit (e.g., A->Kg->B):
  ```python
  # Future enhancement: transitive conversion
  # A -> base -> B where base is the category's reference UOM
  ```
- If none found: raise `frappe.ValidationError` with suggestion to add the conversion factor

### 8.4 Marketplace Shipment Data Migration - Partial Data

**Scenario:** Some `Marketplace Shipment` records have `weight_unit = "kg"` but others have empty values.

**Handling:**
- Empty `weight_unit` values are migrated to the default `Kg`
- Empty `dimension_unit` values are migrated to the default `Cm`
- Log all migration changes for audit:
  ```python
  frappe.log_error(
      f"Migrated {count} Marketplace Shipment records: weight_unit -> weight_uom",
      "UOM Migration Log"
  )
  ```

### 8.5 Duplicate UOM Names (Case Sensitivity)

**Scenario:** Both `kg` and `Kg` exist as separate UOM records.

**Handling:**
- Frappe UOM names are case-sensitive (they are the primary key)
- Before migration, merge duplicates:
  ```python
  # Identify duplicates
  all_uoms = frappe.get_all('UOM', pluck='name')
  lower_map = {}
  for uom in all_uoms:
      key = uom.lower()
      if key in lower_map:
          # Duplicate found: merge lower_map[key] and uom
          canonical = lower_map[key]  # Keep first occurrence
          # Update all references to use canonical
      else:
          lower_map[key] = uom
  ```
- Canonical names should use Title Case for full names (`Kilogram`) and standard abbreviations for short names (`Kg`, `Cm`)

### 8.6 Read-Only / fetch_from UOM Fields

**Scenario:** Fields like `Product Variant.weight_uom` use `fetch_from: "sku_product.weight_uom"` and are `read_only: 1`.

**Handling:**
- Do NOT add `set_query` for read-only fetch_from fields (filter is applied at the source field)
- Ensure the source field (`SKU Product.weight_uom`) has the correct `set_query`
- The fetch_from chain propagates the validated value automatically

# Task 146 — Dinamik Sozlesme Sistemi (Dynamic Contract System)

## 1. Genel Bakis (Overview)

Bu belge, B2B pazar yeri platformunda **dinamik sozlesme olusturma sistemi**nin teknik tasarimini icerir. Mevcut `tr_contract_center` uygulamasindaki `Contract Template` ve `Contract Instance` yapilari temel alinarak, satici ozelliklerine gore **kosul bazli** sozlesme iceriginin otomatik olarak derlenmesini saglayan bir kural motoru (rule engine) planlanmaktadir.

**Problem:** Mevcut yaklasimda her satici tipi, abonelik paketi veya kategori icin ayri sozlesme sablonlari olusturulmak zorundadir. Bu durum, sablon sayisinin kontrol edilemez bicimde artmasina, guncelleme hatarina ve tutarsizliklara yol acmaktadir.

**Hedef:** Tek bir temel sozlesme sablonu (`base_text`) uzerine kosul bazli kurallar tanimlayarak, her saticinin durumuna uygun kisisellestirilmis sozlesme icerigi **otomatik olarak** derlenmesini saglamak.

**Kapsam:**
- `Contract Template` DocType'inin dinamik kural destegi ile genisletilmesi
- `Contract Condition Rule` child table tanimlanmasi
- Kural degerlendirme motoru (rule evaluation engine) algoritmasi
- Admin yonetim arayuzu is akisi
- Mevcut `tr_contract_center` entegrasyonu

**Mevcut Altyapi Referanslari:**
- `tr_contract_center` uygulamasi: Contract Template, Contract Instance, Contract Revision, Contract Rule, ESign Provider, ESign Transaction DocType'lari
- `events.py`: `contract_instance_on_update` — icerik degisikliginde revizyon kaydı olusturma
- `hooks.py`: `doc_events`, `scheduler_events` (daily: check_expiring_contracts, check_pending_signatures; hourly: poll_esign_status)
- `api.py`: `create_instance` — sablon bazli sozlesme ornegi olusturma, versiyon snapshot'i alma

## 5. Kural Degerlendirme Motoru (Rule Evaluation Engine)

### 5.1 Ana Algoritma — Adim Adim Pseudocode

```python
# Pseudocode — Dinamik Sozlesme Derleme Motoru

def compile_dynamic_contract(template_name, seller_profile_name):
    """
    Bir sozlesme sablonunu belirli bir satici icin derler.

    Args:
        template_name: Contract Template adi
        seller_profile_name: Seller Profile adi

    Returns:
        dict: {
            "compiled_content": str,   # Derlenmis tam metin
            "rules_applied": list,     # Uygulanan kurallar
            "rules_skipped": list,     # Atlanan kurallar (nedenle birlikte)
            "content_hash": str,       # SHA256 hash
            "compilation_log": str     # Detayli log
        }
    """

    # ===== ADIM 1: Sablon ve Satici Bilgilerini Al =====
    template = frappe.get_doc("Contract Template", template_name)

    if not template.is_active:
        frappe.throw(_("Contract template is not active: {0}").format(template_name))

    if not template.dynamic_rules_enabled:
        # Dinamik kurallar aktif degil — statik icerigi dondur
        return {
            "compiled_content": template.content or template.base_text,
            "rules_applied": [],
            "rules_skipped": [],
            "content_hash": template.content_hash,
            "compilation_log": "Dynamic rules disabled. Using static content."
        }

    seller = frappe.get_doc("Seller Profile", seller_profile_name)

    # ===== ADIM 2: Satici Ozelliklerini Topla =====
    seller_attributes = collect_seller_attributes(seller)
    """
    seller_attributes = {
        "package": "Premium",
        "category": ["Elektronik", "Bilisim"],
        "revenue_range": 250000,
        "seller_type": "company",
        "location": "Istanbul",
        "registration_age": 180,  # gun
        "custom": {"vip_status": True, "tax_exempt": False}
    }
    """

    # ===== ADIM 3: Aktif Kurallari Al ve Sirala =====
    rules = get_sorted_active_rules(template)
    """
    Siralama: template.rule_evaluation_order'a gore
    - priority_asc: dusuk priority degeri oncelikli
    - priority_desc: yuksek priority degeri oncelikli
    - condition_type_grouped: condition_type'a gore gruplanmis, iceride priority_asc
    """

    # ===== ADIM 4: Her Kurali Degerlendir =====
    compiled_content = template.base_text
    clauses_to_add = []       # (clause_order, clause_text, rule_name)
    markers_to_remove = []    # (clause_marker, rule_name)
    markers_to_replace = {}   # {clause_marker: (clause_text, rule_name)}
    rules_applied = []
    rules_skipped = []
    log_entries = []

    for rule in rules:
        if not rule.is_active:
            rules_skipped.append({
                "rule": rule.rule_name,
                "reason": "Rule is inactive"
            })
            continue

        # 4a. Kosulu degerlendir
        match_result = evaluate_condition(
            condition_type=rule.condition_type,
            condition_operator=rule.condition_operator,
            condition_value=rule.condition_value,
            seller_attributes=seller_attributes
        )

        if not match_result["matched"]:
            rules_skipped.append({
                "rule": rule.rule_name,
                "reason": f"Condition not met: {rule.condition_type} "
                          f"{rule.condition_operator} {rule.condition_value} "
                          f"(actual: {match_result['actual_value']})"
            })
            continue

        # 4b. Cakisma kontrolu (conflict_resolution)
        if template.conflict_resolution == "first_match":
            # Ayni clause_marker icin sadece ilk eslesen uygulanir
            if rule.action in ("remove_clause", "replace_clause"):
                if rule.clause_marker in markers_to_replace or \
                   rule.clause_marker in markers_to_remove:
                    rules_skipped.append({
                        "rule": rule.rule_name,
                        "reason": f"Conflict: marker '{rule.clause_marker}' "
                                  f"already processed (first_match mode)"
                    })
                    continue

        # 4c. Aksiyonu kaydet
        if rule.action == "add_clause":
            clauses_to_add.append((
                rule.clause_order,
                rule.clause_text,
                rule.rule_name
            ))
        elif rule.action == "remove_clause":
            markers_to_remove.append((rule.clause_marker, rule.rule_name))
        elif rule.action == "replace_clause":
            markers_to_replace[rule.clause_marker] = (
                rule.clause_text,
                rule.rule_name
            )

        rules_applied.append({
            "rule": rule.rule_name,
            "condition": f"{rule.condition_type} {rule.condition_operator} "
                         f"{rule.condition_value}",
            "action": rule.action,
            "clause_order": rule.clause_order
        })

        log_entries.append(
            f"[APPLIED] {rule.rule_name}: "
            f"{rule.condition_type}={match_result['actual_value']} "
            f"-> {rule.action}"
        )

    # ===== ADIM 5: Icerigi Derle =====

    # 5a. Marker-bazli degisiklikleri uygula (remove + replace)
    for marker, rule_name in markers_to_remove:
        pattern = f"{{{{{marker}}}}}"  # {{MARKER_NAME}} formatinda
        if pattern in compiled_content:
            compiled_content = compiled_content.replace(pattern, "")
            log_entries.append(f"[REMOVED] Marker '{marker}' by rule '{rule_name}'")
        else:
            log_entries.append(
                f"[WARNING] Marker '{marker}' not found in base_text "
                f"(rule: {rule_name})"
            )

    for marker, (replacement_text, rule_name) in markers_to_replace.items():
        pattern = f"{{{{{marker}}}}}"
        if pattern in compiled_content:
            compiled_content = compiled_content.replace(pattern, replacement_text)
            log_entries.append(
                f"[REPLACED] Marker '{marker}' by rule '{rule_name}'"
            )
        else:
            log_entries.append(
                f"[WARNING] Marker '{marker}' not found in base_text "
                f"(rule: {rule_name})"
            )

    # 5b. Ek maddeleri sirala ve ekle
    clauses_to_add.sort(key=lambda x: x[0])  # clause_order'a gore sirala
    if clauses_to_add:
        compiled_content += "\n\n---\n\n## Ek Maddeler\n\n"
        for order, text, rule_name in clauses_to_add:
            compiled_content += f"{text}\n\n"
            log_entries.append(
                f"[ADDED] Clause at order {order} by rule '{rule_name}'"
            )

    # ===== ADIM 6: Hash Hesapla ve Sonucu Dondur =====
    import hashlib
    content_hash = hashlib.sha256(compiled_content.encode("utf-8")).hexdigest()

    return {
        "compiled_content": compiled_content,
        "rules_applied": rules_applied,
        "rules_skipped": rules_skipped,
        "content_hash": content_hash,
        "compilation_log": "\n".join(log_entries)
    }
```

### 5.2 Kosul Degerlendirme Fonksiyonu

```python
# Pseudocode — Tekil kosul degerlendirmesi

def evaluate_condition(condition_type, condition_operator, condition_value, seller_attributes):
    """
    Tek bir kosul kuralini satici ozelliklerine karsi degerlendirir.

    Returns:
        dict: {"matched": bool, "actual_value": any}
    """

    # 1. Satici ozelligini al
    actual_value = seller_attributes.get(condition_type)

    if actual_value is None:
        return {"matched": False, "actual_value": None}

    # 2. Operatore gore karsilastir
    if condition_operator == "equals":
        matched = str(actual_value) == str(condition_value)

    elif condition_operator == "not_equals":
        matched = str(actual_value) != str(condition_value)

    elif condition_operator == "in":
        # condition_value: "val1,val2,val3" formatinda
        values = [v.strip() for v in condition_value.split(",")]
        if isinstance(actual_value, list):
            # Liste icinde kesisim var mi? (orn: categories)
            matched = bool(set(actual_value) & set(values))
        else:
            matched = str(actual_value) in values

    elif condition_operator == "not_in":
        values = [v.strip() for v in condition_value.split(",")]
        if isinstance(actual_value, list):
            matched = not bool(set(actual_value) & set(values))
        else:
            matched = str(actual_value) not in values

    elif condition_operator == "greater_than":
        matched = float(actual_value) > float(condition_value)

    elif condition_operator == "less_than":
        matched = float(actual_value) < float(condition_value)

    elif condition_operator == "between":
        # condition_value: "min-max" formatinda (orn: "100000-500000")
        parts = condition_value.split("-")
        low, high = float(parts[0]), float(parts[1])
        matched = low <= float(actual_value) <= high

    elif condition_operator == "contains":
        matched = str(condition_value).lower() in str(actual_value).lower()

    else:
        matched = False

    return {"matched": matched, "actual_value": actual_value}
```

### 5.3 Satici Ozellik Toplama Fonksiyonu

```python
# Pseudocode — Satici ozelliklerini topla

def collect_seller_attributes(seller_doc):
    """
    Seller Profile belgesinden kural degerlendirmesi icin
    gerekli tum ozellikleri toplar.

    Args:
        seller_doc: frappe.get_doc("Seller Profile", name)

    Returns:
        dict: Satici ozellikleri
    """
    attributes = {}

    # 1. Abonelik Paketi
    active_sub = frappe.db.get_value(
        "Subscription",
        {"seller_profile": seller_doc.name, "status": "Active"},
        ["name", "subscription_package"],
        as_dict=True
    )
    if active_sub:
        package = frappe.get_doc("Subscription Package", active_sub.subscription_package)
        attributes["package"] = package.package_name
    else:
        attributes["package"] = "Free"

    # 2. Kategoriler (liste olarak)
    categories = [
        row.category_name
        for row in (seller_doc.categories or [])
    ]
    attributes["category"] = categories

    # 3. Yillik Ciro
    attributes["revenue_range"] = seller_doc.annual_revenue or 0

    # 4. Satici Tipi
    attributes["seller_type"] = seller_doc.seller_type or "individual"

    # 5. Konum
    attributes["location"] = seller_doc.city or seller_doc.region or ""

    # 6. Kayit Suresi (gun)
    from frappe.utils import date_diff, today
    attributes["registration_age"] = date_diff(today(), seller_doc.creation)

    # 7. Ozel Ozellikler (JSON alani)
    import json
    try:
        attributes["custom"] = json.loads(seller_doc.custom_attributes or "{}")
    except (json.JSONDecodeError, TypeError):
        attributes["custom"] = {}

    return attributes
```

### 5.4 Siralama ve Filtreleme Fonksiyonu

```python
# Pseudocode — Aktif kurallari al ve sirala

def get_sorted_active_rules(template_doc):
    """
    Sablon icindeki aktif kurallari belirlenen siralama
    stratejisine gore dondurur.

    Args:
        template_doc: frappe.get_doc("Contract Template", name)

    Returns:
        list: Siralanmis kural listesi
    """
    rules = [
        rule for rule in (template_doc.condition_rules or [])
        if rule.is_active
    ]

    order = template_doc.rule_evaluation_order or "priority_asc"

    if order == "priority_asc":
        rules.sort(key=lambda r: (r.priority or 999, r.clause_order or 0))

    elif order == "priority_desc":
        rules.sort(key=lambda r: (-(r.priority or 0), r.clause_order or 0))

    elif order == "condition_type_grouped":
        # condition_type'a gore grupla, iceride priority_asc
        type_order = {
            "package": 1,
            "seller_type": 2,
            "category": 3,
            "revenue_range": 4,
            "location": 5,
            "registration_age": 6,
            "custom": 7
        }
        rules.sort(key=lambda r: (
            type_order.get(r.condition_type, 99),
            r.priority or 999,
            r.clause_order or 0
        ))

    return rules
```

# Pseudocode — Genisletilmis create_instance

@frappe.whitelist(allow_guest=False)
def create_instance_dynamic(template, party_type, party, signature_method=None,
                            expiry_days=None, force_recompile=False):
    """
    Dinamik kural destekli sozlesme ornegi olusturur.
    Mevcut create_instance API'sini genisletir.
    """
    template_doc = frappe.get_doc("Contract Template", template)

    if template_doc.dynamic_rules_enabled:
        # Dinamik derleme
        result = compile_dynamic_contract(template, party)

        # Compiled Output kaydi olustur
        compiled_output = frappe.get_doc({
            "doctype": "Contract Compiled Output",
            "contract_template": template,
            "template_version": template_doc.version,
            "seller": party,
            "compiled_content": result["compiled_content"],
            "content_hash": result["content_hash"],
            "rules_applied": frappe.as_json(result["rules_applied"]),
            "rules_skipped": frappe.as_json(result["rules_skipped"]),
            "compilation_timestamp": frappe.utils.now_datetime(),
            "compilation_log": result["compilation_log"],
            "status": "Ready"
        })
        compiled_output.insert()

        # Contract Instance olustur (derlenmis icerik ile)
        doc = frappe.get_doc({
            "doctype": "Contract Instance",
            "template": template,
            "party_type": party_type,
            "party": party,
            "signature_method": signature_method or "Digital",
            "status": "Draft",
            "template_content_snapshot": result["compiled_content"],
            "template_version_snapshot": template_doc.version,
            "content_hash_snapshot": result["content_hash"]
        })
    else:
        # Statik mod — mevcut davranis
        doc = frappe.get_doc({
            "doctype": "Contract Instance",
            "template": template,
            "party_type": party_type,
            "party": party,
            "signature_method": signature_method or "Digital",
            "status": "Draft"
        })

    if expiry_days:
        doc.valid_until = frappe.utils.add_days(
            frappe.utils.now_datetime(), int(expiry_days)
        )

    doc.insert()

    # Compiled Output'u Contract Instance ile iliskilendir
    if template_doc.dynamic_rules_enabled:
        compiled_output.contract_instance = doc.name
        compiled_output.status = "Used"
        compiled_output.save()

    frappe.db.commit()

    return {
        "success": True,
        "data": {
            "contract_instance": doc.name,
            "dynamic_compilation": template_doc.dynamic_rules_enabled,
            "rules_applied_count": len(result["rules_applied"]) if template_doc.dynamic_rules_enabled else 0,
            "content_hash": doc.content_hash_snapshot

    return {
        "total_sellers": len(all_sellers),
        "affected_sellers": len(affected),
        "affected_percentage": round(len(affected) / max(len(all_sellers), 1) * 100, 2),

## 11. Edge Cases (Ucs Durumlar)

### 11.1 Esanli Duzenleme (Concurrent Editing)

**Senaryo:** Iki admin ayni anda ayni sablon uzerinde kural eklemeye calisiyor.

**Cozum:**
- Frappe'nin yerlesik `modified` alani ve `frappe.flags.ignore_version` mekanizmasi kullanilir
- Ikinci kaydetme denemesinde "Document has been modified after you last read it" uyarisi gosterilir
- Admin, sayfayi yenileyerek guncel versiyonu alir ve degisikligini tekrar uygular
- **Oneri:** Uzun sureli duzenleme islemleri icin "Draft" durumda calismak ve tamamlandiginda "Publish" yapmak

```python
# Frappe'nin mevcut conflict detection mekanizmasi
# Contract Template.save() sirasinda otomatik kontrol:
if self.modified != frappe.db.get_value(self.doctype, self.name, "modified"):
    frappe.throw(
        _("This document has been modified after you last read it. "
          "Please refresh and try again."),
        frappe.TimestampMismatchError
    )
```

### 11.2 Kural Cakismalari (Rule Conflicts)

**Senaryo:** Ayni marker icin birden fazla kural eslesir.

**Cozum: `conflict_resolution` stratejilerine gore:**

| Strateji | Davranis | Kullanim Alani |
|----------|----------|----------------|
| `first_match` | Ilk eslesen kural uygulanir, digerlerini atlar | Basit, hiyerarsik kurallar |
| `all_matching` | Tum eslesen kurallar uygulanir (add_clause icin sorunsuz; replace_clause icin son eslesen kazanir) | Kumulatif maddeler |
| `priority_based` | En dusuk priority degeri olan kural uygulanir | Oncelik gerektiren durumlar |

**Tespit ve Uyari:**

```python
# Pseudocode — Kural cakismasi tespit

def detect_rule_conflicts(template_doc):
    """
    Sablon icerisindeki potansiyel kural cakismalarini tespit eder.
    """
    conflicts = []
    marker_rules = {}  # {marker: [rules]}

    for rule in template_doc.condition_rules:
        if rule.action in ("remove_clause", "replace_clause") and rule.clause_marker:
            if rule.clause_marker not in marker_rules:
                marker_rules[rule.clause_marker] = []
            marker_rules[rule.clause_marker].append(rule)

    for marker, rules in marker_rules.items():
        if len(rules) > 1:
            conflicts.append({
                "marker": marker,
                "rules": [r.rule_name for r in rules],
                "severity": "high" if any(r.action == "replace_clause" for r in rules) else "medium"
            })

    return conflicts
```

### 11.3 Gecersiz Marker Referanslari

**Senaryo:** Kural bir `clause_marker` referans eder ama `base_text` icinde bu marker bulunmaz.

**Cozum:**
- Derleme sirasinda `[WARNING]` log kaydi olusturulur
- Admin'e in-app bildirim gonderilir
- Sozlesme yine de olusturulur (marker atlanir)
- Sablon kaydetme sirasinda validasyon uyarisi gosterilir (hata degil)

```python
# Pseudocode — Marker validasyonu (save sirasinda)

def validate_markers(self):
    """Contract Template save sirasinda marker kontrolu."""
    if not self.dynamic_rules_enabled or not self.base_text:
        return

    # base_text icindeki tum marker'lari bul
    import re
    markers_in_text = set(re.findall(r'\{\{(\w+)\}\}', self.base_text))

    # Kural referanslarindaki marker'lari topla
    markers_in_rules = set()
    for rule in (self.condition_rules or []):
        if rule.clause_marker and rule.action in ("remove_clause", "replace_clause"):
            markers_in_rules.add(rule.clause_marker)

    # Referans edilen ama metinde bulunmayan marker'lar
    missing = markers_in_rules - markers_in_text
    if missing:
        frappe.msgprint(
            _("Warning: The following markers are referenced in rules but "
              "not found in base_text: {0}").format(", ".join(missing)),
            title=_("Marker Validation"),
            indicator="orange"
        )

    # Metinde bulunan ama kurallar tarafindan referans edilmeyen marker'lar
    unused = markers_in_text - markers_in_rules
    if unused:
        frappe.msgprint(
            _("Info: The following markers in base_text have no rules: {0}. "
              "They will remain as-is in the compiled output.").format(", ".join(unused)),
            title=_("Marker Validation"),
            indicator="blue"
        )
```

### 11.4 Satici Ozelligi Bulunamadiginda

**Senaryo:** Kural `revenue_range` kosulu kullaniyor ama saticinin `annual_revenue` alani bos.

**Cozum:**
- `evaluate_condition` fonksiyonu `None` kontrolu yapar ve `matched = False` dondurur
- Kural `rules_skipped` listesine eklenir ve neden: "actual_value is None" olarak loglanir
- Sozlesme derleme islemi devam eder (hataya dusmez)

### 11.5 Sablon Guncelleme Sonrasi Mevcut Sozlesmeler

**Senaryo:** Sablon guncellendi — daha once imzalanmis sozlesmeler etkilenir mi?

**Cozum:**
- **Imzalanmis sozlesmeler etkilenmez** — `template_content_snapshot` ve `content_hash_snapshot` degisikligi olusturuldugu andaki icerigi saklar
- **Bekleyen (Draft/Sent) sozlesmeler** icin admin "Yeniden Derle" tetikleyebilir
- Yeniden derleme sonrasi yeni `Contract Revision` kaydi olusturulur

### 11.6 Bos Kural Seti

**Senaryo:** `dynamic_rules_enabled = 1` ama hicbir aktif kural tanimlanmamis.

**Cozum:**
- `base_text` aynen dondurulur (kural uygulanmadan)
- Log: "No active rules found — using base_text as-is"
- Admin'e uyari: "Dinamik kurallar aktif ama hicbir kural tanimlanmamis"

### 11.7 Cok Sayida Kural ile Performans

**Senaryo:** Bir sablonda 100+ kural tanimlanmis.

**Cozum:**
- Kurallarin child table olarak saklanmasi nedeniyle `frappe.get_doc()` ile birlikte yuklenir (ayri sorgu gerekmez)
- 100+ kural ile bile derleme suresi < 1 saniye beklenir (cogu islem string manipulasyonu)
- Toplu derleme (100+ satici) icin `frappe.enqueue()` ile background job kullanilir
- **Oneri:** Pratik kullanim icin sablon basina 50 kural siniri koymak (yumusak sinir, uyari ile)

## 12. Guvenlik Hususlari (Security Considerations)

### 12.1 Icerik Enjeksiyonu

- `clause_text` alani Text Editor tipinde — HTML icerebilir
- Frappe'nin yerlesik `sanitize_html()` fonksiyonu derleme sirasinda uygulanir
- Jinja template injection onlenmeli — `clause_text` icinde `{% %}` veya `{{ }}` (Jinja syntax) kullanilmasina izin verilmez (Marker'lar `{{MARKER}}` formatinda sadece buyuk harf ve alt cizgi)

### 12.2 Erisim Kontrolu

- Saticilar `Contract Compiled Output` ve `Contract Instance` sadece kendi kayitlarini gorebilir
- Sablon ve kural yonetimi yalnizca `System Manager` ve `Contract Manager` rollerine aciktir
- `preview_compiled_contract` API yalnizca admin rolleri tarafindan cagirilabilir

### 12.3 Denetim Izi

- Her derleme islemi `Contract Compiled Output` olarak saklanir (silinmez)
- `compilation_log` hangi kurallarin uygulandi/atlandigi bilgisini icerir
- `content_hash` ile icerik butunlugu dogrulanabilir

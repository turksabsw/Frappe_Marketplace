## 1. Overview

### 1.1 Problem Statement

In B2B marketplace operations, sellers frequently cannot fulfill an entire order in a single shipment. Common scenarios include:

- **Inventory constraints**: Only 600 of 1,000 ordered units are available now; the remaining 400 will ship next week.
- **Multi-warehouse fulfillment**: Items ship from different warehouse locations on different dates.
- **Production lead times**: Custom/manufactured items ship as production batches complete.
- **Weight/size limits**: Carrier restrictions require splitting large orders across multiple shipments.

The system must track how much of each order line item has been shipped, automatically calculate remaining balances, prevent over-shipment, and update order/sub-order statuses as fulfillment progresses.

### 1.2 Scope

This document defines:

1. **Balance calculation formulas** - Per-line-item qty tracking (ordered vs shipped vs remaining)
2. **Delivery Note structure** - How each partial shipment records which order, which items, and how much was shipped
3. **Auto status updates** - Automatic order status progression based on shipped quantities
4. **Over-shipment prevention** - Validation logic to prevent shipping more than ordered
5. **Integration with existing fulfillment fields** - `items_packed`, `items_shipped`, `items_delivered` on Marketplace Order and Sub Order
6. **Shipment-to-order linking** - How the existing Shipment DocType (tradehub_logistics) connects to partial deliveries

## 2. Balance Calculation System

### 2.1 Core Formulas

#### 2.1.1 Per-Line-Item Balance

For each line item in a Marketplace Order or Sub Order:

```
ordered_qty       = item.qty                              # Original order quantity
shipped_qty       = SUM(Delivery Note Item.qty)           # From all submitted Delivery Notes for this item
                    WHERE Delivery Note.docstatus = 1
                    AND Delivery Note Item.custom_marketplace_order_item = item.name
remaining_qty     = ordered_qty - shipped_qty             # Balance to ship (auto-calculated)
delivered_qty     = item.delivered_qty                     # Updated when delivery is confirmed
returned_qty      = item.returned_qty                     # Updated when return is processed
net_delivered_qty = delivered_qty - returned_qty           # Actual net quantity with buyer
pending_qty       = ordered_qty - delivered_qty            # Quantity not yet confirmed delivered
```

#### 2.1.2 Shipped vs Delivered Distinction

In B2B logistics, "shipped" and "delivered" are distinct events:

| Term | Definition | Trigger |
|------|-----------|---------|
| **Shipped Qty** | Quantity handed over to carrier (left warehouse) | Delivery Note submitted |
| **Delivered Qty** | Quantity confirmed received by buyer | Shipment status = "Delivered" or POD received |
| **Remaining Qty** | Quantity still to be shipped from warehouse | `ordered_qty - shipped_qty` |
| **Pending Delivery Qty** | Quantity shipped but not yet confirmed delivered | `shipped_qty - delivered_qty` |

```
Timeline:
  Order placed (qty=1000)
  -> Shipment 1 submitted (600 shipped)     remaining=400, shipped=600, delivered=0
  -> Shipment 1 delivered (600 confirmed)   remaining=400, shipped=600, delivered=600
  -> Shipment 2 submitted (400 shipped)     remaining=0,   shipped=1000, delivered=600
  -> Shipment 2 delivered (400 confirmed)   remaining=0,   shipped=1000, delivered=1000
```

#### 2.1.3 Order-Level Aggregation

At the order level, quantities are aggregated across all line items:

```python
# Order-level balance summary
total_ordered    = SUM(item.qty for item in order.items)
total_shipped    = SUM(shipped_qty for item in order.items)  # From Delivery Notes
total_delivered  = SUM(item.delivered_qty for item in order.items)
total_remaining  = SUM(remaining_qty for item in order.items)
total_returned   = SUM(item.returned_qty for item in order.items)

# Percentages
fulfillment_pct  = (total_shipped / total_ordered) * 100
delivery_pct     = (total_delivered / total_ordered) * 100
```

### 2.2 Balance Calculation Implementation

#### 2.2.1 `_get_delivered_qty()` (defined in 125.md)

This helper function is the core SQL query for computing shipped quantity from Delivery Notes:

```python
def _get_delivered_qty(order_name, order_item_name):
    """
    Get total qty already delivered/shipped for an order item.

    Sums qty from all submitted Delivery Note Items
    that reference this Marketplace Order Item.

    Args:
        order_name: Marketplace Order name
        order_item_name: Marketplace Order Item name (child row)

    Returns:
        float: Total shipped qty from submitted Delivery Notes
    """
    result = frappe.db.sql("""
        SELECT IFNULL(SUM(dni.qty), 0) as delivered_qty
        FROM `tabDelivery Note Item` dni
        JOIN `tabDelivery Note` dn ON dn.name = dni.parent
        WHERE dn.custom_marketplace_order = %(order_name)s
        AND dni.custom_marketplace_order_item = %(order_item_name)s
        AND dn.docstatus = 1
    """, {
        "order_name": order_name,
        "order_item_name": order_item_name
    }, as_dict=True)

    return flt(result[0].delivered_qty) if result else 0
```

#### 2.2.2 `get_order_balance_summary()` - New API Method

```python
@frappe.whitelist()
def get_order_balance_summary(order_name):
    """
    Get complete balance summary for all items in an order.

    Returns per-item and order-level shipped/remaining quantities
    by querying all submitted Delivery Notes linked to this order.

    Args:
        order_name (str): Marketplace Order name

    Returns:
        dict: {
            "items": [
                {
                    "item_name": str,
                    "listing": str,
                    "sku": str,
                    "ordered_qty": float,
                    "shipped_qty": float,
                    "delivered_qty": float,
                    "remaining_qty": float,
                    "returned_qty": float,
                    "net_qty": float,
                    "fulfillment_status": str,
                    "fulfillment_pct": float,
                    "delivery_notes": [
                        {"name": str, "qty": float, "date": str, "status": str}
                    ]
                }
            ],
            "summary": {
                "total_ordered": float,
                "total_shipped": float,
                "total_delivered": float,
                "total_remaining": float,
                "total_returned": float,
                "fulfillment_pct": float,
                "delivery_pct": float,
                "shipment_count": int,
                "is_fully_shipped": bool,
                "is_fully_delivered": bool
            }
        }
    """
    order = frappe.get_doc("Marketplace Order", order_name)

    # Permission check
    if frappe.session.user != "Administrator":
        if not frappe.has_permission("Marketplace Order", "read", order):
            frappe.throw(_("Not permitted to view this order"))

    items = []
    total_ordered = 0
    total_shipped = 0
    total_delivered = 0
    total_remaining = 0
    total_returned = 0

    for item in order.items:
        # Get shipped qty from Delivery Notes
        shipped_qty = _get_delivered_qty(order_name, item.name)
        remaining_qty = flt(item.qty) - flt(shipped_qty)

        # Get delivery note details for this item
        delivery_notes = frappe.db.sql("""
            SELECT
                dn.name,
                dni.qty,
                dn.posting_date as date,
                dn.docstatus,
                CASE
                    WHEN dn.docstatus = 0 THEN 'Draft'
                    WHEN dn.docstatus = 1 THEN 'Submitted'
                    WHEN dn.docstatus = 2 THEN 'Cancelled'
                END as status
            FROM `tabDelivery Note Item` dni
            JOIN `tabDelivery Note` dn ON dn.name = dni.parent
            WHERE dn.custom_marketplace_order = %(order_name)s
            AND dni.custom_marketplace_order_item = %(item_name)s
            AND dn.docstatus = 1
            ORDER BY dn.posting_date ASC
        """, {
            "order_name": order_name,
            "item_name": item.name
        }, as_dict=True)

        fulfillment_pct = (flt(shipped_qty) / flt(item.qty) * 100) if flt(item.qty) > 0 else 0

        items.append({
            "item_name": item.name,
            "listing": item.listing,
            "title": item.title,
            "sku": item.sku,
            "ordered_qty": flt(item.qty),
            "shipped_qty": flt(shipped_qty),
            "delivered_qty": flt(item.delivered_qty),
            "remaining_qty": max(flt(remaining_qty), 0),
            "returned_qty": flt(item.returned_qty),
            "net_qty": flt(item.delivered_qty) - flt(item.returned_qty),
            "fulfillment_status": item.fulfillment_status,
            "fulfillment_pct": fulfillment_pct,
            "delivery_notes": delivery_notes
        })

        total_ordered += flt(item.qty)
        total_shipped += flt(shipped_qty)
        total_delivered += flt(item.delivered_qty)
        total_remaining += max(flt(remaining_qty), 0)
        total_returned += flt(item.returned_qty)

    # Count unique shipments
    shipment_count = frappe.db.count(
        "Delivery Note",
        filters={
            "custom_marketplace_order": order_name,
            "docstatus": 1
        }
    )

    fulfillment_pct = (total_shipped / total_ordered * 100) if total_ordered > 0 else 0
    delivery_pct = (total_delivered / total_ordered * 100) if total_ordered > 0 else 0

    return {
        "items": items,
        "summary": {
            "total_ordered": total_ordered,
            "total_shipped": total_shipped,
            "total_delivered": total_delivered,
            "total_remaining": total_remaining,
            "total_returned": total_returned,
            "fulfillment_pct": fulfillment_pct,
            "delivery_pct": delivery_pct,
            "shipment_count": shipment_count,
            "is_fully_shipped": total_remaining == 0,
            "is_fully_delivered": total_delivered >= total_ordered
        }
    }
```

## 3. Delivery Note Structure for Partial Shipments

### 3.1 Concept

Each partial shipment is recorded as a separate **ERPNext Delivery Note**. Multiple Delivery Notes can be created from a single Marketplace Order, each containing a subset of the order items with their specific shipped quantities.

```
Marketplace Order MO-2026-00001
  |-- Item A: 1000 units ordered
  |-- Item B: 500 units ordered
  |-- Item C: 200 units ordered
  |
  |-- Delivery Note DN-00001 (Shipment 1 - 2026-03-05)
  |     |-- Item A: 600 units
  |     |-- Item B: 500 units (fully shipped)
  |
  |-- Delivery Note DN-00002 (Shipment 2 - 2026-03-12)
  |     |-- Item A: 400 units (balance completed)
  |     |-- Item C: 100 units (partial)
  |
  |-- Delivery Note DN-00003 (Shipment 3 - 2026-03-19)
        |-- Item C: 100 units (balance completed)
```

### 3.3 Delivery Note Creation Flow (from 125.md)

The `make_delivery_note()` function defined in 125.md handles partial shipment creation:

1. **Query remaining qty**: For each order item, calculate `remaining_qty = qty - _get_delivered_qty()`
2. **Filter items**: Only include items where `remaining_qty > 0`
3. **Pre-fill qty**: Set each Delivery Note Item's qty to the remaining amount (user can adjust down)
4. **Link back**: Set `custom_marketplace_order`, `custom_marketplace_order_item` on created document
5. **Validation**: Ensure source order is submitted and has eligible items

## 4. Auto Status Updates

### 4.1 Fulfillment Status Flow

The fulfillment status progression follows this path based on shipped/delivered quantities:

```
                  +-----------+
                  |  Pending  |  (No items shipped yet)
                  +-----------+
                       |
                       | First item packed
                       v
                +-----------------+
                | Partially Packed|  (Some items packed, not all)
                +-----------------+
                       |
                       | All items packed
                       v
                  +-----------+
                  |  Packed   |  (All items packed, none shipped)
                  +-----------+
                       |
                       | First Delivery Note submitted
                       v
                +-------------------+
                | Partially Shipped |  (remaining_qty > 0 for some items)
                +-------------------+
                       |
                       | All items shipped (remaining_qty = 0 for all)
                       v
                  +-----------+
                  |  Shipped  |  (All items shipped, not all delivered)
                  +-----------+
                       |
                       | First item delivered
                       v
                +---------------------+
                | Partially Delivered |  (delivered_qty < qty for some items)
                +---------------------+
                       |
                       | All items delivered (delivered_qty = qty for all)
                       v
                  +-----------+
                  | Delivered |  (All items confirmed delivered)
                  +-----------+
```

### 4.2 Order Status Flow with Partial Shipment

The Marketplace Order `status` field transitions with partial shipment context:

```
Pending -> Await Payment -> Payment Received -> Confirmed -> Processing -> Packed
                                                                            |
                                                           First Delivery Note submitted
                                                                            |
                                                                            v
                                                                        Shipped
                                                                (fulfillment_status tracks
                                                                 Partially Shipped vs Shipped)
                                                                            |
                                                            Carrier confirms delivery
                                                                            |
                                                                            v
                                                                      Delivered
                                                                            |
                                                              Buyer confirms / auto-complete
                                                                            |
                                                                            v
                                                                      Completed
```

**Key Decision**: The order-level `status` moves to "Shipped" on the FIRST Delivery Note submission. The distinction between partial and full shipment is tracked via `fulfillment_status` (which has "Partially Shipped" and "Shipped" as separate values).

### 4.3 Status Update Triggers

#### 4.3.1 On Delivery Note Submit

When an ERPNext Delivery Note linked to a Marketplace Order is submitted:

```python
def on_delivery_note_submit(doc, method):
    """
    Hook: Called when a Delivery Note is submitted.

    Updates Marketplace Order and Sub Order fulfillment tracking.
    Registered in hooks.py:
        doc_events = {
            "Delivery Note": {
                "on_submit": "tradehub_commerce...on_delivery_note_submit",
                "on_cancel": "tradehub_commerce...on_delivery_note_cancel"
            }
        }
    """
    marketplace_order_name = doc.custom_marketplace_order
    if not marketplace_order_name:
        return

    order = frappe.get_doc("Marketplace Order", marketplace_order_name)

    # --- Step 1: Update per-item shipped quantities ---
    for dn_item in doc.items:
        mo_item_name = dn_item.custom_marketplace_order_item
        if not mo_item_name:
            continue

        # Find the matching Marketplace Order Item
        for order_item in order.items:
            if order_item.name == mo_item_name:
                # Recalculate total shipped qty from all Delivery Notes
                total_shipped = _get_delivered_qty(
                    marketplace_order_name,
                    mo_item_name
                )

                # Update fulfillment status based on shipped qty
                if flt(total_shipped) >= flt(order_item.qty):
                    order_item.db_set("fulfillment_status", "Shipped")
                elif flt(total_shipped) > 0:
                    order_item.db_set("fulfillment_status", "Shipped")
                    # Note: "Partially Shipped" is not in Marketplace Order Item
                    # statuses. At item level, any shipment = "Shipped".
                    # Partial tracking is at order level.
                break

    # --- Step 2: Update order-level fulfillment counts ---
    order.update_fulfillment_counts()

    # --- Step 3: Update order status ---
    _update_order_status_after_shipment(order)

    # --- Step 4: Update Sub Order fulfillment ---
    _update_sub_order_fulfillment(order, doc)

    # --- Step 5: Update Delivery Note metadata ---
    _update_delivery_note_metadata(doc, marketplace_order_name)

    # --- Step 6: Notify buyer ---
    _notify_buyer_shipment(order, doc)
```

#### 4.3.2 `_update_order_status_after_shipment()`

```python
def _update_order_status_after_shipment(order):
    """
    Update order status based on current fulfillment state.

    Status transitions:
    - First shipment: Processing/Packed -> Shipped
    - fulfillment_status tracks partial vs full

    The order.status remains "Shipped" regardless of whether
    shipment is partial or complete. The distinction is in
    fulfillment_status.
    """
    # Only advance if order is in a shippable state
    if order.status not in ["Processing", "Packed", "Shipped"]:
        return

    # Check if any items have been shipped
    has_shipped = any(
        item.fulfillment_status in ["Shipped", "In Transit", "Out for Delivery", "Delivered"]
        for item in order.items
    )

    if has_shipped and order.status in ["Processing", "Packed"]:
        order.db_set("status", "Shipped")
        order.db_set("shipped_at", frappe.utils.now_datetime())

    # Fulfillment status is already updated by update_fulfillment_counts()
    # which sets: Partially Shipped / Shipped / Partially Delivered / Delivered
```

def _update_sub_order_fulfillment(order, delivery_note):
    """
    Update Sub Order fulfillment based on Delivery Note items.

    Each Delivery Note Item maps back to a Marketplace Order Item,
    which has a sub_order Link field. We use this to find and update
    the corresponding Sub Order.
    """
    affected_sub_orders = set()

    for dn_item in delivery_note.items:
        mo_item_name = dn_item.custom_marketplace_order_item
        if not mo_item_name:
            continue

        # Find the Sub Order for this item
        sub_order_name = frappe.db.get_value(
            "Marketplace Order Item",
            mo_item_name,
            "sub_order"
        )
        if sub_order_name:
            affected_sub_orders.add(sub_order_name)

    # Update each affected Sub Order
    for sub_order_name in affected_sub_orders:
        sub_order = frappe.get_doc("Sub Order", sub_order_name)

        # Update Sub Order Item shipped quantities
        for so_item in sub_order.items:
            if so_item.marketplace_order_item:
                shipped_qty = _get_delivered_qty(
                    order.name,
                    so_item.marketplace_order_item
                )
                # Note: Sub Order Item.delivered_qty represents confirmed delivery,
                # not shipment. We track shipped separately.

        # Update Sub Order fulfillment counts
        sub_order.update_fulfillment_counts()

        # Update Sub Order status
        if sub_order.status in ["Processing", "Packed", "Accepted"]:
            sub_order.db_set("status", "Shipped")
            sub_order.db_set("shipped_at", frappe.utils.now_datetime())
            sub_order.db_set("fulfillment_status", "Shipped")

    - Updates fulfillment counts

### 4.4 Delivery Confirmation (Shipped -> Delivered)

When a Shipment (tradehub_logistics) tracking status changes to "Delivered", or when the seller manually confirms delivery:

```python
@frappe.whitelist()
def confirm_delivery(order_name, delivery_note_name=None, items=None):
    """
    Confirm delivery of shipped items.

    Can confirm delivery for:
    - All items in a specific Delivery Note
    - Specific items/quantities (for partial delivery confirmation)
    - All remaining shipped items

    Args:
        order_name (str): Marketplace Order name
        delivery_note_name (str, optional): Specific Delivery Note
        items (list, optional): List of {"item_name": str, "qty": float}
            to confirm delivery for

    Returns:
        dict: Updated fulfillment status

    Status Effect:
        - Updates item.delivered_qty
        - Updates item.fulfillment_status
        - Updates order.fulfillment_status
        - May advance order.status to "Delivered" if fully delivered
    """
    order = frappe.get_doc("Marketplace Order", order_name)

    if delivery_note_name:
        # Confirm all items in this Delivery Note
        dn = frappe.get_doc("Delivery Note", delivery_note_name)
        for dn_item in dn.items:
            mo_item_name = dn_item.custom_marketplace_order_item
            if not mo_item_name:
                continue

            for order_item in order.items:
                if order_item.name == mo_item_name:
                    new_delivered = flt(order_item.delivered_qty) + flt(dn_item.qty)
                    order_item.db_set("delivered_qty", new_delivered)

                    # Update fulfillment status
                    if flt(new_delivered) >= flt(order_item.qty):
                        order_item.db_set("fulfillment_status", "Delivered")
                    elif flt(new_delivered) > 0:
                        order_item.db_set("fulfillment_status", "Partially Delivered")
                    break

    elif items:
        # Confirm specific items with specific quantities
        for item_data in items:
            item_name = item_data.get("item_name")
            qty = flt(item_data.get("qty"))

            for order_item in order.items:
                if order_item.name == item_name:
                    new_delivered = flt(order_item.delivered_qty) + qty
                    if new_delivered > flt(order_item.qty):
                        frappe.throw(
                            _("Cannot confirm delivery of {0} units for {1}. "
                              "Only {2} units remaining.").format(
                                qty, order_item.title,
                                flt(order_item.qty) - flt(order_item.delivered_qty)
                            )
                        )
                    order_item.db_set("delivered_qty", new_delivered)

                    if flt(new_delivered) >= flt(order_item.qty):
                        order_item.db_set("fulfillment_status", "Delivered")
                    elif flt(new_delivered) > 0:
                        order_item.db_set("fulfillment_status", "Partially Delivered")
                    break

    # Update order-level fulfillment
    order.update_fulfillment_counts()

    # Check if fully delivered
    all_delivered = all(
        flt(item.delivered_qty) >= flt(item.qty)
        for item in order.items
    )

    if all_delivered:
        order.db_set("status", "Delivered")
        order.db_set("delivered_at", frappe.utils.now_datetime())
        order.db_set("fulfillment_status", "Delivered")
        order.db_set("fully_fulfilled", 1)

        # Schedule escrow release if applicable
        if order.escrow_status == "Held":
            order.schedule_escrow_release()

    elif any(flt(item.delivered_qty) > 0 for item in order.items):
        order.db_set("fulfillment_status", "Partially Delivered")

    # Update Sub Orders
    _sync_sub_order_delivery(order)

    order.clear_order_cache()

    return {
        "status": "success",
        "order_status": order.status,
        "fulfillment_status": order.fulfillment_status,
        "is_fully_delivered": all_delivered
    }
```

## 5. Over-Shipment Prevention

### 5.1 Validation Rules

Over-shipment must be prevented at multiple points in the workflow:

#### 5.1.1 Delivery Note Validate Hook

```python
def validate_delivery_note(doc, method):
    """
    Hook: Validate Delivery Note before save/submit.

    Prevents over-shipment by checking that the qty in this Delivery Note
    plus all previously submitted Delivery Notes does not exceed the
    ordered qty for each line item.

    Registered in hooks.py:
        doc_events = {
            "Delivery Note": {
                "validate": "tradehub_commerce...validate_delivery_note"
            }
        }
    """
    marketplace_order_name = doc.custom_marketplace_order
    if not marketplace_order_name:
        return

    order = frappe.get_doc("Marketplace Order", marketplace_order_name)

    for dn_item in doc.items:
        mo_item_name = dn_item.custom_marketplace_order_item
        if not mo_item_name:
            continue

        # Find ordered qty
        ordered_qty = 0
        for order_item in order.items:
            if order_item.name == mo_item_name:
                ordered_qty = flt(order_item.qty)
                break

        if ordered_qty == 0:
            frappe.throw(
                _("Item {0} not found in Marketplace Order {1}").format(
                    dn_item.item_name, marketplace_order_name
                )
            )

        # Calculate already shipped qty (excluding THIS Delivery Note if it's being amended)
        already_shipped = _get_delivered_qty_excluding(
            marketplace_order_name,
            mo_item_name,
            exclude_dn=doc.name if not doc.is_new() else None
        )

        # Total that would be shipped after this DN
        total_after = flt(already_shipped) + flt(dn_item.qty)

        if total_after > ordered_qty:
            over_amount = total_after - ordered_qty
            frappe.throw(
                _("Over-shipment detected for item <b>{item}</b>:<br>"
                  "Ordered: {ordered}<br>"
                  "Already shipped: {shipped}<br>"
                  "This shipment: {current}<br>"
                  "Total would be: {total}<br>"
                  "Over by: <b>{over}</b> units").format(
                    item=dn_item.item_name,
                    ordered=ordered_qty,
                    shipped=already_shipped,
                    current=dn_item.qty,
                    total=total_after,
                    over=over_amount
                ),
                title=_("Over-Shipment Not Allowed")
            )

        # Set balance context fields
        dn_item.custom_ordered_qty = ordered_qty
        dn_item.custom_previously_shipped_qty = already_shipped
        dn_item.custom_remaining_after_this = ordered_qty - total_after
```

#### 5.1.2 `_get_delivered_qty_excluding()`

```python
def _get_delivered_qty_excluding(order_name, order_item_name, exclude_dn=None):
    """
    Get total shipped qty, optionally excluding a specific Delivery Note.

    Used during validation to avoid double-counting the current DN
    when it's being saved/amended.

    Args:
        order_name: Marketplace Order name
        order_item_name: Marketplace Order Item name
        exclude_dn: Delivery Note name to exclude (optional)

    Returns:
        float: Total shipped qty
    """
    conditions = """
        dn.custom_marketplace_order = %(order_name)s
        AND dni.custom_marketplace_order_item = %(order_item_name)s
        AND dn.docstatus = 1
    """
    params = {
        "order_name": order_name,
        "order_item_name": order_item_name
    }

    if exclude_dn:
        conditions += " AND dn.name != %(exclude_dn)s"
        params["exclude_dn"] = exclude_dn

    result = frappe.db.sql("""
        SELECT IFNULL(SUM(dni.qty), 0) as shipped_qty
        FROM `tabDelivery Note Item` dni
        JOIN `tabDelivery Note` dn ON dn.name = dni.parent
        WHERE {conditions}
    """.format(conditions=conditions), params, as_dict=True)

    return flt(result[0].shipped_qty) if result else 0
```

### 6.4 `update_fulfillment_counts()` Enhancement

The existing `update_fulfillment_counts()` method (on both `MarketplaceOrder` and `SubOrder`) should be enhanced to include shipped quantity tracking:

```python
def update_fulfillment_counts(self):
    """
    Enhanced fulfillment count update with shipped qty tracking.

    Existing logic (from marketplace_order.py):
    - Counts items by fulfillment_status
    - Sets items_packed, items_shipped, items_delivered, items_returned
    - Determines overall fulfillment_status

    Enhanced with:
    - Per-item shipped_qty from Delivery Notes
    - remaining_qty calculation
    - shipment_count aggregation
    """
    packed = 0
    shipped = 0
    delivered = 0
    returned = 0
    total_items = len(self.items)
    total_shipped_qty = 0

    for item in self.items:
        status = item.fulfillment_status

        # Count by status (existing logic)
        if status in ["Packed", "Shipped", "In Transit", "Out for Delivery", "Delivered"]:
            packed += 1
        if status in ["Shipped", "In Transit", "Out for Delivery", "Delivered"]:
            shipped += 1
        if status == "Delivered":
            delivered += 1
        if status == "Returned":
            returned += 1

        # Calculate shipped qty from Delivery Notes (new)
        item_shipped_qty = _get_delivered_qty(self.name, item.name)
        item.db_set("shipped_qty", item_shipped_qty)
        item.db_set("remaining_qty", max(flt(item.qty) - flt(item_shipped_qty), 0))
        total_shipped_qty += item_shipped_qty

    # Set existing count fields
    self.db_set("items_packed", packed)
    self.db_set("items_shipped", shipped)
    self.db_set("items_delivered", delivered)
    self.db_set("items_returned", returned)

    # Set new aggregate fields
    self.db_set("shipped_qty_total", total_shipped_qty)

    # Shipment count
    shipment_count = frappe.db.count(
        "Delivery Note",
        filters={
            "custom_marketplace_order": self.name,
            "docstatus": 1
        }
    )
    self.db_set("shipment_count", shipment_count)

    if shipment_count > 0:
        last_date = frappe.db.sql("""
            SELECT MAX(posting_date) as last_date
            FROM `tabDelivery Note`
            WHERE custom_marketplace_order = %s AND docstatus = 1
        """, self.name, as_dict=True)
        if last_date and last_date[0].last_date:
            self.db_set("last_shipment_date", last_date[0].last_date)

    # Determine overall fulfillment status (existing logic enhanced)
    if delivered == total_items:
        self.db_set("fully_fulfilled", 1)
        self.db_set("fulfillment_status", "Delivered")
    elif delivered > 0:
        self.db_set("fulfillment_status", "Partially Delivered")
    elif shipped == total_items:
        self.db_set("fulfillment_status", "Shipped")
    elif shipped > 0:
        self.db_set("fulfillment_status", "Partially Shipped")
    elif packed == total_items:
        self.db_set("fulfillment_status", "Packed")
    elif packed > 0:
        self.db_set("fulfillment_status", "Partially Packed")
```

def _notify_buyer_shipment(order, delivery_note):
    """
    Notify buyer about partial/full shipment.

    Sends notification via frappe.publish_realtime and optionally email.
    """
    # Get balance summary for context
    balance = get_order_balance_summary(order.name)
    summary = balance["summary"]

    is_final = summary["is_fully_shipped"]

    message = {
        "order_id": order.order_id,
        "order_name": order.name,
        "delivery_note": delivery_note.name,
        "shipment_number": delivery_note.custom_shipment_number,
        "is_final_shipment": is_final,

        "total_shipped": summary["total_shipped"],
        "total_remaining": summary["total_remaining"],
        "fulfillment_pct": summary["fulfillment_pct"]
    }

    # Real-time notification
    frappe.publish_realtime(
        "order_shipment" if not is_final else "order_fully_shipped",
        message,
        user=order.buyer
    )

    # Email notification (if enabled)
    if frappe.db.get_single_value("TradeHub Commerce Settings", "notify_buyer_on_shipment"):
        subject = _("Order {0} - Shipment {1} Dispatched").format(
            order.order_id,
            delivery_note.custom_shipment_number
        )
        if is_final:
            subject = _("Order {0} - All Items Shipped!").format(order.order_id)

        # Send email via standard Frappe email template
        frappe.sendmail(
            recipients=[order.buyer_email],
            subject=subject,
            template="order_shipment_notification",
            args=message,
            reference_doctype="Marketplace Order",
            reference_name=order.name
        )

## 10. Edge Cases

### 10.1 Over-Shipment Attempt

**Scenario**: Seller tries to ship 700 units of an item where only 400 remain.

**Handling**:
- Server-side: `validate_delivery_note()` throws `frappe.throw()` with detailed error showing ordered, shipped, and remaining quantities.
- Client-side: `qty` field change handler caps the value and shows warning.
- The validation runs on BOTH save and submit to catch any bypass.

### 10.2 Zero Remaining but Not Completed

**Scenario**: All 1,000 units shipped and delivered, but buyer has returned 200 units. Order should not auto-complete.

**Handling**:
- `fully_fulfilled` checks `delivered_qty >= qty` for ALL items.
- Returns do not reduce `delivered_qty`; they increment `returned_qty`.
- `net_qty = delivered_qty - returned_qty` can be checked separately.
- Auto-completion only triggers if `fully_fulfilled = 1` AND `items_returned = 0`.
- If returns exist, order stays in "Delivered" status until return is resolved.

### 10.3 Cancelled Delivery Note

**Scenario**: Delivery Note DN-00002 (400 units of Item A) is cancelled after submission.

**Handling**:
- `on_delivery_note_cancel()` recalculates all shipped quantities.
- `_get_delivered_qty()` automatically excludes cancelled DNs (docstatus = 2).
- Order status may revert from "Shipped" to "Processing" if no submitted DNs remain.
- Fulfillment counts are recalculated.
- Buyer is notified of the cancellation.

### 10.4 Concurrent Delivery Note Creation

**Scenario**: Two warehouse operators simultaneously create Delivery Notes for the same order item.

**Handling**:
- Server-side validation on submit checks real-time shipped qty.
- If both try to ship the remaining balance, the second submission will fail validation.
- Use `frappe.db.sql` with fresh query (not cached) to get current shipped qty.
- Consider `frappe.db.get_value` with `for_update=True` for locking in high-concurrency environments.

```python
# In validate_delivery_note, use fresh DB query
already_shipped = _get_delivered_qty_excluding(
    marketplace_order_name,
    mo_item_name,
    exclude_dn=doc.name if not doc.is_new() else None
)
# This always queries the database, not cache
```

### 10.5 Multi-Item Partial Shipment

**Scenario**: Order has 3 items. Shipment 1 ships Item A fully, Item B partially, Item C not at all.

**Handling**:
- Each Delivery Note can contain any subset of order items.
- Items with `remaining_qty = 0` are excluded from the next `make_delivery_note()` call.
- `condition` function in `make_delivery_note()` filters items with no remaining qty.
- The order `fulfillment_status` reflects the mixed state: "Partially Shipped".

### 10.6 Return After Partial Shipment

**Scenario**: 600 of 1000 units shipped and delivered. Buyer returns 100 units. Then seller ships remaining 400.

**Handling**:
- Return updates `returned_qty` on the order item (200 -> 100 returned).
- `remaining_qty` calculation is `qty - shipped_qty`, NOT `qty - delivered_qty + returned_qty`.
- The seller should still ship the remaining 400 (based on original order qty minus shipped).
- `net_delivered_qty = delivered_qty - returned_qty = 600 - 100 = 500` after return.
- After final shipment: `shipped_qty = 1000`, `delivered_qty = 1000`, `returned_qty = 100`, `net = 900`.

### 10.7 Amendment to Submitted Delivery Note

**Scenario**: Delivery Note was submitted with wrong qty (typed 600 instead of 500).

**Handling**:
- ERPNext's standard amend flow creates a new version (DN-00001-1).
- The original DN is cancelled (docstatus = 2, excluded from calculations).
- The amended DN has the corrected qty.
- All balance calculations use `docstatus = 1` filter, automatically handling amendments.

### 10.8 Order with Single Item, Single Shipment (Non-Partial)

**Scenario**: Simple order with 1 item, fully shipped in one go.

**Handling**:
- `custom_is_partial_shipment = 0`, `custom_shipment_number = 1`, `custom_total_shipments = 1`.
- `remaining_qty = 0` after the single Delivery Note.
- Status flow: Processing -> Shipped -> Delivered -> Completed (normal flow).
- No partial shipment UI complexity shown.

### 12.3 Batch Operations

For orders with many items (50+ line items), use batch queries:

```python
def _get_all_delivered_qtys(order_name):
    """
    Get shipped quantities for ALL items in an order in a single query.

    More efficient than calling _get_delivered_qty() per item.
    """
    results = frappe.db.sql("""
        SELECT
            dni.custom_marketplace_order_item as item_name,
            SUM(dni.qty) as shipped_qty
        FROM `tabDelivery Note Item` dni
        JOIN `tabDelivery Note` dn ON dn.name = dni.parent
        WHERE dn.custom_marketplace_order = %(order_name)s
        AND dn.docstatus = 1
        GROUP BY dni.custom_marketplace_order_item
    """, {"order_name": order_name}, as_dict=True)

    return {r.item_name: flt(r.shipped_qty) for r in results}
```

# Task 135 - Buyer KPI Dashboard System Plan

## 1. Overview

This document defines the **Buyer KPI Dashboard System** for the TradeHub marketplace platform. The system provides an interactive, multi-context dashboard that visualizes auto-calculated buyer KPI scores with five chart types. It introduces the `Buyer KPI Score Log` DocType for historical tracking and defines scheduled recalculation jobs.

Key capabilities:

- **Buyer KPI Score Log DocType**: A historical record storing per-buyer, per-template KPI evaluation results with overall score and per-metric scores for trend analysis.
- **Buyer KPI Metric Score child DocType**: A child table capturing individual metric scores per evaluation for drill-down analysis.
- **Five chart types**: Radar/Spider, Gauge/Speedometer, Bar Chart, Line Chart, and Heatmap for comprehensive data visualization.
- **Three display contexts**: Buyer profile (full detail), Seller panel (anonymous summary), and Admin dashboard (aggregate analytics).
- **Data source mapping**: Structured data collection from `Sales Order`, `Payment Entry`, `Return Request`, and `Dispute` DocTypes.
- **Scheduled recalculation**: Configurable daily/weekly job for automatic KPI score computation with locking and error handling.

### Design Principles

1. **Score Log as Immutable Record**: Each `Buyer KPI Score Log` entry is a point-in-time snapshot. Once finalized, scores are not retroactively modified. This ensures audit integrity and reliable trend analysis.
2. **Template-Driven Evaluation**: The dashboard renders data according to the metrics defined in the associated `Buyer KPI Template` (from `133.md`). If the template changes, new evaluations use the new template; historical logs retain the template version at calculation time.
3. **Context-Appropriate Display**: Each display context (buyer, seller, admin) shows a tailored view. Buyers see full detail; sellers see anonymous aggregates; admins see platform-wide analytics.
4. **Performance First**: Chart data is pre-computed and stored in the Score Log, not calculated on-the-fly. Dashboard rendering reads from pre-aggregated records.
5. **Metric Naming Consistency**: All metric codes align exactly with the 9 metrics from the default `Buyer KPI Template` in `133.md` and the naming contract from `119.md` (Section 8.3).

## 4. Business Logic & Formulas

### 4.1 Score Log Creation Flow

When a buyer KPI score is calculated (either by scheduled job or manual trigger), the system creates a `Buyer KPI Score Log` entry:

```python
def create_buyer_kpi_score_log(buyer_name, template_name=None):
    """
    Calculate buyer KPI scores and create a Score Log entry.

    Args:
        buyer_name: Buyer Profile name.
        template_name: Buyer KPI Template name. If None, resolves using
                       template assignment logic from 133.md (Section 4.6).

    Returns:
        Name of the created Buyer KPI Score Log.
    """
    # Step 1: Resolve template (customer group > buyer level > default)
    template = resolve_buyer_kpi_template(buyer_name, template_name)
    profile = frappe.get_doc("Buyer Profile", buyer_name)

    # Step 2: Check eligibility
    if profile.total_orders < template.min_orders_required:
        return None  # Insufficient orders
    if days_since(profile.joined_at) < template.min_days_active:
        return None  # Insufficient activity

    # Step 3: Calculate scores using 133.md scoring logic
    result = calculate_buyer_kpi_score(buyer_name, template.name)

    if result.get("status") != "calculated":
        return None

    # Step 4: Fetch previous score for trend calculation
    previous_log = frappe.db.get_value(
        "Buyer KPI Score Log",
        {
            "buyer": buyer_name,
            "template": template.name,
            "status": "Finalized"
        },
        ["name", "overall_score", "calculation_date"],
        order_by="calculation_date DESC",
        as_dict=True
    )

    previous_score = previous_log.overall_score if previous_log else 0
    score_change = result["overall_score"] - previous_score

    # Step 5: Determine trend
    if score_change > 2.0:
        trend = "Rising"
    elif score_change < -2.0:
        trend = "Declining"
    else:
        trend = "Stable"

    # Step 6: Calculate percentile rank
    percentile = calculate_buyer_percentile(
        result["overall_score"], template.name
    )

    # Step 7: Get current level and grade
    buyer_level = frappe.db.get_value(
        "Buyer Profile", buyer_name, "buyer_level"
    )
    buyer_grade = frappe.db.get_value(
        "Customer Grade",
        {"buyer": buyer_name, "status": "Active"},
        "grade",
        order_by="calculation_date DESC"
    )

    # Step 8: Create Score Log
    log = frappe.new_doc("Buyer KPI Score Log")
    log.buyer = buyer_name
    log.template = template.name
    log.template_version = template.version
    log.score_type = "Periodic"
    log.score_period = get_current_period(template.evaluation_period)
    log.status = "Finalized"
    log.calculation_date = frappe.utils.today()
    log.evaluation_period_start = get_period_start(template.evaluation_period)
    log.evaluation_period_end = get_period_end(template.evaluation_period)

    # Overall score
    log.overall_score = result["overall_score"]
    log.previous_score = previous_score
    log.score_change = score_change
    log.score_trend = trend
    log.percentile_rank = percentile
    log.level_at_calculation = buyer_level
    log.grade_at_calculation = buyer_grade or ""
    log.passing_status = (
        "Passed" if result["overall_score"] >= template.passing_score
        else "Failed"
    )

    # Per-metric summary scores
    scores = result["per_metric_scores"]
    log.total_spend_score = scores.get("TOTAL_SPEND", 0)
    log.order_freq_score = scores.get("ORDER_FREQ", 0)
    log.payment_score = scores.get("PAYMENT_ON_TIME", 0)
    log.return_rate_score = scores.get("BUYER_RETURN_RATE", 0)
    log.avg_order_val_score = scores.get("AVG_ORDER_VAL", 0)
    log.dispute_rate_score = scores.get("BUYER_DISPUTE_RATE", 0)
    log.feedback_rate_score = scores.get("FEEDBACK_RATE", 0)
    log.account_age_score = scores.get("ACCOUNT_AGE", 0)
    log.last_activity_score = scores.get("LAST_ACTIVITY", 0)

    # Raw metric values
    raw = result["raw_values"]
    log.raw_total_spend = raw.get("TOTAL_SPEND", 0)
    log.raw_order_frequency = raw.get("ORDER_FREQ", 0)
    log.raw_payment_on_time = raw.get("PAYMENT_ON_TIME", 0)
    log.raw_return_rate = raw.get("BUYER_RETURN_RATE", 0)
    log.raw_avg_order_value = raw.get("AVG_ORDER_VAL", 0)
    log.raw_dispute_rate = raw.get("BUYER_DISPUTE_RATE", 0)
    log.raw_feedback_rate = raw.get("FEEDBACK_RATE", 0)
    log.raw_account_age = raw.get("ACCOUNT_AGE", 0)
    log.raw_last_activity = raw.get("LAST_ACTIVITY", 0)

    # Scoring curve applied
    log.scoring_curve_applied = template.scoring_curve

    # Applied weights (snapshot from template at calculation time)
    for metric in template.metrics:
        if not metric.is_active:
            continue
        weight_field_map = {
            "TOTAL_SPEND": "total_spend_weight",
            "ORDER_FREQ": "order_freq_weight",
            "PAYMENT_ON_TIME": "payment_weight",
            "BUYER_RETURN_RATE": "return_rate_weight",
            "AVG_ORDER_VAL": "avg_order_val_weight",
            "BUYER_DISPUTE_RATE": "dispute_rate_weight",
            "FEEDBACK_RATE": "feedback_rate_weight",
            "ACCOUNT_AGE": "account_age_weight",
            "LAST_ACTIVITY": "last_activity_weight",
        }
        field = weight_field_map.get(metric.metric_code)
        if field:
            log.set(field, metric.weight_pct)

    # Per-metric child table entries
    for metric in template.metrics:
        if not metric.is_active:
            continue
        code = metric.metric_code
        raw_val = raw.get(code, 0)
        norm_score = scores.get(code, 0)
        weighted = norm_score * (metric.weight_pct / 100.0)

        # Determine threshold level
        threshold_level = determine_threshold_level(
            raw_val, metric
        )

        # Get previous raw value
        prev_raw = get_previous_metric_raw_value(
            buyer_name, template.name, code
        )

        log.append("metric_scores", {
            "metric_code": code,
            "metric_name": metric.metric_name,
            "metric_type": metric.metric_type,
            "raw_value": raw_val,
            "normalized_score": norm_score,
            "weighted_score": round(weighted, 2),
            "weight_pct": metric.weight_pct,
            "threshold_level": threshold_level,
            "target_value": metric.threshold_excellent,
            "threshold_type": metric.threshold_type,
            "previous_raw_value": prev_raw,
            "value_change": round(raw_val - (prev_raw or 0), 4),
            "unit": metric.unit,
        })

    # Evaluation metadata
    log.orders_evaluated = count_orders_in_period(
        buyer_name,
        log.evaluation_period_start,
        log.evaluation_period_end
    )
    log.payments_evaluated = count_payments_in_period(
        buyer_name,
        log.evaluation_period_start,
        log.evaluation_period_end
    )
    log.returns_evaluated = count_returns_in_period(
        buyer_name,
        log.evaluation_period_start,
        log.evaluation_period_end
    )
    log.disputes_evaluated = count_disputes_in_period(
        buyer_name,
        log.evaluation_period_start,
        log.evaluation_period_end
    )
    log.reviews_evaluated = count_reviews_in_period(
        buyer_name,
        log.evaluation_period_start,
        log.evaluation_period_end
    )

    # Data quality assessment
    if log.orders_evaluated >= template.min_orders_required:
        log.data_quality_flag = "Sufficient"
    elif log.orders_evaluated >= template.min_orders_required // 2:
        log.data_quality_flag = "Limited"
    else:
        log.data_quality_flag = "Insufficient"

    log.created_at = frappe.utils.now()
    log.created_by = frappe.session.user
    log.finalized_at = frappe.utils.now()
    log.finalized_by = frappe.session.user

    log.insert(ignore_permissions=True)
    frappe.db.commit()

    return log.name


def determine_threshold_level(raw_value, metric):
    """
    Determine which threshold level a raw value falls into.

    Returns: "Poor", "Fair", "Good", or "Excellent"
    """
    higher_is_better = (metric.threshold_type == "Higher is Better")

    if higher_is_better:
        if raw_value >= (metric.threshold_excellent or 0):
            return "Excellent"
        elif raw_value >= (metric.threshold_good or 0):
            return "Good"
        elif raw_value >= (metric.threshold_fair or 0):
            return "Fair"
        else:
            return "Poor"
    else:
        # Lower is better: poor > fair > good > excellent
        if raw_value <= (metric.threshold_excellent or 0):
            return "Excellent"
        elif raw_value <= (metric.threshold_good or 0):
            return "Good"
        elif raw_value <= (metric.threshold_fair or 0):
            return "Fair"
        else:
            return "Poor"


def calculate_buyer_percentile(overall_score, template_name):
    """
    Calculate the buyer's percentile rank among all finalized scores
    for the same template.
    """
    total_count = frappe.db.count(
        "Buyer KPI Score Log",
        {"template": template_name, "status": "Finalized"}
    )
    if total_count == 0:
        return 50.0  # Default to median for first evaluation

    below_count = frappe.db.count(
        "Buyer KPI Score Log",
        {
            "template": template_name,
            "status": "Finalized",
            "overall_score": ("<", overall_score)
        }
    )

    return round((below_count / total_count) * 100, 1)
```

### 5.1 Buyer KPI Score Recalculation Job

**Job Name**: `tradehub_core.tradehub_core.tasks.calculate_buyer_kpi_scores`
**Default Frequency**: Weekly (configurable per template via `evaluation_frequency`)
**Cron Expression**: `0 7 * * 0` (Sunday 07:00 UTC)

This job runs after the seller score calculation (Sun 04:00 from `132.md`) and buyer grade calculation (Sun 05:00 from `129.md`) to ensure all prerequisite data is current.


#### Job Implementation

```python
def calculate_buyer_kpi_scores():
    """
    Scheduled job: Calculate buyer KPI scores for all eligible buyers.

    Runs weekly by default; frequency per template is configurable.
    Uses advisory locking to prevent concurrent execution.
    """
    lock_key = "buyer_kpi_score_calculation"

    # Prevent concurrent execution
    if frappe.cache().get_value(lock_key):
        frappe.logger().warning(
            "Buyer KPI score calculation already in progress. Skipping."
        )
        return

    try:
        frappe.cache().set_value(lock_key, True, expires_in_sec=3600)

        # Get all active templates
        templates = frappe.get_all(
            "Buyer KPI Template",
            filters={"status": "Active", "auto_evaluate": 1},
            fields=["name", "evaluation_frequency", "min_orders_required",
                     "min_days_active"]
        )

        for template in templates:
            # Check if this template should be evaluated this run
            if not should_evaluate_template(template):
                continue

            process_template_evaluations(template)

    except Exception:
        frappe.log_error(
            frappe.get_traceback(),
            "Buyer KPI Score Calculation Failed"
        )
    finally:
        frappe.cache().delete_value(lock_key)


def should_evaluate_template(template):
    """
    Check if a template should be evaluated in this run based on
    its evaluation_frequency setting.

    Weekly job checks: Daily templates run every time, Weekly run every
    time, Monthly templates run on first Sunday of month.
    """
    freq = template.evaluation_frequency
    today = frappe.utils.today()

    if freq == "Daily":
        return True
    elif freq == "Weekly":
        return True
    elif freq == "Monthly":
        # Run on first week of month
        day = frappe.utils.getdate(today).day
        return day <= 7
    elif freq == "On Demand":
        return False  # Never auto-evaluated

    return True


def process_template_evaluations(template):
    """
    Evaluate all eligible buyers for a given template.
    Processes in batches to manage memory and DB connections.
    """
    batch_size = 100
    offset = 0

    while True:
        buyers = frappe.get_all(
            "Buyer Profile",
            filters={
                "status": "Active",
                "total_orders": [">=", template.min_orders_required]
            },
            fields=["name"],
            limit_page_length=batch_size,
            limit_start=offset,
            order_by="name ASC"
        )

        if not buyers:
            break

        for buyer in buyers:
            try:
                create_buyer_kpi_score_log(
                    buyer.name, template.name
                )
            except Exception:
                frappe.log_error(
                    frappe.get_traceback(),
                    f"KPI Score Error: {buyer.name}"
                )

        frappe.db.commit()
        offset += batch_size

    frappe.logger().info(
        f"Buyer KPI score calculation completed for template: "
        f"{template.name}"
    )
```

```python
def aggregate_buyer_kpi_summaries():
    """
    Pre-aggregate buyer KPI score summaries for dashboard display.

    Calculates:
    - Score distribution (0-20, 20-40, 40-60, 60-80, 80-100)
    - Top 10 risers (biggest positive score_change)
    - Top 10 fallers (biggest negative score_change)
    - Average score by buyer level
    - Passing rate percentage
    """
    try:
        # Get latest finalized scores per buyer
        latest_scores = frappe.db.sql("""
            SELECT bksl.buyer, bksl.overall_score, bksl.score_change,
                   bksl.score_trend, bksl.level_at_calculation,
                   bksl.passing_status, bksl.template,
                   bp.buyer_name
            FROM `tabBuyer KPI Score Log` bksl
            INNER JOIN (
                SELECT buyer, MAX(calculation_date) as max_date
                FROM `tabBuyer KPI Score Log`
                WHERE status = 'Finalized'
                GROUP BY buyer
            ) latest ON bksl.buyer = latest.buyer
                AND bksl.calculation_date = latest.max_date
            LEFT JOIN `tabBuyer Profile` bp ON bksl.buyer = bp.name
            WHERE bksl.status = 'Finalized'
        """, as_dict=True)

        summary = {
            "distribution": {
                "0-20": 0, "20-40": 0, "40-60": 0,
                "60-80": 0, "80-100": 0
            },
            "top_risers": [],
            "top_fallers": [],
            "avg_by_level": {},
            "total_evaluated": len(latest_scores),
            "passing_count": 0,
            "calculated_at": frappe.utils.now(),
        }

        for score in latest_scores:
            # Distribution
            s = score.overall_score
            if s < 20:
                summary["distribution"]["0-20"] += 1
            elif s < 40:
                summary["distribution"]["20-40"] += 1
            elif s < 60:
                summary["distribution"]["40-60"] += 1
            elif s < 80:
                summary["distribution"]["60-80"] += 1
            else:
                summary["distribution"]["80-100"] += 1

            # Passing count
            if score.passing_status == "Passed":
                summary["passing_count"] += 1

            # By level
            level = score.level_at_calculation or "Unassigned"
            if level not in summary["avg_by_level"]:
                summary["avg_by_level"][level] = {
                    "total_score": 0, "count": 0
                }
            summary["avg_by_level"][level]["total_score"] += s
            summary["avg_by_level"][level]["count"] += 1

        # Calculate averages
        for level, data in summary["avg_by_level"].items():
            data["average"] = round(
                data["total_score"] / max(data["count"], 1), 2
            )

        # Top risers (sorted by score_change descending)
        sorted_by_change = sorted(
            latest_scores,
            key=lambda x: x.score_change or 0,
            reverse=True
        )
        summary["top_risers"] = [
            {
                "buyer": s.buyer,
                "buyer_name": s.buyer_name,
                "score": s.overall_score,
                "change": s.score_change
            }
            for s in sorted_by_change[:10]
            if (s.score_change or 0) > 0
        ]

        # Top fallers (sorted by score_change ascending)
        summary["top_fallers"] = [
            {
                "buyer": s.buyer,
                "buyer_name": s.buyer_name,
                "score": s.overall_score,
                "change": s.score_change
            }
            for s in sorted_by_change[-10:]
            if (s.score_change or 0) < 0
        ][::-1]  # Reverse to show biggest decline first

        # Store in cache for dashboard use
        frappe.cache().set_value(
            "buyer_kpi_dashboard_summary",
            summary,
            expires_in_sec=86400  # 24 hours
        )

        frappe.logger().info(
            f"Buyer KPI summary aggregated: "
            f"{summary['total_evaluated']} buyers evaluated"
        )

    except Exception:
        frappe.log_error(
            frappe.get_traceback(),
            "Buyer KPI Summary Aggregation Failed"
        )
```

### 5.5 Error Handling & Recovery

| Scenario | Handling |
|----------|----------|
| Job already running | Advisory lock prevents concurrent execution; logged as warning |
| Single buyer fails | Logged to Error Log; other buyers continue processing |
| Database connection lost | Transaction rolled back at batch boundary; next run resumes |
| Template not found | Skipped with warning log |
| Insufficient data | Score Log created with `data_quality_flag = "Limited"` or skipped if below minimum |
| Timeout (>1 hour) | Lock expires automatically; next scheduled run proceeds normally |

```python
@frappe.whitelist()
def get_buyer_activity_heatmap(buyer_name, months=12):
    """
    Get daily order counts for heatmap rendering.

    Args:
        buyer_name: Buyer Profile name.
        months: Number of months to look back (default 12).

    Returns:
        List of {date, count} dicts.
    """
    start_date = frappe.utils.add_months(
        frappe.utils.today(), -months
    )

    daily_counts = frappe.db.sql("""
        SELECT
            DATE(order_date) as date,
            COUNT(*) as count
        FROM `tabMarketplace Order`
        WHERE buyer = %(buyer)s
          AND order_date >= %(start_date)s
          AND status NOT IN ('Cancelled', 'Draft')
        GROUP BY DATE(order_date)
        ORDER BY date ASC
    """, {"buyer": buyer_name, "start_date": start_date}, as_dict=True)

    return daily_counts
```

```python
        summary = compute_admin_dashboard_summary(template_name)

    # Add aggregate trend (always live query for freshness)
    summary["aggregate_trend"] = get_aggregate_kpi_trend(
        template_name, months=12
    )

    # Add per-metric performance averages
    summary["metric_performance"] = get_metric_performance_averages(
        template_name
    )

    return summary


def get_aggregate_kpi_trend(template_name=None, months=12):
    """
    Get monthly average overall_score across all buyers.
    """
    start_date = frappe.utils.add_months(
        frappe.utils.today(), -months
    )

    filters = {"status": "Finalized", "calculation_date": [">=", start_date]}
    if template_name:
        filters["template"] = template_name

    monthly_avg = frappe.db.sql("""
        SELECT
            score_period,
            AVG(overall_score) as avg_score,
            COUNT(*) as buyer_count,
            AVG(percentile_rank) as avg_percentile
        FROM `tabBuyer KPI Score Log`
        WHERE status = 'Finalized'
          AND calculation_date >= %(start_date)s
          {template_filter}
        GROUP BY score_period
        ORDER BY score_period ASC
    """.format(
        template_filter=(
            "AND template = %(template)s" if template_name else ""
        )
    ), {
        "start_date": start_date,
        "template": template_name
    }, as_dict=True)

    return monthly_avg


def get_metric_performance_averages(template_name=None):
    """
    Get average normalized score per metric across all buyers
    (latest evaluation only).
    """
    # Use latest score per buyer
    result = frappe.db.sql("""
        SELECT
            AVG(total_spend_score) as avg_total_spend,
            AVG(order_freq_score) as avg_order_freq,
            AVG(payment_score) as avg_payment,
            AVG(return_rate_score) as avg_return_rate,
            AVG(avg_order_val_score) as avg_order_val,
            AVG(dispute_rate_score) as avg_dispute_rate,
            AVG(feedback_rate_score) as avg_feedback_rate,
            AVG(account_age_score) as avg_account_age,
            AVG(last_activity_score) as avg_last_activity
        FROM `tabBuyer KPI Score Log` bksl
        INNER JOIN (
            SELECT buyer, MAX(calculation_date) as max_date
            FROM `tabBuyer KPI Score Log`
            WHERE status = 'Finalized'
            GROUP BY buyer
        ) latest ON bksl.buyer = latest.buyer
            AND bksl.calculation_date = latest.max_date
        WHERE bksl.status = 'Finalized'
    """, as_dict=True)

    if not result:
        return {}

    return {
        "TOTAL_SPEND": round(result[0].avg_total_spend or 0, 2),
        "ORDER_FREQ": round(result[0].avg_order_freq or 0, 2),
        "PAYMENT_ON_TIME": round(result[0].avg_payment or 0, 2),
        "BUYER_RETURN_RATE": round(result[0].avg_return_rate or 0, 2),
        "AVG_ORDER_VAL": round(result[0].avg_order_val or 0, 2),
        "BUYER_DISPUTE_RATE": round(result[0].avg_dispute_rate or 0, 2),
        "FEEDBACK_RATE": round(result[0].avg_feedback_rate or 0, 2),
        "ACCOUNT_AGE": round(result[0].avg_account_age or 0, 2),
        "LAST_ACTIVITY": round(result[0].avg_last_activity or 0, 2),
    }
```

### 9.2 Migration Script

A data migration runs during `bench migrate` to seed initial Score Log entries for existing buyers with sufficient order history:

```python
# tradehub_core/patches/create_initial_buyer_kpi_score_logs.py

def execute():
    """
    Create initial Buyer KPI Score Log entries for existing active
    buyers who meet the default template's minimum requirements.

    This runs once during initial deployment to populate historical data.
    """
    template = frappe.db.get_value(
        "Buyer KPI Template",
        {"is_default": 1, "status": "Active"},
        "name"
    )

    if not template:
        frappe.logger().warning(
            "No default Buyer KPI Template found. "
            "Run 133.md migration first."
        )
        return

    template_doc = frappe.get_doc("Buyer KPI Template", template)

    eligible_buyers = frappe.get_all(
        "Buyer Profile",
        filters={
            "status": "Active",
            "total_orders": [">=", template_doc.min_orders_required]
        },
        fields=["name"],
        limit_page_length=0
    )

    created = 0
    skipped = 0

    for buyer in eligible_buyers:
        try:
            # Check if score log already exists
            existing = frappe.db.exists(
                "Buyer KPI Score Log",
                {"buyer": buyer.name, "template": template}
            )
            if existing:
                skipped += 1
                continue

            log_name = create_buyer_kpi_score_log(
                buyer.name, template
            )
            if log_name:
                created += 1
        except Exception:
            frappe.log_error(
                frappe.get_traceback(),
                f"Initial KPI Score Error: {buyer.name}"
            )

    frappe.db.commit()

    frappe.logger().info(
        f"Initial buyer KPI score logs: "
        f"{created} created, {skipped} skipped (already existed)"
    )
```

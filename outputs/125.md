# 125 - Marketplace Order Actions / Create Button Plan

## Overview

This document defines the complete **Create dropdown** action system for the Marketplace Order DocType in TradeHub. It covers all document-creation actions available from a submitted Marketplace Order, including conditional visibility rules per order status, `@frappe.whitelist()` method signatures with `get_mapped_doc` field mappings, client-side JS invocation via `frappe.model.open_mapped_doc`, dashboard connections, and linked-document back-link structure.

**Reference Codebase:**
- `marketplace_order.py` - Existing whitelist patterns, status transitions, ERPNext sync utilities
- `marketplace_order.js` - Existing button patterns (`frm.add_custom_button`), tenant isolation filters
- `tradehub_core/utils/erpnext_sync.py` - Existing `get_mapped_doc` usage (e.g., `make_item_from_sku`)
- `marketplace_order.json` - Existing ERPNext link fields (`erpnext_sales_order`, `erpnext_delivery_note`, `erpnext_sales_invoice`, `erpnext_payment_entry`)

**Cross-References:**
- **147.md** (Seller Panel Order Documents) - Extends this plan with seller-panel permissions and ownership validation
- **126.md** (Partial Shipment Tracking) - Delivery Note creation defined here is the foundation for partial shipment
- **123.md** (Seller-Grouped Checkout) - Order structure feeds into this action system
- **117.md** (Payment Terms) - Payment Entry/Request integrate with payment terms logic

## 2. Create Dropdown Options

### 2.1 Complete Create Button List

The following actions appear in the **Create** dropdown button group on the Marketplace Order form. Each action creates a linked downstream document.

| # | Create Action | Target DocType | Conditional Visibility | Status Auto-Advance |
|---|---------------|----------------|------------------------|---------------------|
| 1 | Sales Invoice | Sales Invoice | `docstatus == 1` AND `status` in `[Confirmed, Invoiced, Processing, Packed]` AND no existing linked Sales Invoice | `Confirmed` -> `Invoiced` |
| 2 | Delivery Note | Delivery Note | `docstatus == 1` AND `status` in `[Invoiced, Processing, Packed]` AND Sales Invoice exists | No auto-advance (see 126.md for partial shipment status) |
| 3 | Payment Entry | Payment Entry | `docstatus == 1` AND `status` in `[Confirmed, Invoiced, Await Payment, Payment Received]` AND `payment_status` != `Paid` | Updates `payment_status` |
| 4 | Payment Request | Payment Request | `docstatus == 1` AND `status` in `[Confirmed, Invoiced, Await Payment]` AND `payment_status` in `[Pending, Partially Paid]` | No auto-advance |
| 5 | Shipment | Shipment | `docstatus == 1` AND `status` in `[Processing, Packed]` AND Delivery Note exists | `Processing/Packed` -> `Shipped` |
| 6 | Return / Credit Note | Sales Invoice (is_return=1) | `docstatus == 1` AND `status` in `[Delivered, Completed]` AND original Sales Invoice exists | `Delivered/Completed` -> triggers refund flow |
| 7 | Dispute | Marketplace Dispute | `docstatus == 1` AND `status` in `[Delivered, Completed, Shipped, In Transit]` | `status` -> `Disputed` |

### 2.2 Conditional Visibility Rules Detail

#### 2.2.1 Sales Invoice

**When Visible:**
- Order is submitted (`docstatus === 1`)
- Status is one of: `Confirmed`, `Invoiced` (for correction), `Processing`, `Packed`
- No Sales Invoice is already linked (`!frm.doc.erpnext_sales_invoice`)
- Payment status is not `Cancelled` or `Refunded`

**When Hidden:**
- Draft or cancelled orders
- Sales Invoice already exists (show "View Sales Invoice" link instead)
- Status is `Delivered`, `Completed`, `Refunded`, or `Disputed`

```javascript
// Visibility check
let can_create_si = frm.doc.docstatus === 1
    && ['Confirmed', 'Processing', 'Packed'].includes(frm.doc.status)
    && !frm.doc.erpnext_sales_invoice
    && !['Cancelled', 'Refunded'].includes(frm.doc.payment_status);
```

#### 2.2.2 Delivery Note

**When Visible:**
- Order is submitted (`docstatus === 1`)
- Status is one of: `Invoiced`, `Processing`, `Packed`
- Sales Invoice exists (invoice-first workflow)
- Not fully delivered (`fulfillment_status` != `Delivered`)
- At least one item has remaining qty to deliver

**When Hidden:**
- No Sales Invoice created yet
- All items fully delivered
- Order is cancelled, refunded, or on hold

```javascript
let can_create_dn = frm.doc.docstatus === 1
    && ['Invoiced', 'Processing', 'Packed'].includes(frm.doc.status)
    && frm.doc.erpnext_sales_invoice
    && !frm.doc.fully_fulfilled;
```

#### 2.2.3 Payment Entry

**When Visible:**
- Order is submitted
- Payment is not fully received (`payment_status` in `[Pending, Await Verification, Partially Paid]`)
- Status allows payment (`Confirmed`, `Invoiced`, `Await Payment`, `Payment Received`)

**When Hidden:**
- Already fully paid
- Order is cancelled or refunded

```javascript
let can_create_pe = frm.doc.docstatus === 1
    && ['Pending', 'Await Verification', 'Partially Paid'].includes(frm.doc.payment_status)
    && !['Cancelled', 'Refunded'].includes(frm.doc.status);
```

#### 2.2.4 Payment Request

**When Visible:**
- Order is submitted
- Payment status is `Pending` or `Partially Paid`
- Status is `Confirmed`, `Invoiced`, or `Await Payment`
- No active (unsettled) Payment Request exists

```javascript
let can_create_pr = frm.doc.docstatus === 1
    && ['Pending', 'Partially Paid'].includes(frm.doc.payment_status)
    && ['Confirmed', 'Invoiced', 'Await Payment'].includes(frm.doc.status);
```

#### 2.2.5 Shipment

**When Visible:**
- Order is submitted
- Status is `Processing` or `Packed`
- At least one Delivery Note exists and is submitted
- Not all items shipped yet

```javascript
let can_create_shipment = frm.doc.docstatus === 1
    && ['Processing', 'Packed'].includes(frm.doc.status)
    && frm.doc.erpnext_delivery_note;
```

#### 2.2.6 Return / Credit Note

**When Visible:**
- Order is submitted
- Status is `Delivered` or `Completed`
- Original Sales Invoice exists and is submitted
- Return window has not expired (configurable, default 14 days)

```javascript
let can_create_return = frm.doc.docstatus === 1
    && ['Delivered', 'Completed'].includes(frm.doc.status)
    && frm.doc.erpnext_sales_invoice;
```

#### 2.2.7 Dispute

**When Visible:**
- Order is submitted
- Status is `Delivered`, `Completed`, `Shipped`, or `In Transit`
- No active dispute exists for this order

```javascript
let can_create_dispute = frm.doc.docstatus === 1
    && ['Delivered', 'Completed', 'Shipped', 'In Transit'].includes(frm.doc.status);
```

### 3.1 make_sales_invoice

```python
@frappe.whitelist()
def make_sales_invoice(source_name, target_doc=None):
    """
    Create a Sales Invoice from a Marketplace Order.

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Sales Invoice to map onto

    Returns:
        Document: Mapped Sales Invoice document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Status must be in [Confirmed, Processing, Packed]
        - No existing Sales Invoice linked

    Status Effect:
        After Sales Invoice is submitted: order status -> "Invoiced"
    """
    from frappe.model.mapper import get_mapped_doc

    def set_missing_values(source, target):
        """Set computed and default values on the Sales Invoice."""
        target.flags.ignore_permissions = True

        # Set invoice type based on order
        if source.requires_invoice and source.invoice_type == "E-Fatura":
            target.custom_e_invoice_enabled = 1

        # Set customer from ERPNext link or create
        if source.erpnext_customer:
            target.customer = source.erpnext_customer
        else:
            target.customer = _get_or_create_customer(source)

        # Set due date from Payment Terms if linked (ref: 117.md)
        if source.payment_terms_template:
            target.payment_terms_template = source.payment_terms_template

        # Set posting date
        target.posting_date = frappe.utils.nowdate()
        target.due_date = frappe.utils.add_days(
            frappe.utils.nowdate(),
            frappe.db.get_single_value("Selling Settings", "payment_days") or 30
        )

        # Set currency
        target.currency = source.currency or "TRY"

        # Set taxes
        target.taxes_and_charges = _get_default_tax_template(source)

        # Link back to marketplace order
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        # Calculate totals
        target.run_method("set_missing_values")
        target.run_method("calculate_taxes_and_totals")

    def update_item(source_doc, target_doc, source_parent):
        """Map individual order items to invoice items."""
        # Get or create ERPNext Item from listing
        target_doc.item_code = _get_erpnext_item_code(source_doc)
        target_doc.item_name = source_doc.title
        target_doc.description = source_doc.title
        target_doc.qty = source_doc.qty
        target_doc.rate = source_doc.unit_price
        target_doc.uom = source_doc.stock_uom or "Nos"
        target_doc.stock_uom = source_doc.stock_uom or "Nos"
        target_doc.conversion_factor = 1

    def postprocess(source, target):
        """After mapping is complete."""
        target.set_posting_time = 1

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Sales Invoice",
                "field_map": {
                    "name": "custom_marketplace_order",
                    "order_id": "custom_marketplace_order_id",
                    "buyer_name": "customer_name",
                    "buyer_email": "contact_email",
                    "buyer_phone": "contact_mobile",
                    "shipping_address_name": "shipping_address_name",
                    "billing_address_name": "customer_address",
                    "currency": "currency",
                    "grand_total": "grand_total",
                    "buyer_tax_id": "tax_id",
                    "buyer_company_name": "company_name",
                },
                "validation": {
                    "docstatus": ["=", 1]
                }
            },
            "Marketplace Order Item": {
                "doctype": "Sales Invoice Item",
                "field_map": {
                    "listing": "custom_listing",
                    "title": "item_name",
                    "sku": "custom_sku",
                    "qty": "qty",
                    "unit_price": "rate",
                    "stock_uom": "uom",
                    "tax_rate": "custom_tax_rate",
                    "discount_value": "discount_percentage",
                    "line_total": "amount",
                },
                "postprocess": update_item,
            }
        },
        target_doc,
        set_missing_values,
        postprocess
    )
```

### 3.2 make_delivery_note

```python
@frappe.whitelist()
def make_delivery_note(source_name, target_doc=None):
    """
    Create a Delivery Note from a Marketplace Order.

    Supports partial shipment: only items with remaining qty > 0
    are included. See 126.md for balance tracking details.

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Delivery Note to map onto

    Returns:
        Document: Mapped Delivery Note document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Status must be in [Invoiced, Processing, Packed]
        - Sales Invoice must exist
        - At least one item must have remaining qty > 0

    Status Effect:
        After Delivery Note is submitted: fulfillment status updates
        via update_fulfillment_counts()
    """
    from frappe.model.mapper import get_mapped_doc

    def set_missing_values(source, target):
        """Set computed and default values on the Delivery Note."""
        target.flags.ignore_permissions = True

        # Set customer
        if source.erpnext_customer:
            target.customer = source.erpnext_customer
        else:
            target.customer = _get_or_create_customer(source)

        # Set posting date
        target.posting_date = frappe.utils.nowdate()

        # Set currency
        target.currency = source.currency or "TRY"

        # Link to Sales Invoice
        if source.erpnext_sales_invoice:
            target.custom_against_sales_invoice = source.erpnext_sales_invoice

        # Link back to marketplace order
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        # Set shipping address
        target.shipping_address_name = source.shipping_address_name
        target.shipping_address = _format_address(source)

        # Set warehouse
        target.set_warehouse = _get_default_warehouse()

        target.run_method("set_missing_values")
        target.run_method("calculate_taxes_and_totals")

    def condition(source_doc):
        """Only include items with remaining qty to deliver."""
        # Calculate already-delivered qty for this item
        delivered_qty = _get_delivered_qty(
            source_doc.parent,
            source_doc.name
        )
        remaining_qty = flt(source_doc.qty) - flt(delivered_qty)
        return remaining_qty > 0

    def update_item(source_doc, target_doc, source_parent):
        """Map individual order items to delivery note items."""
        target_doc.item_code = _get_erpnext_item_code(source_doc)
        target_doc.item_name = source_doc.title
        target_doc.description = source_doc.title
        target_doc.uom = source_doc.stock_uom or "Nos"
        target_doc.stock_uom = source_doc.stock_uom or "Nos"
        target_doc.conversion_factor = 1

        # Set qty to remaining qty (partial shipment support)
        delivered_qty = _get_delivered_qty(
            source_doc.parent,
            source_doc.name
        )
        target_doc.qty = flt(source_doc.qty) - flt(delivered_qty)
        target_doc.rate = source_doc.unit_price

        # Store reference to original order item
        target_doc.custom_marketplace_order_item = source_doc.name

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Delivery Note",
                "field_map": {
                    "name": "custom_marketplace_order",
                    "order_id": "custom_marketplace_order_id",
                    "buyer_name": "customer_name",
                    "shipping_address_name": "shipping_address_name",
                    "currency": "currency",
                },
                "validation": {
                    "docstatus": ["=", 1]
                }
            },
            "Marketplace Order Item": {
                "doctype": "Delivery Note Item",
                "field_map": {
                    "listing": "custom_listing",
                    "title": "item_name",
                    "sku": "custom_sku",
                    "qty": "qty",
                    "unit_price": "rate",
                    "stock_uom": "uom",
                },
                "condition": condition,
                "postprocess": update_item,
            }
        },
        target_doc,
        set_missing_values
    )
```

```python
@frappe.whitelist()
def make_payment_entry(source_name, target_doc=None):
    """
    Create a Payment Entry from a Marketplace Order.

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Payment Entry to map onto

    Returns:
        Document: Mapped Payment Entry document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Payment status must not be 'Paid' or 'Refunded'
        - Status must not be 'Cancelled'

    Status Effect:
        After Payment Entry is submitted:
        - payment_status updates (Paid / Partially Paid)
        - If auto_confirm: status -> "Confirmed"
    """
    from frappe.model.mapper import get_mapped_doc

    order = frappe.get_doc("Marketplace Order", source_name)

    if order.payment_status == "Paid":
        frappe.throw(_("Payment has already been received for this order"))

    outstanding_amount = flt(order.grand_total) - flt(order.paid_amount)

    def set_missing_values(source, target):
        """Set computed values on the Payment Entry."""
        target.payment_type = "Receive"
        target.party_type = "Customer"
        target.party = source.erpnext_customer or _get_or_create_customer(source)
        target.party_name = source.buyer_name
        target.paid_amount = outstanding_amount
        target.received_amount = outstanding_amount
        target.source_exchange_rate = 1
        target.target_exchange_rate = 1
        target.posting_date = frappe.utils.nowdate()

        # Set mode of payment from order
        if source.payment_method:
            target.mode_of_payment = _map_payment_method(source.payment_method)

        # Set reference to Sales Invoice if exists
        if source.erpnext_sales_invoice:
            target.append("references", {
                "reference_doctype": "Sales Invoice",
                "reference_name": source.erpnext_sales_invoice,
                "allocated_amount": outstanding_amount,
            })

        # Link back to marketplace order
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        # Set paid_from and paid_to accounts
        target.paid_to = _get_default_receivable_account()
        target.paid_from = _get_default_payment_account(source.payment_method)

        target.setup_party_account_field()
        target.set_missing_values()

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Payment Entry",
                "field_map": {
                    "name": "custom_marketplace_order",
                    "order_id": "custom_marketplace_order_id",
                    "grand_total": "paid_amount",
                    "currency": "paid_from_account_currency",
                },
                "validation": {
                    "docstatus": ["=", 1]
                }
            }
        },
        target_doc,
        set_missing_values
    )
```

```python
@frappe.whitelist()
def make_payment_request(source_name, target_doc=None):
    """
    Create a Payment Request from a Marketplace Order.

    Sends a payment link to the buyer via email or generates
    a payment URL for online payment.

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Payment Request to map onto

    Returns:
        Document: Mapped Payment Request document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Payment status must be 'Pending' or 'Partially Paid'
        - No active (unsettled) Payment Request for this order

    Status Effect:
        No immediate status change. Payment Request triggers
        payment flow; status updates via Payment Entry callback.
    """
    from frappe.model.mapper import get_mapped_doc

    order = frappe.get_doc("Marketplace Order", source_name)

    # Check for existing active payment requests
    existing = frappe.db.exists("Payment Request", {
        "reference_doctype": "Marketplace Order",
        "reference_name": source_name,
        "status": ["in", ["Initiated", "Requested"]],
        "docstatus": 1
    })
    if existing:
        frappe.throw(
            _("An active Payment Request {0} already exists for this order").format(existing)
        )

    outstanding_amount = flt(order.grand_total) - flt(order.paid_amount)

    def set_missing_values(source, target):
        """Set computed values on the Payment Request."""
        target.payment_request_type = "Inward"
        target.party_type = "Customer"
        target.party = source.erpnext_customer or _get_or_create_customer(source)
        target.email_to = source.buyer_email
        target.grand_total = outstanding_amount
        target.currency = source.currency or "TRY"
        target.posting_date = frappe.utils.nowdate()

        # Reference link
        target.reference_doctype = "Sales Invoice" if source.erpnext_sales_invoice else "Marketplace Order"
        target.reference_name = source.erpnext_sales_invoice or source.name

        # Link back
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        # Set payment gateway if configured
        payment_gateway = frappe.db.get_single_value(
            "TradeHub Commerce Settings", "default_payment_gateway"
        )
        if payment_gateway:
            target.payment_gateway = payment_gateway

        target.run_method("set_missing_values")

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Payment Request",
                "field_map": {
                    "name": "custom_marketplace_order",
                    "order_id": "custom_marketplace_order_id",
                    "grand_total": "grand_total",
                    "currency": "currency",
                    "buyer_email": "email_to",
                },
                "validation": {
                    "docstatus": ["=", 1]
                }
            }
        },
        target_doc,
        set_missing_values
    )
```

```python
@frappe.whitelist()
def make_shipment(source_name, target_doc=None):
    """
    Create a Shipment record from a Marketplace Order.

    Uses the tradehub_logistics Shipment DocType (not ERPNext's).

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Shipment to map onto

    Returns:
        Document: Mapped Shipment document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Status must be in [Processing, Packed]
        - At least one Delivery Note must exist

    Status Effect:
        After Shipment is saved with tracking info:
        - Order status -> "Shipped"
        - Fulfillment status updates via update_fulfillment_counts()
    """
    from frappe.model.mapper import get_mapped_doc

    order = frappe.get_doc("Marketplace Order", source_name)

    if order.status not in ["Processing", "Packed"]:
        frappe.throw(
            _("Shipment can only be created for orders in Processing or Packed status")

        """Set computed values on the Shipment."""
        target.shipment_type = "Outgoing"
        target.pickup_date = frappe.utils.nowdate()

        # Set origin from warehouse/company address
        target.pickup_company = frappe.defaults.get_defaults().get("company")

        target.delivery_contact_name = source.buyer_name
        target.delivery_address_line1 = source.shipping_address_line1
        target.delivery_address_line2 = source.shipping_address_line2
        target.delivery_city = source.shipping_city
        target.delivery_state = source.shipping_state
        target.delivery_pincode = source.shipping_postal_code
        target.delivery_country = source.shipping_country
        target.delivery_contact_phone = source.shipping_phone or source.buyer_phone

        # Link to delivery note
        if source.erpnext_delivery_note:
            target.custom_delivery_note = source.erpnext_delivery_note

        # Link back to marketplace order
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        # Set package weight from order
        target.total_weight = source.total_weight or 0

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Shipment",
```

```python
@frappe.whitelist()
def make_return_credit_note(source_name, target_doc=None):
    """
    Create a Return (Credit Note) Sales Invoice from a Marketplace Order.

    Creates a Sales Invoice with is_return=1 and negative quantities,
    referencing the original Sales Invoice.

    Args:
        source_name (str): Marketplace Order name
        target_doc (Document, optional): Existing Sales Invoice to map onto

    Returns:
        Document: Mapped Credit Note Sales Invoice document (unsaved)

    Conditions:
        - Source order must be submitted (docstatus == 1)
        - Status must be in [Delivered, Completed]
        - Original Sales Invoice must exist and be submitted
        - Return window must not be expired (default 14 days)

    Status Effect:
        After Credit Note is submitted:
        - Triggers refund flow (refund_requested = 1)
        - If full return: status -> "Refunded"
        - If partial return: payment_status -> "Partially Refunded"
    """
    from frappe.model.mapper import get_mapped_doc

    order = frappe.get_doc("Marketplace Order", source_name)

    if not order.erpnext_sales_invoice:
        frappe.throw(_("Original Sales Invoice is required to create a Return"))

    # Validate return window
    if order.delivered_at:
        days_since_delivery = frappe.utils.date_diff(
            frappe.utils.nowdate(),
            frappe.utils.getdate(order.delivered_at)
        )
        return_window = frappe.db.get_single_value(
            "TradeHub Commerce Settings", "return_window_days"
        ) or 14
        if days_since_delivery > return_window:
            frappe.throw(
                _("Return window of {0} days has expired. Delivered on {1}.").format(
                    return_window, order.delivered_at
                )
            )

    def set_missing_values(source, target):
        """Set values for the credit note."""
        target.is_return = 1
        target.return_against = source.erpnext_sales_invoice
        target.posting_date = frappe.utils.nowdate()
        target.update_stock = 1  # Return items to stock

        # Set customer
        if source.erpnext_customer:
            target.customer = source.erpnext_customer

        # Link back
        target.custom_marketplace_order = source.name
        target.custom_marketplace_order_id = source.order_id

        target.run_method("set_missing_values")
        target.run_method("calculate_taxes_and_totals")

    def update_item(source_doc, target_doc, source_parent):
        """Set negative quantities for return items."""
        target_doc.item_code = _get_erpnext_item_code(source_doc)
        target_doc.item_name = source_doc.title
        target_doc.qty = -1 * flt(source_doc.qty)  # Negative for return
        target_doc.rate = source_doc.unit_price
        target_doc.uom = source_doc.stock_uom or "Nos"
        target_doc.stock_uom = source_doc.stock_uom or "Nos"
        target_doc.conversion_factor = 1

    return get_mapped_doc(
        "Marketplace Order",
        source_name,
        {
            "Marketplace Order": {
                "doctype": "Sales Invoice",
                "field_map": {
                    "name": "custom_marketplace_order",
                    "order_id": "custom_marketplace_order_id",
                    "erpnext_sales_invoice": "return_against",
                    "currency": "currency",
                },
                "validation": {
                    "docstatus": ["=", 1]
                }
            },
            "Marketplace Order Item": {
                "doctype": "Sales Invoice Item",
                "field_map": {
                    "listing": "custom_listing",
                    "title": "item_name",
                    "sku": "custom_sku",
                    "qty": "qty",
                    "unit_price": "rate",
                    "stock_uom": "uom",
                },
                "postprocess": update_item,
            }
        },
        target_doc,
        set_missing_values
    )
```

```python
def _get_or_create_customer(order):
    """
    Get or create ERPNext Customer from Marketplace Order buyer.

    Delegates to the existing method on MarketplaceOrder class
    or uses centralized sync utility.

    Args:
        order: Marketplace Order document

    Returns:
        str: Customer name/ID
    """
    if order.erpnext_customer:
        return order.erpnext_customer

    # Use the existing method on the document
    customer = order.get_or_create_erpnext_customer()
    if customer:
        order.db_set("erpnext_customer", customer)
    return customer


def _get_erpnext_item_code(order_item):
    """
    Get ERPNext Item Code for a Marketplace Order Item.

    Looks up the linked Listing -> SKU Product -> ERPNext Item mapping.
    If no ERPNext Item exists, creates one via sync utility.

    Args:
        order_item: Marketplace Order Item row

    Returns:
        str: ERPNext Item Code
    """
    if order_item.erpnext_item_code:
        return order_item.erpnext_item_code

    # Try to get from listing's SKU
    listing = frappe.get_doc("Listing", order_item.listing)
    if listing.sku_product:
        sku = frappe.get_doc("SKU Product", listing.sku_product)
        if sku.erpnext_item_code:
            return sku.erpnext_item_code

    # Fallback: use listing title as item_code
    return order_item.sku or order_item.listing

def _get_delivered_qty(order_name, order_item_name):
    """
    Get total qty already delivered for an order item.

    Sums qty from all submitted Delivery Note Items
    that reference this Marketplace Order Item.

    Args:
        order_name: Marketplace Order name
        order_item_name: Marketplace Order Item name (child row)

    Returns:
        float: Total delivered qty
    """
    result = frappe.db.sql("""
        SELECT IFNULL(SUM(dni.qty), 0) as delivered_qty
        FROM `tabDelivery Note Item` dni
        JOIN `tabDelivery Note` dn ON dn.name = dni.parent
        WHERE dn.custom_marketplace_order = %(order_name)s
        AND dni.custom_marketplace_order_item = %(order_item_name)s
        AND dn.docstatus = 1
    """, {
        "order_name": order_name,
        "order_item_name": order_item_name
    }, as_dict=True)

    return flt(result[0].delivered_qty) if result else 0


def _format_address(order):
    """
    Format shipping address as a display string.

    Args:
        order: Marketplace Order document

    Returns:
        str: Formatted address string
    """
    parts = [
        order.shipping_address_line1,
        order.shipping_address_line2,
        order.shipping_city,
        order.shipping_state,
        order.shipping_postal_code,
        order.shipping_country,
    ]
    return "\n".join(filter(None, parts))
```

```python
# Post-creation callback methods in marketplace_order_actions.py

def on_sales_invoice_submit(doc, method):
    """
    When a Sales Invoice linked to a Marketplace Order is submitted:
    1. Update order's erpnext_sales_invoice field
    2. Advance order status to 'Invoiced' (if currently 'Confirmed')
    """
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order
    order = frappe.get_doc("Marketplace Order", order_name)

    # Set link
    order.db_set("erpnext_sales_invoice", doc.name)

    # Advance status
    if order.status == "Confirmed":
        order.db_set("status", "Invoiced")

    order.clear_order_cache()


def on_sales_invoice_cancel(doc, method):
    """When a linked Sales Invoice is cancelled, clear the link."""
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order

    # Only clear if this is the currently-linked invoice
    current_link = frappe.db.get_value(
        "Marketplace Order", order_name, "erpnext_sales_invoice"
    )
    if current_link == doc.name:
        frappe.db.set_value("Marketplace Order", order_name, "erpnext_sales_invoice", None)

        # Revert status if it was Invoiced
        order_status = frappe.db.get_value("Marketplace Order", order_name, "status")
        if order_status == "Invoiced":
            frappe.db.set_value("Marketplace Order", order_name, "status", "Confirmed")


def on_delivery_note_submit(doc, method):
    """
    When a Delivery Note linked to a Marketplace Order is submitted:
    1. Update order's erpnext_delivery_note field (latest DN)
    2. Update fulfillment counts via update_fulfillment_counts()
    3. Auto-advance status if applicable
    """
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order
    order = frappe.get_doc("Marketplace Order", order_name)

    # Set link (to latest DN)
    order.db_set("erpnext_delivery_note", doc.name)

    # Update fulfillment tracking
    order.update_fulfillment_counts()

    order.clear_order_cache()


def on_delivery_note_cancel(doc, method):
    """When a linked Delivery Note is cancelled, update fulfillment counts."""
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order
    order = frappe.get_doc("Marketplace Order", order_name)

    # Recalculate fulfillment (the cancelled DN won't count)
    order.update_fulfillment_counts()

    # Update DN link to previous DN (if any)
    previous_dn = frappe.db.get_value(
        "Delivery Note",
        {
            "custom_marketplace_order": order_name,
            "docstatus": 1,
            "name": ["!=", doc.name]
        },
        "name",
        order_by="creation desc"
    )
    order.db_set("erpnext_delivery_note", previous_dn)
    order.clear_order_cache()


def on_payment_entry_submit(doc, method):
    """
    When a Payment Entry linked to a Marketplace Order is submitted:
    1. Update order's erpnext_payment_entry field
    2. Record payment on the order (updates payment_status)
    3. Auto-confirm if applicable
    """
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order
    order = frappe.get_doc("Marketplace Order", order_name)

    # Set link
    order.db_set("erpnext_payment_entry", doc.name)

    # Record payment
    order.record_payment(
        amount=doc.paid_amount,
        payment_reference=doc.name,
        payment_method=doc.mode_of_payment
    )

    order.clear_order_cache()


def on_payment_entry_cancel(doc, method):
    """When a linked Payment Entry is cancelled, adjust payment status."""
    if not doc.custom_marketplace_order:
        return

    order_name = doc.custom_marketplace_order
    order = frappe.get_doc("Marketplace Order", order_name)

    # Recalculate paid amount from remaining valid Payment Entries
    total_paid = frappe.db.sql("""
        SELECT IFNULL(SUM(paid_amount), 0) as total
        FROM `tabPayment Entry`
        WHERE custom_marketplace_order = %(order)s
        AND docstatus = 1
        AND name != %(cancelled)s
    """, {"order": order_name, "cancelled": doc.name}, as_dict=True)

    new_paid = flt(total_paid[0].total) if total_paid else 0
    order.db_set("paid_amount", new_paid)

    # Update payment status
    if new_paid <= 0:
        order.db_set("payment_status", "Pending")
    elif new_paid < flt(order.grand_total):
        order.db_set("payment_status", "Partially Paid")

    # Update PE link
    latest_pe = frappe.db.get_value(
        "Payment Entry",
        {
            "custom_marketplace_order": order_name,
            "docstatus": 1,
            "name": ["!=", doc.name]
        },
        "name",
        order_by="creation desc"
    )
    order.db_set("erpnext_payment_entry", latest_pe)
    order.clear_order_cache()
```

## 8. Concurrency & Edge Cases

### 8.1 Concurrent Create Actions

**Problem:** Two users attempt to create the same document type (e.g., Sales Invoice) simultaneously for the same order.

**Solution:**
1. Use `frappe.lock_doc("Marketplace Order", source_name)` at the start of each `make_*` method
2. Re-check the link field before proceeding (double-check pattern)
3. Release lock after the target document is saved

```python
@frappe.whitelist()
def make_sales_invoice(source_name, target_doc=None):
    # Acquire lock
    frappe.lock_doc("Marketplace Order", source_name)

    try:
        # Re-check: no invoice created between button click and execution
        current_si = frappe.db.get_value(
            "Marketplace Order", source_name, "erpnext_sales_invoice"
        )
        if current_si:
            frappe.throw(
                _("Sales Invoice {0} has already been created for this order").format(current_si)
            )

        # ... proceed with get_mapped_doc ...
    finally:
        frappe.unlock_doc("Marketplace Order", source_name)
```

### 8.2 Multiple Delivery Notes (Partial Shipment)

The `erpnext_delivery_note` field on Marketplace Order stores only the **latest** Delivery Note. For full tracking of all partial shipments, query all Delivery Notes linked via `custom_marketplace_order`.

See **126.md** for complete partial shipment balance tracking.

### 8.3 Return After Partial Delivery

If only some items were delivered and a return is initiated:
- The Credit Note should only include delivered items
- The `make_return_credit_note` method should filter items by `fulfillment_status == "Delivered"`
- Partial returns create a Credit Note with only the returned item quantities

### 8.4 Payment Entry for Partial Amounts

Multiple Payment Entries can exist for a single order (partial payments). The `paid_amount` field on the order accumulates across all submitted Payment Entries. The `payment_status` reflects the aggregate state.

### 8.5 Dispute While Shipment In Transit

If a dispute is filed while the order is in `Shipped` or `In Transit` status:
- The order status changes to `Disputed`
- The shipment continues (cannot be recalled in transit)
- Resolution may result in: `Completed` (dispute rejected) or `Refunded` (dispute accepted)

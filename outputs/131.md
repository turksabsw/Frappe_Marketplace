## 2. VKN Validation Algorithm (10-Digit Checksum)

### Overview

The **Vergi Kimlik Numarasi** (VKN / Tax Identification Number) is a 10-digit number assigned by the Turkish Revenue Administration (GIB - Gelir Idaresi Baskanligi) to every taxpayer entity. The 10th digit is a check digit computed from the first 9 digits.

### Algorithm

Given VKN digits `d[0] d[1] d[2] d[3] d[4] d[5] d[6] d[7] d[8] d[9]`:

**Step 1** - For each digit position `i` (0 through 8), compute an intermediate value:

```
v[i] = (d[i] + (9 - i)) mod 10
```

**Step 2** - For each `v[i]`, apply the power-modulus transformation:

```
if v[i] == 0:
    c[i] = 0
else:
    c[i] = (v[i] * 2^(9-i)) mod 9
    if c[i] == 0:
        c[i] = 9
```

**Step 3** - Sum all contributions and compute the check digit:

```
total = c[0] + c[1] + c[2] + ... + c[8]
check_digit = (10 - (total mod 10)) mod 10
```

**Step 4** - Validate: `d[9]` must equal `check_digit`

### Constraints

| Rule | Condition |
|------|-----------|
| Length | Exactly 10 digits |
| Characters | Digits only (0-9) |
| All zeros | `0000000000` is invalid |
| Check digit | Must satisfy the checksum algorithm |

### Python Implementation

```python
def validate_vkn(vkn: str) -> dict:
    """
    Validate a Turkish VKN (Vergi Kimlik Numarasi).

    Returns:
        dict: {"valid": bool, "error": str or None}
    """
    if not vkn or not isinstance(vkn, str):
        return {"valid": False, "error": "VKN is required"}

    vkn = vkn.strip()

    if len(vkn) != 10:
        return {"valid": False, "error": "VKN must be exactly 10 digits"}

    if not vkn.isdigit():
        return {"valid": False, "error": "VKN must contain only digits"}

    if vkn == "0000000000":
        return {"valid": False, "error": "VKN cannot be all zeros"}

    digits = [int(c) for c in vkn]
    total = 0

    for i in range(9):
        v = (digits[i] + (9 - i)) % 10
        if v != 0:
            v = (v * pow(2, 9 - i)) % 9
            if v == 0:
                v = 9
        total += v

    check = (10 - (total % 10)) % 10

    if digits[9] != check:
        return {"valid": False, "error": "VKN checksum validation failed"}

    return {"valid": True, "error": None}
```

### JavaScript Implementation (Client-Side)

```javascript
function validate_vkn(vkn) {
    if (!vkn || typeof vkn !== 'string') {
        return {valid: false, error: __('VKN is required')};
    }

    vkn = vkn.trim();

    if (vkn.length !== 10) {
        return {valid: false, error: __('VKN must be exactly 10 digits')};
    }

    if (!/^\d{10}$/.test(vkn)) {
        return {valid: false, error: __('VKN must contain only digits')};
    }

    if (vkn === '0000000000') {
        return {valid: false, error: __('VKN cannot be all zeros')};
    }

    var digits = vkn.split('').map(Number);
    var total = 0;

    for (var i = 0; i < 9; i++) {
        var v = (digits[i] + (9 - i)) % 10;
        if (v !== 0) {
            v = (v * Math.pow(2, 9 - i)) % 9;
            if (v === 0) {
                v = 9;
            }
        }
        total += v;
    }

    var check = (10 - (total % 10)) % 10;

    if (digits[9] !== check) {
        return {valid: false, error: __('VKN checksum validation failed')};
    }

    return {valid: true, error: null};
}
```

### Worked Example

VKN: `1234567890` (example - check digit computation):

| i | d[i] | (d[i] + 9-i) % 10 = v | v != 0? | (v * 2^(9-i)) % 9 | c[i] |
|---|------|------------------------|---------|---------------------|------|
| 0 | 1 | (1 + 9) % 10 = 0 | No | - | 0 |
| 1 | 2 | (2 + 8) % 10 = 0 | No | - | 0 |
| 2 | 3 | (3 + 7) % 10 = 0 | No | - | 0 |
| 3 | 4 | (4 + 6) % 10 = 0 | No | - | 0 |
| 4 | 5 | (5 + 5) % 10 = 0 | No | - | 0 |
| 5 | 6 | (6 + 4) % 10 = 0 | No | - | 0 |
| 6 | 7 | (7 + 3) % 10 = 0 | No | - | 0 |
| 7 | 8 | (8 + 2) % 10 = 0 | No | - | 0 |
| 8 | 9 | (9 + 1) % 10 = 0 | No | - | 0 |

total = 0, check = (10 - 0) % 10 = 0. So `1234567890` has check digit 0.

## 3. TC Kimlik No Validation Algorithm (11-Digit Checksum)

### Overview

The **TC Kimlik Numarasi** (TCKN / Turkish Citizenship Number) is an 11-digit national identification number assigned to every Turkish citizen. The first digit cannot be zero. Digits 10 and 11 are check digits computed from the preceding digits.

### Algorithm

Given TCKN digits `d[1] d[2] d[3] d[4] d[5] d[6] d[7] d[8] d[9] d[10] d[11]` (1-indexed):

**Rule 1** - First digit is non-zero: `d[1] != 0`

**Rule 2** - 10th digit check (parity check):

```
odd_sum  = d[1] + d[3] + d[5] + d[7] + d[9]
even_sum = d[2] + d[4] + d[6] + d[8]
d[10] = ((odd_sum * 7) - even_sum) mod 10
```

> **Note:** If the modulus result is negative, add 10 to get the positive remainder. In most programming languages, `((odd_sum * 7) - even_sum) % 10` handles this correctly when using a language with non-negative modulo (Python). In JavaScript, use `(((odd_sum * 7) - even_sum) % 10 + 10) % 10` to ensure a positive result.

**Rule 3** - 11th digit check (total sum check):

```
d[11] = (d[1] + d[2] + d[3] + d[4] + d[5] + d[6] + d[7] + d[8] + d[9] + d[10]) mod 10
```

### Constraints

| Rule | Condition |
|------|-----------|
| Length | Exactly 11 digits |
| Characters | Digits only (0-9) |
| First digit | Must not be 0 |
| All zeros | `00000000000` is invalid |
| 10th digit | Must satisfy parity check |
| 11th digit | Must satisfy total sum check |

### Python Implementation

```python
def validate_tckn(tckn: str) -> dict:
    """
    Validate a Turkish TCKN (TC Kimlik Numarasi).

    Returns:
        dict: {"valid": bool, "error": str or None}
    """
    if not tckn or not isinstance(tckn, str):
        return {"valid": False, "error": "TCKN is required"}

    tckn = tckn.strip()

    if len(tckn) != 11:
        return {"valid": False, "error": "TCKN must be exactly 11 digits"}

    if not tckn.isdigit():
        return {"valid": False, "error": "TCKN must contain only digits"}

    if tckn[0] == "0":
        return {"valid": False, "error": "TCKN cannot start with zero"}

    if tckn == "00000000000":
        return {"valid": False, "error": "TCKN cannot be all zeros"}

    d = [int(c) for c in tckn]

    # Rule 2: 10th digit check (d[9] in 0-indexed)
    odd_sum = d[0] + d[2] + d[4] + d[6] + d[8]
    even_sum = d[1] + d[3] + d[5] + d[7]
    d10_expected = ((odd_sum * 7) - even_sum) % 10

    if d[9] != d10_expected:
        return {"valid": False, "error": "TCKN 10th digit checksum failed"}

    # Rule 3: 11th digit check (d[10] in 0-indexed)
    first_10_sum = sum(d[:10])
    d11_expected = first_10_sum % 10

    if d[10] != d11_expected:
        return {"valid": False, "error": "TCKN 11th digit checksum failed"}

    return {"valid": True, "error": None}
```

```javascript
function validate_tckn(tckn) {
    if (!tckn || typeof tckn !== 'string') {
        return {valid: false, error: __('TCKN is required')};
    }

    tckn = tckn.trim();

    if (tckn.length !== 11) {
        return {valid: false, error: __('TCKN must be exactly 11 digits')};
    }

    if (!/^\d{11}$/.test(tckn)) {
        return {valid: false, error: __('TCKN must contain only digits')};
    }

    if (tckn[0] === '0') {
        return {valid: false, error: __('TCKN cannot start with zero')};
    }

    var d = tckn.split('').map(Number);

    // Rule 2: 10th digit check
    var odd_sum = d[0] + d[2] + d[4] + d[6] + d[8];
    var even_sum = d[1] + d[3] + d[5] + d[7];
    var d10_expected = (((odd_sum * 7) - even_sum) % 10 + 10) % 10;

    if (d[9] !== d10_expected) {
        return {valid: false, error: __('TCKN checksum validation failed')};
    }

    // Rule 3: 11th digit check
    var first_10_sum = 0;
    for (var i = 0; i < 10; i++) {
        first_10_sum += d[i];
    }
    var d11_expected = first_10_sum % 10;

    if (d[10] !== d11_expected) {
        return {valid: false, error: __('TCKN checksum validation failed')};
    }

    return {valid: true, error: null};
}
```

### Worked Example

TCKN: `10000000146` (known valid test TCKN from existing test suite):

| Position | d[i] | Value |
|----------|-------|-------|
| 1 | d[0] | 1 |
| 2 | d[1] | 0 |
| 3 | d[2] | 0 |
| 4 | d[3] | 0 |
| 5 | d[4] | 0 |
| 6 | d[5] | 0 |
| 7 | d[6] | 0 |
| 8 | d[7] | 0 |
| 9 | d[8] | 1 |
| 10 | d[9] | 4 (check digit 1) |
| 11 | d[10] | 6 (check digit 2) |

**Step 1 - 10th digit check:**

- odd_sum = d[0] + d[2] + d[4] + d[6] + d[8] = 1 + 0 + 0 + 0 + 1 = 2
- even_sum = d[1] + d[3] + d[5] + d[7] = 0 + 0 + 0 + 0 = 0
- d10_expected = ((2 * 7) - 0) % 10 = 14 % 10 = **4** ✓ (d[9] = 4)

**Step 2 - 11th digit check:**

- first_10_sum = 1 + 0 + 0 + 0 + 0 + 0 + 0 + 0 + 1 + 4 = 6
- d11_expected = 6 % 10 = **6** ✓ (d[10] = 6)

**Verification passes.**

## 4. GIB API Integration Research

### GIB (Gelir Idaresi Baskanligi) Services

The Turkish Revenue Administration provides web services for tax ID verification:

| Service | URL | Purpose |
|---------|-----|---------|
| VKN Verification | `https://ivd.gib.gov.tr/tvd_server/asil-erisim/sorgu` | Verify VKN against registered taxpayer name |
| TCKN Verification (NVI) | `https://tckimlik.nvi.gov.tr/Service/KPSPublic.asmx` | Verify TCKN against personal details (name, birth year) |

### GIB VKN Verification SOAP Request

```xml
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <vergiNoDogrula xmlns="http://gib.gov.tr/">
      <vergiNo>1234567890</vergiNo>
      <turu>tpiKurum</turu>
    </vergiNoDogrula>
  </soap:Body>
</soap:Envelope>
```

### NVI TCKN Verification SOAP Request

```xml
<soap:Envelope xmlns:soap="http://schemas.xmlsoap.org/soap/envelope/">
  <soap:Body>
    <TCKimlikNoDogrula xmlns="http://tckimlik.nvi.gov.tr/WS">
      <TCKimlikNo>10000000146</TCKimlikNo>
      <Ad>ALI</Ad>

      <DogumYili>1990</DogumYili>
    </TCKimlikNoDogrula>
  </soap:Body>

```

### Integration Feasibility Assessment

| Factor | VKN (GIB) | TCKN (NVI) |
|--------|-----------|------------|
| Public access | Restricted - may require institutional agreement | Public SOAP service |
| Rate limiting | Strict - potential IP blocking | Moderate rate limits |
| Response time | 1-5 seconds | 1-3 seconds |
| Availability | Business hours preferred | 24/7 with occasional maintenance |
| Data required | VKN + entity type | TCKN + first name + last name + birth year |
| KVKK concerns | Low (business data) | High (personal data - requires explicit consent) |
| Cost | Free (government service) | Free (government service) |
| Reliability | Intermittent connectivity issues reported | Generally reliable |

### Recommended Strategy: Tiered Validation

**Tier 1 - Format & Checksum Validation (Always Active)**
- Client-side: Immediate feedback on digit format and checksum
- Server-side: Re-validate on save (never trust client-only validation)
- No external dependencies, instant response

**Tier 2 - GIB/NVI API Verification (Optional, Configurable)**
- Enabled via `Identity Verification Settings` single DocType
- Async verification with retry logic (max 3 attempts)
- Cache successful verifications for 24 hours
- Fallback to Tier 1 if API unavailable

**Tier 3 - Manual Admin Review (Fallback)**
- Auto-created when Tier 2 fails or returns ambiguous results
- Admin reviews uploaded documents against submitted numbers
- Required for edge cases (new businesses, name mismatches)

### Fallback Flow

```
User submits VKN/TCKN
    ├── Tier 1: Format + Checksum
    │   ├── FAIL → Immediate rejection (client-side)
    │   └── PASS → Continue
    │
    ├── Tier 2: GIB/NVI API (if enabled)
    │   ├── VERIFIED → Auto-approve
    │   ├── MISMATCH → Create manual review case
    │   ├── API ERROR → Fallback to Tier 3
    │   └── TIMEOUT → Fallback to Tier 3
    │
    └── Tier 3: Manual Review
        ├── Admin reviews documents
        ├── APPROVE → Mark verified
        └── REJECT → Notify user with reason
```

```python
# In tradehub_core/utils/identity_verification.py

import frappe

def check_identity_verification(user, required_level="any"):
    """
    Check if user meets identity verification requirements.

    Args:
        user: User email/ID
        required_level: 'tckn', 'vkn', 'both', or 'any'

    Returns:
        dict: {"verified": bool, "level": str, "message": str}
    """
    requests = frappe.get_all(
        "Identity Verification Request",
        filters={
            "applicant_user": user,
            "status": "Approved",
            "is_expired": 0
        },
        fields=["verification_type"]
    )

    has_tckn = any(r.verification_type == "TCKN" for r in requests)
    has_vkn = any(r.verification_type == "VKN" for r in requests)

    if required_level == "tckn" and not has_tckn:
        return {
            "verified": False,
            "level": "none" if not has_vkn else "vkn_only",
            "message": "TCKN verification is required for this action"
        }
    elif required_level == "vkn" and not has_vkn:
        return {
            "verified": False,
            "level": "none" if not has_tckn else "tckn_only",
            "message": "VKN verification is required for this action"
        }
    elif required_level == "both" and not (has_tckn and has_vkn):
        return {
            "verified": False,
            "level": "partial",
            "message": "Both TCKN and VKN verification are required"
        }
    elif required_level == "any" and not (has_tckn or has_vkn):
        return {
            "verified": False,
            "level": "none",
            "message": "Identity verification is required for this action"
        }

    return {
        "verified": True,
        "level": "both" if (has_tckn and has_vkn) else ("tckn" if has_tckn else "vkn"),
        "message": None
    }


def enforce_verification(doc, method):
    """
    Before-save hook to enforce verification restrictions.
    Add to hooks.py for relevant DocTypes.
    """
    enforcement_rules = {
        "Listing": {"required_level": "vkn", "action": "create listings"},
        "Review": {"required_level": "tckn", "action": "submit reviews"},
        "RFQ": {"required_level": "any", "action": "create RFQ requests"},
    }

    doctype = doc.doctype
    if doctype not in enforcement_rules:
        return

    rule = enforcement_rules[doctype]
    user = doc.owner or frappe.session.user
    result = check_identity_verification(user, rule["required_level"])

    if not result["verified"]:
        frappe.throw(
            frappe._("Identity verification is required to {0}. {1}").format(
                rule["action"], result["message"]
            ),
            title=frappe._("Verification Required")
        )
```

```python
    month_start = frappe.utils.get_first_day(frappe.utils.today())
    order_count = frappe.db.count("Order", {
        "buyer_user": user,
        "creation": [">=", month_start]
    })

    if order_count >= 3:
        return {
            "allowed": False,
            "message": "Unverified users are limited to 3 orders per month. "
                       "Please verify your identity to continue."
        }

    # Check monthly spend
    monthly_spend = frappe.db.sql("""
        SELECT COALESCE(SUM(grand_total), 0)
        FROM `tabOrder`
        WHERE buyer_user = %s AND creation >= %s
    """, (user, month_start))[0][0]

    if monthly_spend >= 5000:
        return {
            "allowed": False,
            "message": "Unverified users are limited to 5,000 TRY per month. "
                       "Please verify your identity to continue."
        }

    return {"allowed": True, "message": None}
```

### Approval Callback Implementation

```python
# In identity_verification_request.py

class IdentityVerificationRequest(Document):

    def on_update_after_submit(self):
        """Handle post-approval/rejection actions."""
        if self.status == "Approved":
            self._on_approval()
        elif self.status == "Rejected":
            self._on_rejection()

    def _on_approval(self):
        """Propagate approval to linked profiles."""
        self.verified_at = frappe.utils.now_datetime()

        # Calculate expiry date
        settings = frappe.get_single("Identity Verification Settings")
        expiry_days = settings.verification_expiry_days or 365
        self.expires_at = frappe.utils.add_days(frappe.utils.today(), expiry_days)

        # Update KYC Profile
        if self.kyc_profile:
            kyc = frappe.get_doc("KYC Profile", self.kyc_profile)
            if self.verification_type == "TCKN":
                kyc.tckn = self.id_number
                kyc.tax_id_type = "TC Kimlik No"
            else:
                kyc.tax_id = self.id_number
                kyc.tax_id_type = "Vergi No"
            kyc.id_verified = 1
            kyc.verification_status = "Completed"
            kyc.verification_method = "Hybrid" if self.api_verified else "Manual"
            kyc.verified_at = self.verified_at
            kyc.verified_by = frappe.session.user
            kyc.flags.ignore_permissions = True
            kyc.save()

        # Update Seller Profile
        if self.seller_profile:
            seller = frappe.get_doc("Seller Profile", self.seller_profile)
            seller.tax_id = self.id_number
            seller.tax_id_type = self.verification_type
            seller.identity_verified = 1
            seller.verification_status = "Verified"
            seller.verified_at = self.verified_at
            seller.verified_by = frappe.session.user
            seller.flags.ignore_permissions = True
            seller.save()

        # Send approval notification
        frappe.sendmail(
            recipients=[self.applicant_user],
            subject=frappe._("Identity Verification Approved"),
            template="identity_verification_approved",
            args={
                "applicant_name": self.applicant_name,
                "verification_type": self.verification_type,
                "verified_at": self.verified_at
            }
        )

    def _on_rejection(self):
        """Handle rejection notification."""
        frappe.sendmail(
            recipients=[self.applicant_user],
            subject=frappe._("Identity Verification Update"),
            template="identity_verification_rejected",
            args={
                "applicant_name": self.applicant_name,
                "rejection_reason": self.rejection_reason_detail,
                "can_resubmit": self.can_resubmit,
                "resubmit_after": self.resubmit_after
            }
        )
```

## 11. Edge Cases

### 11.1 Duplicate Submissions (Same Number, Multiple Users)

**Scenario:** User A submits VKN `1234567890`. User B later submits the same VKN.

**Detection Logic:**

```python
def check_duplicate_id_number(doc):
    """Check if this identity number is already verified for another user."""
    existing = frappe.db.get_value(
        "Identity Verification Request",
        {
            "id_number": doc.id_number,
            "verification_type": doc.verification_type,
            "status": "Approved",
            "is_expired": 0,
            "applicant_user": ["!=", doc.applicant_user],
            "name": ["!=", doc.name]
        },
        ["name", "applicant_user"],
        as_dict=True
    )

    if existing:
        doc.is_duplicate = 1
        doc.duplicate_user = existing.applicant_user
        doc.duplicate_request = existing.name
        doc.priority = "High"  # Auto-escalate
        return True

    return False
```

**Resolution Options:**
1. **Reject with explanation** - The number is already verified for another account
2. **Escalate to admin** - Admin manually investigates potential fraud or legitimate transfer
3. **Allow with flag** - For organizations where multiple users may share a VKN (different representatives)

**Policy Decision:** For **VKN**, allow sharing within the same Organization (multiple Organization Members can reference the same VKN). For **TCKN**, enforce strict uniqueness (one TCKN per user account).

### 11.2 Expired Verification

**Scenario:** Verification was approved 365 days ago and has expired.

**Automated Expiry Process:**

```python
# Scheduled task: runs daily via hooks.py
# scheduler_events.daily = ["tradehub_core.tasks.check_verification_expiry"]

def check_verification_expiry():
    """Mark expired verifications and notify users."""
    expired_requests = frappe.get_all(
        "Identity Verification Request",
        filters={
            "status": "Approved",
            "is_expired": 0,
            "expires_at": ["<", frappe.utils.today()]
        },
        fields=["name", "applicant_user", "applicant_name", "verification_type",
                "seller_profile", "kyc_profile"]
    )

    for req in expired_requests:
        doc = frappe.get_doc("Identity Verification Request", req.name)
        doc.is_expired = 1
        doc.status = "Expired"
        doc.flags.ignore_permissions = True
        doc.save()

        # Downgrade linked profiles
        if req.seller_profile:
            frappe.db.set_value("Seller Profile", req.seller_profile,
                               "identity_verified", 0)

        if req.kyc_profile:
            frappe.db.set_value("KYC Profile", req.kyc_profile,
                               "verification_status", "Pending")

        # Notify user
        frappe.sendmail(
            recipients=[req.applicant_user],
            subject=frappe._("Identity Verification Expired"),
            template="identity_verification_expired",
            args={
                "applicant_name": req.applicant_name,
                "verification_type": req.verification_type,
                "resubmit_url": "/app/identity-verification-request/new"
            }
        )

    # Log summary
    if expired_requests:
        frappe.logger().info(
            f"Identity verification expiry: {len(expired_requests)} "
            f"verifications expired and users notified."
        )
```

**Advance Warning Notifications:**
- 30 days before expiry: Email notification with re-verification link
- 7 days before expiry: Urgent email + system notification
- On expiry day: Status downgraded, restrictions applied immediately

### 11.3 Re-verification After Rejection

**Scenario:** User submits incorrect documents, gets rejected, and wants to try again.

**Rules:**
- `can_resubmit` flag determines if resubmission is allowed (default: true)
- `resubmit_after` date enforces a cooling-off period (default: immediate for document issues, 30 days for fraud suspicion)
- `resubmission_count` tracks how many times the user has resubmitted
- Maximum 5 submissions total (configurable in Identity Verification Settings)
- After 5 failed attempts, admin intervention is required

**Resubmission Flow:**

```python
def create_resubmission(user, previous_request_name):
    """Create a new verification request as a resubmission."""
    previous = frappe.get_doc("Identity Verification Request", previous_request_name)

    if not previous.can_resubmit:
        frappe.throw(frappe._("Resubmission is not allowed for this request."))

    if previous.resubmit_after and frappe.utils.getdate() < frappe.utils.getdate(previous.resubmit_after):
        frappe.throw(frappe._("Resubmission is not allowed until {0}").format(
            previous.resubmit_after
        ))

    # Check max submissions
    settings = frappe.get_single("Identity Verification Settings")
    total_submissions = frappe.db.count("Identity Verification Request", {
        "applicant_user": user,
        "verification_type": previous.verification_type
    })

    if total_submissions >= (settings.max_submissions_per_user or 5):
        frappe.throw(frappe._(
            "Maximum submission limit reached. "
            "Please contact support for assistance."
        ))

    new_request = frappe.new_doc("Identity Verification Request")
    new_request.applicant_user = user
    new_request.verification_type = previous.verification_type
    new_request.id_number = previous.id_number
    new_request.first_name = previous.first_name
    new_request.last_name = previous.last_name
    new_request.birth_year = previous.birth_year
    new_request.kyc_profile = previous.kyc_profile
    new_request.seller_profile = previous.seller_profile
    new_request.previous_request = previous.name
    new_request.resubmission_count = (previous.resubmission_count or 0) + 1
    new_request.save()

    return new_request.name
```

### 11.4 Concurrent Submissions

**Scenario:** User submits TCKN verification from two browser tabs simultaneously.

**Prevention:**

```python
def validate(self):
    """Prevent concurrent pending requests for the same type."""
    if self.is_new() or self.status == "Draft":
        existing_pending = frappe.db.exists(
            "Identity Verification Request",
            {
                "applicant_user": self.applicant_user,
                "verification_type": self.verification_type,
                "status": ["in", ["Pending", "In Review"]],
                "name": ["!=", self.name]
            }
        )
        if existing_pending:
            frappe.throw(
                frappe._("You already have a pending {0} verification request ({1}). "
                         "Please wait for it to be processed.").format(
                    self.verification_type, existing_pending
                )
            )
```

### 11.5 Name Change After Verification

**Scenario:** User legally changes their name after TCKN verification.

**Handling:**
- User must submit a new verification request with updated documents
- Previous verification is revoked (status set to "Expired")
- Admin reviews new documents confirming name change
- New verification request linked to previous via `previous_request`

### 11.6 Organization VKN Transfer

**Scenario:** Seller's business changes legal entity, resulting in a new VKN.

**Handling:**
- Previous VKN verification is revoked
- New VKN verification request submitted
- Admin verifies trade registry gazette showing entity change
- Seller Profile updated with new VKN on approval
- Transaction history maintained (no data loss)

# Task 107 - Contract Automation Plan

## Executive Summary

TradeHub's `tradehub_compliance` app contains a well-designed contract management system comprising four DocTypes: **Contract Template** (versioned Jinja templates with SHA256 integrity), **Contract Instance** (party-linked contract records with full status workflow), **Contract Rule** (configurable trigger-based automation rules with conditions), and **Contract Revision** (immutable revision history). However, the automation layer is **entirely dormant** -- the `doc_events` hook in `tradehub_compliance/hooks.py` is commented out, meaning no contracts are automatically generated at any business trigger point. This plan details the activation of contract automation through 3 primary trigger candidates (Seller Application approval, Marketplace Order submission, Seller Profile creation), the data binding specifications for each, template variable resolution via Jinja, and the Jinja sanitization strategy for undefined variables.

## Current State Analysis

### DocType Architecture Overview

```
Contract Template (1)  ---->  Contract Instance (N)  ---->  Contract Revision (N)
     |                              |
     |--- template_code             |--- party_type (Dynamic Link)
     |--- content (Jinja)           |--- party (e.g., Seller Profile)
     |--- version (auto-incr)       |--- template_version_snapshot
     |--- content_hash (SHA256)     |--- template_content_snapshot
     |--- contract_type             |--- content_hash_snapshot
     |--- signature_method          |--- status workflow
     |--- valid_days                |--- signature_method / esign_provider
     |--- auto_expire               |--- signed_pdf
     |--- tenant / company          |--- tenant / company
                                    |
Contract Rule (N)  <-------->  Contract Template (1)
     |
     |--- trigger_point (Registration, First Order, etc.)
     |--- trigger_doctype (e.g., Seller Application)
     |--- trigger_event (on_submit, after_insert, etc.)
     |--- conditions (child table: Contract Rule Condition)
     |--- target filters (apply_to, user_type, seller_level)
     |--- blocking_action (Check)
     |--- notification settings
     |--- execution statistics
```

```python
def on_seller_application_update(doc, method):
    """Triggered when Seller Application is updated.

    Only fires contract automation when status transitions to 'Approved'.
    """
    if doc.status != "Approved":
        return

    # Check if status actually changed (not just a re-save)
    previous_status = doc.get_doc_before_save()
    if previous_status and previous_status.status == "Approved":
        return

    evaluate_contract_rules(
        trigger_point="Registration",
        trigger_doctype="Seller Application",
        source_doc=doc,
        party_type="Seller Profile",
        party=doc.seller_profile
    )


def on_order_submit(doc, method):
    """Triggered when Marketplace Order is submitted.

    Checks if buyer needs a first-order contract (Buyer Agreement).
    """
    # Check if buyer already has a signed Buyer Agreement
    existing = frappe.db.exists("Contract Instance", {
        "party_type": "User",
        "party": doc.buyer,
        "contract_type": "Buyer Agreement",
        "status": "Signed"
    })

    if existing:
        return

    evaluate_contract_rules(
        trigger_point="First Order",
        trigger_doctype="Marketplace Order",
        source_doc=doc,
        party_type="User",
        party=doc.buyer
    )
```

#### 2.2 Rule Evaluation Engine

```python
def evaluate_contract_rules(trigger_point, trigger_doctype, source_doc,
                            party_type, party):
    """Find and evaluate matching Contract Rules for the given trigger.

    Args:
        trigger_point: The trigger category (e.g., "Registration")
        trigger_doctype: The triggering DocType name
        source_doc: The document instance that triggered the event
        party_type: DocType name for the contract party
        party: Name of the contract party document
    """
    if not party:
        frappe.log_error(
            title=_("Contract Automation: Missing party"),
            message=_("Cannot create contract without a party. "
                      "Trigger: {0} on {1} ({2})").format(
                          trigger_point, trigger_doctype, source_doc.name)
        )
        return

    # Find active rules matching the trigger
    rules = frappe.get_all(
        "Contract Rule",
        filters={
            "status": "Active",
            "trigger_point": trigger_point,
            "trigger_doctype": trigger_doctype
        },
        fields=["name", "contract_template", "priority", "match_type",
                "blocking_action", "apply_to", "target_user_type",
                "target_seller_level", "target_buyer_level",
                "send_notification", "notification_template",
                "valid_from", "valid_until", "tenant"],
        order_by="priority DESC"
    )

    for rule_data in rules:
        rule = frappe.get_doc("Contract Rule", rule_data.name)

        # Check rule validity period
        if not is_rule_valid(rule):
            continue

        # Check tenant scope
        if rule.tenant and rule.tenant != getattr(source_doc, "tenant", None):
            continue

        # Check target filters
        if not matches_target_filters(rule, source_doc):
            continue

        # Evaluate conditions
        if not evaluate_conditions(rule, source_doc):
            continue

        # Check for existing unsigned contract (prevent duplicates)
        if has_pending_contract(party_type, party, rule.contract_template):
            continue

        # Create the contract instance
        contract = create_contract_instance(
            rule=rule,
            source_doc=source_doc,
            party_type=party_type,
            party=party
        )

        if contract:
            # Update rule execution stats
            update_rule_stats(rule)

            # Send notification if configured
            if rule.send_notification:
                send_contract_notification(contract, rule)

            # Block action if configured
            if rule.blocking_action:
                handle_blocking_action(source_doc, contract)


def is_rule_valid(rule):
    """Check if the rule is within its validity period."""
    current_date = getdate(today())

    if rule.valid_from and getdate(rule.valid_from) > current_date:
        return False

    if rule.valid_until and getdate(rule.valid_until) < current_date:
        return False

    return True


def matches_target_filters(rule, source_doc):
    """Check if the source document matches the rule's target filters."""
    if rule.apply_to == "All Users":
        return True

    if rule.apply_to == "Sellers Only":
        return source_doc.doctype in ("Seller Application", "Seller Profile")

    if rule.apply_to == "Buyers Only":
        return source_doc.doctype in ("Marketplace Order",)

    if rule.apply_to == "Specific User Type":
        doc_user_type = getattr(source_doc, "seller_type",
                                getattr(source_doc, "buyer_type", None))
        return doc_user_type == rule.target_user_type

    if rule.apply_to == "Specific Level":
        doc_seller_level = getattr(source_doc, "seller_tier", None)
        doc_buyer_level = getattr(source_doc, "buyer_level", None)
        if rule.target_seller_level and doc_seller_level:
            return doc_seller_level == rule.target_seller_level
        if rule.target_buyer_level and doc_buyer_level:
            return doc_buyer_level == rule.target_buyer_level

    return True
```

#### 2.3 Condition Evaluator

```python
def evaluate_conditions(rule, source_doc):
    """Evaluate the rule's condition child table against the source document.

    The conditions use abstract field names (e.g., 'seller_type', 'country')
    that must be mapped to actual document fields.
    """
    if not rule.conditions:
        return True  # No conditions = always match

    results = []
    for condition in rule.conditions:
        field_value = resolve_condition_field(condition.field, source_doc)
        result = compare_value(field_value, condition.operator, condition.value)
        results.append((result, condition.logic))

    # Combine results based on match_type
    if rule.match_type == "All":
        return all(r[0] for r in results)
    else:  # Any
        return any(r[0] for r in results)


# Mapping from Contract Rule Condition field names to document field paths
CONDITION_FIELD_MAP = {

    },

    },

    },

    },
    "total_revenue": {
        "Seller Profile": "total_sales_amount",
    },
}


def resolve_condition_field(field_name, source_doc):
    """Resolve abstract condition field name to actual document value."""
    doctype = source_doc.doctype
    mapping = CONDITION_FIELD_MAP.get(field_name, {})
    field_or_func = mapping.get(doctype)

    if callable(field_or_func):
        return field_or_func(source_doc)
    elif field_or_func:
        return getattr(source_doc, field_or_func, None)
    return None


def compare_value(field_value, operator, condition_value):
    """Compare a field value against a condition value using the operator."""
    if operator == "is set":
        return bool(field_value)
    if operator == "is not set":
        return not bool(field_value)

    if field_value is None:
        return False

    # Numeric comparison for numeric operators
    if operator in (">", "<", ">=", "<="):
        try:
            return _numeric_compare(float(field_value),
                                    operator,
                                    float(condition_value))
        except (ValueError, TypeError):
            return False

    str_field = str(field_value)
    str_condition = str(condition_value)

    if operator == "=":
        return str_field == str_condition
    elif operator == "!=":
        return str_field != str_condition
    elif operator == "contains":
        return str_condition in str_field
    elif operator == "not contains":
        return str_condition not in str_field
    elif operator == "in":
        values = [v.strip() for v in str_condition.split(",")]
        return str_field in values
    elif operator == "not in":
        values = [v.strip() for v in str_condition.split(",")]
        return str_field not in values

    return False


def _numeric_compare(a, operator, b):
    """Perform numeric comparison."""
    if operator == ">":
        return a > b
    elif operator == "<":
        return a < b
    elif operator == ">=":
        return a >= b
    elif operator == "<=":
        return a <= b
    return False
```

```python
def build_template_context(source_doc, party_type, party, template):
    """Build the Jinja template variable context from source data.

    Returns a dictionary of variables available for template rendering.
    """
    context = {
        # Universal variables
        "date": today(),
        "datetime": now_datetime(),
        "year": getdate(today()).year,

        # Template metadata
        "template_name": template.title,
        "template_code": template.template_code,
        "contract_type": template.contract_type,
        "template_version": template.version,

        # Tenant context
        "tenant": getattr(source_doc, "tenant", ""),
        "tenant_name": getattr(source_doc, "tenant_name", ""),
        "company": getattr(source_doc, "company", ""),
    }

    # Load party-specific data
    if party_type and party:
        party_doc = frappe.get_doc(party_type, party)
        context["party_name"] = resolve_party_name(party_type, party)
        context["party_type"] = party_type

        # Add all party document fields as template variables
        for field in party_doc.meta.fields:
            if field.fieldtype not in ("Section Break", "Column Break",
                                       "Tab Break", "Table", "Table MultiSelect",
                                       "HTML", "Image", "Attach", "Attach Image"):
                value = getattr(party_doc, field.fieldname, "")
                context[field.fieldname] = value or ""

    # Add source document fields (prefixed with source_)
    for field in source_doc.meta.fields:
        if field.fieldtype not in ("Section Break", "Column Break",
                                   "Tab Break", "Table", "Table MultiSelect",
                                   "HTML", "Image", "Attach", "Attach Image"):
            value = getattr(source_doc, field.fieldname, "")
            context["source_" + field.fieldname] = value or ""

    # Explicit overrides for common variables
    context.update(_get_explicit_bindings(source_doc, party_type, party))

    return context
```

#### 3.2 Jinja Template Resolution with Sanitization

```python
class SilentUndefined(Undefined):
    """Custom Jinja2 Undefined that returns empty string instead of raising.

    Prevents template rendering failures when a variable is not in the context.
    Logs the undefined variable access for debugging.
    """

    def __str__(self):
        frappe.log_error(
            title="Contract Template: Undefined variable",
            message="Variable '{0}' not found in template context".format(
                self._undefined_name
            )
        )
        return ""

    def __iter__(self):
        return iter([])

    def __bool__(self):
        return False


def resolve_jinja_template(template_content, context):
    """Resolve Jinja variables in contract template content.

    Uses a sandboxed Jinja environment with:
    - SilentUndefined: undefined variables render as empty string
    - No auto-escaping: content is HTML (Text Editor field)
    - No file system access: BaseLoader prevents template inclusion

    Args:
        template_content: The raw template HTML with Jinja variables
        context: Dictionary of variable name -> value mappings

    Returns:
        The resolved HTML content string
    """
    env = Environment(
        loader=BaseLoader(),
        undefined=SilentUndefined,
        autoescape=False,    # Content is HTML from Text Editor
        keep_trailing_newline=True
    )

    try:
        jinja_template = env.from_string(template_content)
        resolved = jinja_template.render(**context)
    except Exception as e:
        frappe.log_error(
            title=_("Contract Template: Jinja rendering error"),
            message=str(e)
        )
        # Fallback: return template with unresolved variables
        resolved = template_content

    return resolved


def resolve_party_name(party_type, party):
    """Resolve the display name for a party.

    Maps party_type to the appropriate name field.
    """
    name_field_map = {
        "Seller Profile": "seller_name",
        "Seller Application": "business_name",
        "User": "full_name",
        "Buyer Profile": "full_name",
        "Organization": "organization_name",
    }

    name_field = name_field_map.get(party_type, "name")

    try:
        return frappe.db.get_value(party_type, party, name_field) or party
    except Exception:
        return party
```

## Jinja Sanitization Strategy

### Problem Statement

Contract templates use Jinja2 syntax (`{{variable_name}}`) for dynamic content. If a template references a variable that is not provided in the context, the default Jinja behavior is to **raise an `UndefinedError`**, which would crash the contract creation process.

### Sanitization Approach

#### 1. SilentUndefined Class

A custom `jinja2.Undefined` subclass that returns an empty string instead of raising an exception:

```python
class SilentUndefined(Undefined):
    def __str__(self):
        # Log for debugging, return empty string for rendering
        frappe.log_error(
            title="Contract Template: Undefined variable",
            message="Variable '{0}' not found".format(self._undefined_name)
        )
        return ""

    def __iter__(self):
        return iter([])

    def __bool__(self):
        return False
```

**Behavior**:
- `{{undefined_var}}` renders as `""` (empty string)
- `{% if undefined_var %}...{% endif %}` evaluates to `False`
- `{% for item in undefined_var %}...{% endfor %}` iterates zero times
- Each undefined access is logged in Frappe Error Log for debugging

#### 2. Input Sanitization

All context values are sanitized before template rendering:

```python
def sanitize_context(context):
    """Sanitize all context values to prevent Jinja injection."""
    sanitized = {}
    for key, value in context.items():
        if isinstance(value, str):
            # Strip Jinja syntax from values to prevent injection
            sanitized[key] = value.replace("{{", "").replace("}}", "")
            sanitized[key] = sanitized[key].replace("{%", "").replace("%}", "")
        else:
            sanitized[key] = value
    return sanitized
```

#### 3. Template Validation on Publish

When a Contract Template transitions from Draft to Published, validate that all referenced variables are documented:

```python
def validate_template_variables(template_content):

    import re

    variables = re.findall(pattern, template_content)

    known_variables = {
        "party_name", "contact_name", "contact_email", "contact_phone",
        "business_name", "seller_type", "buyer_type", "tax_id", "tax_office",
        "city", "country", "address", "iban", "bank_name", "order_id",

        "mersis_number", "buyer_email", "buyer_phone", "buyer_tax_id",
        "organization_name", "shipping_city", "shipping_country",
        "commission_plan",
    }

    unknown = set(variables) - known_variables
    if unknown:
```

## Risks & Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| **Template rendering failure** crashes the triggering action (order submission, approval) | High | Use `SilentUndefined` Jinja class; wrap all rendering in try/except; never raise in doc_events handlers |
| **Blocking action deadlock** -- order stuck on hold with no way to release | High | Add manual override button on Contract Instance; admin can force-sign or skip; timeout escalation |
| **Cross-app dependency** -- tradehub_compliance hooks into tradehub_seller/tradehub_commerce events | Medium | Use Frappe's standard `doc_events` mechanism (designed for cross-app hooks); ensure `required_apps` includes `tradehub_core` |
| **Performance impact** on high-volume order submission | Medium | Cache active Contract Rules; use `frappe.db.exists()` for duplicate check; async notification sending |
| **Stale Contract Rule** still triggering after business logic changes | Medium | Rule validity period (`valid_from`/`valid_until`); status management (Active/Inactive); priority ordering |
| **Template version mismatch** -- old contract references new template | Low | Snapshot mechanism (template_version_snapshot, template_content_snapshot, content_hash_snapshot) ensures immutability |
| **Jinja injection** via user-supplied variable values (e.g., party_name containing `{{ }}`) | Medium | Strip Jinja syntax from all context values before rendering; use `sanitize_context()` |
| **Missing party reference** at trigger time (Seller Profile not yet created) | Medium | Check for party existence before contract creation; log error and skip if party is None |
| **Contract Revision read_only not enforced** -- content_snapshot/content_hash can be edited | Low | Fix `read_only: 0` fields in Contract Revision JSON to `read_only: 1` during implementation |
| **Email notification failure** blocks contract creation | Low | Wrap notification in try/except; contract creation should succeed even if email fails |

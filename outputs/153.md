# Task 153 — Sozlesme Merkezi ve Onay Merkezi (Contract Center & Consent Center)

## 1. Genel Bakis (Overview)

Bu belge, B2B pazar yeri platformunda **Sozlesme Merkezi (Contract Center)** ve **Onay Merkezi (Consent Center)** sistemlerinin birlesik teknik tasarimini icerir. Mevcut `tr_contract_center` ve `tr_consent_center` uygulamalari temel alinarak, her iki merkez icin gelismis ozellikler, versiyonlama mekanizmasi, hukuki denetim izi (legal audit trail) ve KVKK/GDPR uyumluluk gereksinimleri planlanmaktadir.

**Problem:** Mevcut sistemde sozlesme ve onay yonetimi ayri uygulamalar olarak calismasina ragmen, merkezi bir kullanici deneyimi sunulmamaktadir. Kullanicilar imzaladiklari sozlesmeleri, verdikleri onaylari ve bunlarin guncel versiyonlarini tek bir noktadan goruntuleyememektedir. Ayrica, versiyon degisikliklerinde yeniden kabul (re-acceptance) mekanizmasi bulunmamaktadir.

**Hedef:**
- Sozlesme Merkezi: Tum sozlesme tiplerinin (satici, alici, komisyon, NDA, SLA) merkezi yonetimi, dinamik icerik kurallari ve dijital/islak imza akisi
- Onay Merkezi: KVKK/GDPR uyumlu onay tiplerinin (kisisel veri isleme, pazarlama, veri paylasimi, cerez) merkezi yonetimi, kosul bazli onay metinleri
- Her iki merkez icin versiyonlama ve yeniden kabul mekanizmasi
- Degistirilemez (immutable) hukuki denetim izi
- Kullanici dashboard'unda "Sozlesmelerim" ve "Onay Tercihlerim" bolumleri

**Mevcut Altyapi Referanslari:**

| Uygulama | DocType'lar | Durum |
|----------|------------|-------|
| `tr_contract_center` | Contract Template, Contract Instance, Contract Revision, Contract Rule, ESign Provider, ESign Transaction | Mevcut — genisletilecek |
| `tr_consent_center` | Consent Record, Consent Topic, Consent Text, Consent Audit Log, Consent Method, Consent Channel | Mevcut — genisletilecek |

**Mevcut API Referanslari:**
- `tr_contract_center/api.py`: `create_instance`, `sign_wet`, `init_digital_sign`, `esign_callback`, `get_contracts`, `get_contract_detail`
- `tr_consent_center/api.py`: `grant_consent`, `revoke_consent`, `get_consents`, `get_current_text`, `verify_consent`, `check_consent`, `get_audit_trail`

**Task 146 ile Iliski:** Task 146'da planlanan Dinamik Sozlesme Sistemi (Contract Condition Rule, kural degerlendirme motoru) bu belgedeki Contract Center ile entegre calisacaktir. Contract Center, Task 146'daki dinamik kurallar uzerinden kisisellestirilmis sozlesme icerigini uretir ve kullaniciya sunar.

### 3.2 Kosul Bazli Onay Metinleri (Condition-Based Consent Texts)

Onay metinleri kullanici ozelliklerine gore farklilik gosterebilir. Mevcut `Consent Text` DocType'i genisletilerek kosul bazli metin secimi desteklenir.

#### 3.2.1 Consent Text Condition Rule — Yeni Child Table

Mevcut `Consent Text` DocType'ina eklenecek kosul kurali child table'i:

| Field Name | Fieldtype | Options | Required | Description |
|------------|-----------|---------|----------|-------------|
| `condition_type` | Select | user_type\nlocation\nlanguage\ncategory\nsubscription_plan | Yes | Kosul tipi |
| `condition_value` | Data | - | Yes | Kosul degeri (orn: "seller", "Istanbul", "tr") |
| `priority` | Int | - | Yes | Oncelik sirasi (dusuk = yuksek oncelik) |
| `is_active` | Check | - | No | Kural aktif/pasif (default: 1) |

**Metin Secim Algoritmasi:**

```python
# Pseudocode: Kosul bazli onay metni secimi
def get_consent_text_for_user(topic, user):
    """
    Kullanici ozelliklerine gore uygun onay metnini sec.
    """
    # 1. Topic icin tum aktif metinleri al
    texts = get_active_consent_texts(topic)

    # 2. Kullanici ozelliklerini belirle
    user_attrs = {
        "user_type": get_user_type(user),     # seller / buyer
        "location": get_user_location(user),   # Istanbul, Ankara...
        "language": get_user_language(user),    # tr / en
        "category": get_user_category(user),   # Elektronik, Gida...
        "subscription_plan": get_user_plan(user)  # Starter, Premium...
    }

    # 3. Her metin icin kosul kurallarini degerlendir
    best_match = None
    best_priority = float('inf')

    for text in texts:
        if not text.condition_rules:
            # Kosulsuz metin = varsayilan (en dusuk oncelik)
            if best_match is None:
                best_match = text
            continue

        all_match = True
        total_priority = 0

        for rule in text.condition_rules:
            if not rule.is_active:
                continue
            if user_attrs.get(rule.condition_type) != rule.condition_value:
                all_match = False
                break
            total_priority += rule.priority

        if all_match and total_priority < best_priority:
            best_match = text
            best_priority = total_priority

    return best_match
```

**Re-acceptance Pseudocode:**

```python
def trigger_reacceptance(template_name):
    """
    Sablon yeni versiyonu yayinlandiginda yeniden kabul tetiklenir.
    """
    template = frappe.get_doc("Contract Template", template_name)

    # Bu sablona bagli tum signed instance'lari bul
    signed_instances = frappe.get_all(
        "Contract Instance",
        filters={
            "template": template_name,
            "status": "Signed",
            "template_version_snapshot": ["<", template.version]
        },
        fields=["name", "party_type", "party"]
    )

    for instance in signed_instances:
        # Status guncelle
        frappe.db.set_value(
            "Contract Instance", instance.name,
            {
                "status": "Version Update Required",
                "pending_template_version": template.version
            }
        )

        # Bildirim gonder
        send_reacceptance_notification(
            instance.party_type,
            instance.party,
            template.title,
            template.version
        )

        # Audit log
        create_legal_audit_log(
            event_type="version_update_required",
            document_type="Contract Instance",
            document_name=instance.name,
            details=f"Template version updated from {instance.template_version_snapshot} to {template.version}"
        )

    return len(signed_instances)
```

### 5.2 Degistirilemezlik (Immutability) Mekanizmasi

Legal Audit Trail kayitlari olusturulduktan sonra **hicbir sekilde** degistirilemez veya silinemez:

```python
# Pseudocode: Immutability kontrolleri
class LegalAuditTrail(Document):
    def validate(self):
        if not self.is_new():
            frappe.throw(
                _("Legal Audit Trail records cannot be modified after creation"),
                frappe.PermissionError
            )

    def before_save(self):
        if not self.is_new():
            frappe.throw(
                _("Legal Audit Trail records are immutable"),
                frappe.PermissionError
            )

    def on_trash(self):
        frappe.throw(
            _("Legal Audit Trail records cannot be deleted. "
              "This is required for KVKK/GDPR compliance."),
            frappe.PermissionError
        )

    def before_insert(self):
        # Otomatik alanlar
        self.timestamp = now_datetime()
        self.user_id = frappe.session.user
        self.ip_address = frappe.local.request_ip or "127.0.0.1"
        self.user_agent = frappe.get_request_header("User-Agent") or ""
        self.session_id = frappe.session.sid or ""

        # Icerik hash'i dogrulama
        if self.consent_text_snapshot:
            import hashlib
            self.content_hash = hashlib.sha256(
                self.consent_text_snapshot.encode('utf-8')
            ).hexdigest()
```

```python
# Pseudocode: Kullanici onay tercihleri
def get_my_consent_preferences(user, party_type, party):
    """
    Kullanicinin tum onay tercihlerini kategorize ederek dondurur.
    """
    # Tum topic'lari al
    topics = frappe.get_all(
        "Consent Topic",
        filters={"enabled": 1},
        fields=["name", "topic_name", "topic_code", "category",
                "mandatory", "requires_explicit_consent",
                "allow_partial_consent", "display_order"],
        order_by="display_order asc"
    )

    # Kullanicinin mevcut onaylarini al
    consents = frappe.get_all(
        "Consent Record",
        filters={
            "party_type": party_type,
            "party": party,
            "status": ["in", ["Active", "Pending", "Reacceptance Required"]]
        },
        fields=["name", "consent_topic", "consent_method",
                "status", "granted_at", "consent_text_version",
                "pending_text_version"]
    )

    # Topic bazinda gruplama
    consent_map = {}
    for c in consents:
        key = c.consent_topic
        if key not in consent_map:
            consent_map[key] = []
        consent_map[key].append(c)

    result = {
        "mandatory": [],
        "communication": [],
        "third_party": [],
        "analytics": [],
        "reacceptance_required": []
    }

    for topic in topics:
        topic_consents = consent_map.get(topic.name, [])
        topic_data = {
            "topic": topic,
            "consents": topic_consents,
            "has_active": any(c.status == "Active" for c in topic_consents),
            "needs_reacceptance": any(
                c.status == "Reacceptance Required" for c in topic_consents
            )
        }

        if topic.mandatory:
            result["mandatory"].append(topic_data)
        elif topic.category in ["Marketing", "Communication"]:
            result["communication"].append(topic_data)
        elif topic.category == "Third Party":
            result["third_party"].append(topic_data)
        elif topic.category == "Analytics":
            result["analytics"].append(topic_data)

        if topic_data["needs_reacceptance"]:
            result["reacceptance_required"].append(topic_data)

    return result
```

### 7.3 Onay Geri Cekme (Consent Withdrawal)

```python
# Pseudocode: Onay geri cekme islem akisi
def withdraw_consent(party_type, party, topic, reason=None):
    """
    Kullanicinin bir onayini geri cekmesi.
    KVKK Madde 7: Acik rizanin geri alinmasi her zaman mumkundur.
    """
    # 1. Zorunlu onay kontrolu
    topic_doc = frappe.get_doc("Consent Topic", topic)
    if topic_doc.mandatory:
        frappe.throw(
            _("Bu onay zorunludur ve geri cekilemez. "
              "Hesabinizi kapatmak icin destek ile iletisime gecin.")
        )

    # 2. Mevcut revoke_consent API'sini cagir
    result = revoke_consent(
        party_type=party_type,
        party=party,
        topic=topic,
        reason=reason,
        ip_address=frappe.local.request_ip
    )

    # 3. Legal audit trail
    create_legal_audit_log(
        event_type="consent_revoked",
        document_type="Consent Record",
        document_name=result.get("data", {}).get("consent_record"),
        party_type=party_type,
        party=party,
        old_status="Active",
        new_status="Revoked",
        details={"reason": reason, "withdrawal_method": "dashboard"},
        legal_reference="KVKK Madde 7"
    )

    # 4. Etkilenen sistemleri bilgilendir
    notify_systems_on_consent_withdrawal(party_type, party, topic)

    return result
```

- [ ] **Veri Tasima**: Kullanici verilerini JSON/CSV/PDF olarak indirme imkani
- [ ] **Veri Silme**: Veri silme talebi mekanizmasi (saklama suresi tamamlananlari silebilme)
- [ ] **Denetim Izi**: Degistirilemez (immutable) kayit, SHA256 hash ile butunluk dogrulama
- [ ] **Cocuk Onay**: 18 yas alti kullanicilar icin veli onay mekanizmasi (opsiyonel, ileride)
- [ ] **Cross-Border Transfer**: Yurt disi veri aktarimi icin ek onay mekanizmasi
- [ ] **Data Breach Notification**: Veri ihlali durumunda bildirim mekanizmasi (opsiyonel)
- [ ] **DPO Contact**: Veri Koruma Gorevlisi iletisim bilgisi onay metinlerinde yer alacak
- [ ] **Retention Policy**: Saklama suresi dolunca otomatik silme/anonimize etme (mevcut: purge_expired_records)

## 11. Edge Cases (Ozel Durumlar)

### 11.1 Esanli Versiyon Guncelleme

**Senaryo:** Admin, sozlesme sablonunu gunceller. Ayni anda baska bir admin de ayni sablonu gunceller.

**Cozum:**
- Frappe'nin yerlesik `modified` kontrol mekanizmasi (optimistic locking) kullanilir
- `frappe.throw(_("Document has been modified after you have opened it"))` hatasi verilir
- Admin kullanicilar icin "Son Degisiklikleri Goster" butonu ile conflict resolution

### 11.2 Zorunlu Onay Geri Cekme

**Senaryo:** Kullanici KVKK zorunlu onayini geri cekmek istiyor.

**Cozum:**
- Zorunlu onaylar (`mandatory = 1`) icin geri cekme engellenir
- Kullaniciya mesaj: "Bu onay platformu kullanmak icin zorunludur. Hesabinizi kapatmak isterseniz destek ile iletisime gecin."
- KVKK Madde 7'ye gore: Acik rizanin geri alinmasi mumkundur, ancak zorunlu isleme kosullari (Madde 5/2) kapsamindaki islemler onaya bagli degildir

### 11.3 Suresi Dolan Yeniden Kabul

**Senaryo:** Kullanici `reacceptance_grace_days` suresi icinde yeni versiyonu kabul etmiyor.

**Cozum:**
- Sozlesme: `Contract Instance` status -> "Expired" (sozlesme gecersiz)
- Onay: `Consent Record` status -> "Expired" (onay geri cekilmis sayilir)
- Kullaniciya etki: Ilgili ozellikler devre disi birakilir
- Admin bildirimi: "Kullanici {party} yeniden kabul suresi icinde kabul etmedi"

### 11.4 Coklu Dil (Localization)

**Senaryo:** Turkce ve Ingilizce metinlerin paralel yurutulmesi.

**Cozum:**
- `Consent Text` ve `Contract Template` icin `language` alani
- Her dil icin ayri metin versiyonlari (ayni topic/template icin)
- Kullanicinin dil tercihine gore otomatik metin secimi
- Hukuki referanslar dile gore farklilik gosterir (KVKK vs. GDPR)

### 11.5 Bulk Registration

**Senaryo:** Yeni kullanici kaydi sirasinda 3-5 zorunlu sozlesme + 2-3 zorunlu onay sunulmasi.

**Cozum:**
- Kayit formu sonunda "Sozlesmeler ve Onaylar" adimi
- Tum zorunlu ogeleri tek sayfada gosterme (accordion/tab formati)
- Her bir oge icin ayri checkbox (KVKK geregi bundled consent yasak)
- "Tum Zorunlulari Kabul Et" butonu (sadece zorunlu ogeleri isaretler)
- Toplu imzalama icin `bulk_sign_contracts` API

### 11.6 Veri Silme Talebi

**Senaryo:** Kullanici KVKK Madde 11 kapsaminda veri silme talebi olusturuyor.

**Cozum:**
- Legal Audit Trail kayitlari **SILINEMEZ** (yasal zorunluluk — saklama suresi icerisinde)
- Consent Record'lar saklama suresi (`retention_until`) dolana kadar silinemez (mevcut: `test_cannot_delete_before_retention`)
- Kisisel veriler anonimize edilebilir (ad/soyad -> "Anonimize Kullanici", e-posta -> hash)
- Silme talebi is akisi: Talep -> DPO Inceleme -> Onay -> Anonimizasyon/Silme -> Teyit

### 11.7 E-Sign Saglayici Ariza

**Senaryo:** Dijital imza saglayici (ESign Provider) cevrimdisi veya hatali.

**Cozum:**
- `poll_esign_status` hourly task ile durum kontrolu (mevcut)
- 48 saat icinde yanit gelmezse kullaniciya bildirim: "Dijital imza islemi tamamlanamadi. Islak imza yontemiyle devam edebilirsiniz."
- Fallback: Islak imza yontemine gecis secenegi
- Admin dashboard'da e-sign hata sayaci

### 11.8 Geriye Uyumluluk

**Senaryo:** Mevcut `Consent Audit Log` kayitlari ve `Contract Revision` kayitlari.

**Cozum:**
- Mevcut DocType'lar korunur, yeni alanlar eklenir (geriye uyumlu)
- Yeni `Legal Audit Trail` ek olarak olusturulur (mevcut kayitlar korunur)
- Gecis doneminde her iki sisteme yazilir (dual-write)
- Migrasyon script'i: Mevcut `Consent Audit Log` kayitlarini `Legal Audit Trail`'e kopyalar (opsiyonel)

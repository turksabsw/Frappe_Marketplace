# Dinamik Özellik Seti Sistemi (Dynamic Attribute Set System)

## 1. Bağlam (Context)

### Mevcut Durum

TradeHub platformunda şu anda **iki ayrı özellik (attribute) altyapısı** paralel olarak çalışmaktadır. Bunlar farklı zaman dilimlerinde, farklı ihtiyaçlara göre tasarlanmıştır. Bu belge, her iki sistemi analiz ederek boşlukları tespit edecek, yeni/genişletilmiş child tablo tanımlarını sunacak ve kategori bazlı otomatik atama mantığını planlayacaktır.

---

## 2. Analiz (Analysis)

### 2.1 İki Sistem Arasındaki Fark Analizi

| Özellik | Sistem A (Legacy) | Sistem B (PIM) |
|---------|-------------------|----------------|
| **Özellik Tanımlayıcı** | `attribute_name` (Data) | `attribute_code` (Data, lowercase) |
| **Tip Sayısı** | 6 (Select, Text, Numeric, Boolean, Date, Color) | 22 (kapsamlı) |
| **Gruplama** | Attribute Set (flat gruplar) | PIM Attribute Group (form organizasyonu) |
| **Miras Alma** | ✅ `inherit_from` ile set bazlı | ❌ Yok |
| **Override Desteği** | ✅ Attribute Set Item'da is_required, is_variant override | ❌ Yok |
| **Lokalizasyon** | ❌ Yok | ✅ `is_localizable`, `locale` |
| **Kanal Kapsamı** | ❌ Yok | ✅ `is_scopable`, `channel` |
| **Marketplace Eşleme** | ❌ Yok | ✅ Amazon, eBay, Alibaba, Trendyol, Hepsiburada, N11 |
| **Değer Saklama** | Tek `value` alanı (string) | Multi-type (value_text, value_int, value_float, vb.) |
| **Validasyon** | ❌ Yok (numeric min/max hariç) | ✅ `validation_regex`, `min_value`, `max_value` |
| **Benzersizlik** | ❌ Yok | ✅ `is_unique` |
| **Kategori Bağlantısı** | Category.attribute_set → Attribute Set | ❌ Doğrudan bağlantı yok |
| **Kullanıcılar** | Listing (Seller'lar) | PIM Product (Yöneticiler) |

### 2.2 Tespit Edilen Boşluklar (Gap Analysis)

#### Boşluk 1: Product Category'de Attribute Set Yok
Product Category (yeni, multi-tenant) DocType'ında `attribute_set` alanı bulunmuyor. Yalnızca eski Category DocType'ı Attribute Set'e bağlanabiliyor. **102.md planına göre** ürün DocType'ları Product Category'ye geçirilecek, bu nedenle Product Category'ye de `attribute_set` bağlantısı gerekli.

#### Boşluk 2: Tek Attribute Set Limiti
Hem Category hem Listing DocType'larında sadece **tek bir** `attribute_set` Link alanı var. Gerçek dünyada bir ürün birden fazla özellik grubuna ihtiyaç duyabilir:
- "Elektronik > Bilgisayarlar" → Genel Özellikler + Donanım Özellikleri + Ekran Özellikleri
- "Giyim > Elbise" → Genel Özellikler + Beden/Ölçü + Kumaş Özellikleri

#### Boşluk 3: Legacy ve PIM Arası Köprü Yok
Listing DocType `Attribute` (legacy) master'a bağlanırken, PIM Product `PIM Attribute` master'a bağlanıyor. İki sistem arasında mapping veya köprü mekanizması yok.

#### Boşluk 4: Kategori Değişikliğinde Otomatik Atama Yok
Bir ürünün kategorisi değiştirildiğinde, yeni kategorinin özellik setinin otomatik atanması için mekanizma yok. Manuel müdahale gerekiyor.

#### Boşluk 5: Çoklu Set Bazlı Validasyon Yok
Mevcut sistemde bir ürünün hangi attribute set'lerinden hangi özellikleri doldurması gerektiğine dair zorunluluk kontrolü (required validation) yok.

#### Boşluk 6: Frontend'de Gruplanmış Gösterim Yok
Özellik değerleri düz bir tablo (child table) olarak gösteriliyor. Attribute Group veya Attribute Set bazlı sekmeler/akordeon yapısı yok.

### 2.3 Hedef Mimari

Aşağıdaki tasarım, her iki sistemi uyumlu hale getirirken mevcut altyapıyı bozmadan genişletir:

```
Product Category (yeni)
├── attribute_set Link → Attribute Set (birincil set - geriye uyumluluk)
└── category_attribute_sets Table → Category Attribute Set (yeni child table)
    ├── Genel Özellikler (Attribute Set) [is_default: 1, display_order: 0]
    ├── Donanım Özellikleri (Attribute Set) [display_order: 1]
    └── Ekran Özellikleri (Attribute Set) [display_order: 2]

Listing / PIM Product
├── attribute_set Link → Attribute Set (birincil set - korunur)
├── product_attribute_sets Table → Product Attribute Set (yeni child table)
│   ├── Genel Özellikler [auto_assigned: 1, source_category: "Bilgisayarlar"]
│   └── Donanım Özellikleri [auto_assigned: 1, source_category: "Bilgisayarlar"]
├── attribute_values Table → (mevcut child table - korunur)
└── Validasyon: Tüm is_required özellikler doldurulmuş mu? ✓
```

## 5. Validasyon Kuralları (Validation Rules)

### 5.1 Zorunlu Özellik Kontrolü (Required Attribute Validation)

Bir ürün kaydedilirken, atanmış Attribute Set'lerdeki `is_required: 1` olan tüm özelliklerin doldurulmuş olması kontrol edilmelidir.

**Mantık:**

```python
# listing.py ve pim_product.py validate() metodu (planlama notu — kod oluşturulmayacak)
def validate_required_attributes(self):
    """Atanmış özellik setlerindeki zorunlu özelliklerin doldurulup doldurulmadığını kontrol et"""
    # 1. product_attribute_sets child tablosundaki aktif setleri al
    active_sets = [row for row in (self.product_attribute_sets or []) if row.is_active]

    # 2. Her setin özelliklerini kontrol et
    filled_attributes = {row.attribute for row in (self.attribute_values or [])}

    missing = []
    for pas in active_sets:
        attribute_set = frappe.get_doc("Attribute Set", pas.attribute_set)
        for item in attribute_set.attributes:
            if item.is_required and item.attribute not in filled_attributes:
                missing.append(f"{item.attribute_label or item.attribute} ({pas.attribute_set_name})")

    if missing:
        frappe.throw(
            _("Required attributes are missing: {0}").format(", ".join(missing)),
            title=_("Missing Required Attributes")
        )
```

### 5.2 Tip Kontrolü (Type Validation)

Özellik değerlerinin, özellik tanımındaki tipe uygun olması kontrol edilmelidir:

| Attribute Type | value alanı | Validasyon Kuralı | Hata Mesajı |
|---------------|------------|-------------------|-------------|
| Select | value | Attribute Value tablosundaki seçeneklerden biri olmalı | "'{0}' is not a valid option for {1}. Valid options: {2}" |
| Numeric | value | Sayısal değer olmalı, min/max aralığında | "Value for {0} must be a number between {1} and {2}" |
| Boolean | value | "0" veya "1" olmalı | "Value for {0} must be Yes or No" |
| Date | value | Geçerli tarih formatı (YYYY-MM-DD) | "Value for {0} must be a valid date" |
| Color | value | Hex renk kodu formatı (#XXXXXX) | "Value for {0} must be a valid hex color code" |
| Text | value | Boş olmayan string | "Value for {0} cannot be empty" |

**Python Validasyon Mantığı:**

```python
# validate_attribute_types() planlama notu
def validate_attribute_types(self):
    """Özellik değerlerinin tip uyumluluğunu kontrol et"""
    for row in (self.attribute_values or []):
        if not row.attribute:
            continue

        attr_doc = frappe.get_cached_doc("Attribute", row.attribute)

        if attr_doc.attribute_type == "Select":
            valid_options = [v.attribute_value for v in attr_doc.attribute_values]
            if row.value and row.value not in valid_options:
                frappe.throw(
                    _("'{0}' is not a valid option for {1}. Valid options: {2}").format(
                        row.value, attr_doc.attribute_label or attr_doc.attribute_name,
                        ", ".join(valid_options)
                    )
                )

        elif attr_doc.attribute_type == "Numeric":
            try:
                val = float(row.value)
            except (ValueError, TypeError):
                frappe.throw(
                    _("Value for {0} must be a number").format(
                        attr_doc.attribute_label or attr_doc.attribute_name
                    )
                )
            if attr_doc.numeric_min is not None and val < attr_doc.numeric_min:
                frappe.throw(
                    _("Value for {0} must be at least {1}").format(
                        attr_doc.attribute_label, attr_doc.numeric_min
                    )
                )
            if attr_doc.numeric_max is not None and val > attr_doc.numeric_max:
                frappe.throw(
                    _("Value for {0} must be at most {1}").format(
                        attr_doc.attribute_label, attr_doc.numeric_max
                    )
                )

        elif attr_doc.attribute_type == "Boolean":
            if row.value not in ("0", "1", "true", "false", "True", "False"):
                frappe.throw(
                    _("Value for {0} must be Yes or No").format(
                        attr_doc.attribute_label or attr_doc.attribute_name
                    )
                )
```

### 5.3 Benzersizlik Kontrolü (Unique Validation)

Attribute Definition'da `is_unique: 1` olan özellikler için, değerin diğer ürünlerde kullanılmadığı kontrol edilmelidir:

```python
# validate_unique_attributes() planlama notu
def validate_unique_attributes(self):
    """Benzersiz özellik değerlerini kontrol et"""
    for row in (self.attribute_values or []):
        if not row.attribute:
            continue

        # Attribute Set Item veya Attribute Definition'daki is_unique kontrolü
        # Attribute master'daki is_unique flag'i kontrol et (PIM Attribute için)
        attr_doc = frappe.get_cached_doc("Attribute", row.attribute)

        # Legacy Attribute'te is_unique yoksa atla
        # Eğer Attribute Definition kullanılıyorsa oradan kontrol et
        # Bu örnekte basit bir duplicate kontrol:
        if hasattr(attr_doc, 'is_unique') and attr_doc.is_unique:
            existing = frappe.db.sql("""
                SELECT parent FROM `tabListing Attribute Value`
                WHERE attribute = %s AND value = %s AND parent != %s
                LIMIT 1
            """, (row.attribute, row.value, self.name))

            if existing:
                frappe.throw(
                    _("Value '{0}' for attribute '{1}' is already used by {2}").format(
                        row.value, attr_doc.attribute_name, existing[0][0]
                    ),
                    title=_("Duplicate Attribute Value")
                )
```

### 6.2 Uygulama Yaklaşımı: Custom Form Renderer

Frappe'nin standart child table görünümü yerine, Attribute Set bazlı gruplandırma için özel bir form render mekanizması kullanılır:

```javascript
// listing.js veya pim_product.js (planlama notu — kod oluşturulmayacak)
frappe.ui.form.on("Listing", {
    refresh: function(frm) {
        // Özellikler sekmesinde akordeon render'ı
        render_attribute_accordion(frm);
    },

    category: function(frm) {
        // Kategori değiştiğinde özellik setlerini otomatik ata
        if (frm.doc.category) {
            frappe.call({
                method: "tradehub_catalog.api.attributes.get_category_attribute_sets",
                args: { category: frm.doc.category },
                callback: function(r) {
                    if (r.message) {
                        auto_assign_attribute_sets(frm, r.message);
                    }
                }
            });
        }
    }
});

function render_attribute_accordion(frm) {
    // 1. product_attribute_sets'ten aktif setleri al
    // 2. Her set için attribute listesini çek
    // 3. attribute_values'dan mevcut değerleri eşle
    // 4. Akordeon HTML oluştur ve form'a inject et
    // 5. Input değişikliklerini attribute_values child table'a geri yaz

    const container = frm.fields_dict.attribute_values_section.$wrapper;
    container.empty();

    const sets = frm.doc.product_attribute_sets || [];
    sets.sort((a, b) => (a.display_order || 0) - (b.display_order || 0));

    sets.forEach(function(set_row) {
        if (!set_row.is_active) return;

        const section = $(`
            <div class="attribute-set-accordion" data-set="${set_row.attribute_set}">
                <div class="accordion-header">
                    <span class="indicator-pill ${set_row.source === 'Auto' ? 'blue' : 'gray'}">
                        ${set_row.source}
                    </span>
                    <strong>${set_row.attribute_set_name || set_row.attribute_set}</strong>
                </div>
                <div class="accordion-body"></div>
            </div>
        `);

        container.append(section);

        // Her attribute için input render et
        frappe.call({
            method: "tradehub_catalog.api.attributes.get_attribute_set_items",
            args: { attribute_set: set_row.attribute_set },
            callback: function(r) {
                if (r.message) {
                    r.message.forEach(function(attr) {
                        render_attribute_input(section.find('.accordion-body'), attr, frm);
                    });
                }
            }
        });
    });
}
```

### 7.3 Veri Migrasyonu

Mevcut tek `attribute_set` Link alanlarını çoklu sisteme dönüştürme:

```python
# migrate_attribute_sets.py (planlama notu — kod oluşturulmayacak)
def migrate_existing_attribute_sets():
    """Mevcut tek attribute_set atamalarını çoklu sisteme taşı"""

    # 1. Category'deki attribute_set'leri category_attribute_sets child table'a kopyala
    categories = frappe.get_all("Category", filters={"attribute_set": ["is", "set"]},
                                 fields=["name", "attribute_set"])
    for cat in categories:
        doc = frappe.get_doc("Category", cat.name)
        if not doc.category_attribute_sets:
            doc.append("category_attribute_sets", {
                "attribute_set": cat.attribute_set,
                "is_default": 1,
                "is_required": 1,
                "display_order": 0
            })
            doc.save(ignore_permissions=True)

    # 2. Listing'deki attribute_set'leri product_attribute_sets child table'a kopyala
    listings = frappe.get_all("Listing", filters={"attribute_set": ["is", "set"]},
                              fields=["name", "attribute_set"])
    for lst in listings:
        doc = frappe.get_doc("Listing", lst.name)
        if not doc.product_attribute_sets:
            doc.append("product_attribute_sets", {
                "attribute_set": lst.attribute_set,
                "source": "Manual",
                "is_active": 1,
                "display_order": 0
            })
            doc.save(ignore_permissions=True)

    # 3. Mevcut attribute_values'daki attribute_set alanını doldur
    frappe.db.sql("""
        UPDATE `tabListing Attribute Value` lav
        INNER JOIN `tabListing` l ON lav.parent = l.name
        SET lav.attribute_set = l.attribute_set
        WHERE lav.attribute_set IS NULL AND l.attribute_set IS NOT NULL
    """)

    frappe.db.commit()
```

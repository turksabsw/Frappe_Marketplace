# Satici Kategori Teklif Sistemi (Seller Category Proposal System)

## 1. Baglam (Context)

### Mevcut Durum

TradeHub platformunda kategoriler yalnizca **System Manager** rolune sahip yoneticiler tarafindan olusturulabilmektedir. Satircilar, mevcut kategori agacinda bulunmayan urunler icin yeni kategori ihtiyaclarini iletebilecek resmi bir kanal bulunmamaktadir. Bu belge, saticilarin yeni kategori onerileri sunabilecegi, yonetici onay sureci ile denetlenen bir **Category Proposal** DocType ve Frappe Workflow tanimini planlamaktadir.

---

### 1.1 Mevcut Kategori Yonetimi

#### Category DocType (Legacy)

- **Konum:** `tradehub_catalog/doctype/category/category.json`
- **Adlandirma:** `autoname: "field:category_name"` (benzersiz ad)
- **Agac Yapisi:** `is_tree: 1`, `nsm_parent_field: "parent_category"`
- **Izinler:**
  - System Manager: Tam CRUD + share + export
  - All: Salt okunur (read + print + report + export)

#### Product Category DocType (Yeni / Multi-Tenant)

- **Konum:** `tradehub_catalog/doctype/product_category/product_category.json`
- **Adlandirma:** `autoname: "field:category_name"` (benzersiz ad)
- **Agac Yapisi:** `is_tree: 1`, `nsm_parent_field: "parent_product_category"`
- **Multi-Tenant:** `tenant` Link + `is_global` Check alanlari
- **Izinler:**
  - System Manager: Tam CRUD + share + export
  - Seller: Salt okunur (read + print + report + export)
  - Buyer: Salt okunur (read + print)

### 1.2 Satici Profili (Seller Profile)

- **Konum:** `tradehub_seller/doctype/seller_profile/seller_profile.json`
- **Adlandirma:** `autoname: "SELLER-.#####"`
- **Onemli Alanlar:**

| fieldname | fieldtype | label | options | reqd | description |
|-----------|-----------|-------|---------|------|-------------|
| seller_name | Data | Seller Name | — | 1 | Satici adi |
| user | Link | User | User | 1 | Bagli kullanici hesabi (unique) |
| seller_type | Select | Seller Type | Individual\nBusiness\nEnterprise | 1 | Satici tipi |
| status | Select | Status | Pending\nUnder Review\nActive\nSuspended\nBlocked\nInactive\nVacation | 1 | Satici durumu |
| tenant | Link | Tenant | Tenant | 1 | Multi-tenant izolasyon |
| seller_tier | Data | Seller Tier | — | 0 | Satici seviyesi |
| seller_score | Float | Seller Score | — | 0 | Performans puani |

### 1.3 Mevcut Teklif/Onay Sistemi

TradeHub platformunda su anda **hicbir teklif (proposal) veya oneri (suggestion) sistemi bulunmamaktadir**. Satircilar yeni kategori ihtiyaclarini yalnizca destek talepleri uzerinden iletebilmektedir. Bu durum:

- Taleplerin izlenememesine,
- Tekrarlanan taleplerin tespit edilememesine,
- Yonetici is yukunu artirmasina,
- Satici memnuniyetsizligine yol acmaktadir.

### 1.4 Frappe Workflow Altyapisi

Frappe Framework, yerlesik **Workflow** DocType ile belge durumu gecislerini yonetir:

- **Workflow:** Durum gecis kurallarini tanimlar (state, transition, allowed roles)
- **Workflow State:** Her bir durum icin renk, simge ve stil tanimlar
- **Workflow Action:** Kullanicilara bekleyen islem bildirimleri gonderir
- Workflow bir DocType'a atandiginda, `workflow_state` alani otomatik eklenir
- Durum gecisleri `on_update` hook'u ile tetiklenebilir

## 2. Analiz (Analysis)

### 2.1 Bosluk Analizi (Gap Analysis)

| Gereksinim | Mevcut Durum | Ihtiyac | Aksiyon |
|------------|--------------|---------|---------|
| Satici kategori onerisi | Yok | Satircilar yeni kategori teklif edebilmeli | Category Proposal DocType olustur |
| Onay is akisi | Yok | 4 asamali durum gecisi | Frappe Workflow tanimlama |
| Otomatik kategori olusturma | Yok | Onaylanan teklif -> gercek kategori | on_update hook ile auto-create |
| Bildirim sistemi | Yok | Durum degisikliklerinde e-posta | Notification DocType kaydi |
| Spam onleme | Yok | Ayni satici cok fazla teklif gonderememeli | Rate-limit validasyonu |
| Tekrar teklif kontrolu | Yok | Ayni isimde teklif tekrarina izin vermeme | Duplicate check validasyonu |
| Satici izolasyonu | Yok | Satirci yalniz kendi tekliflerini gorebilmeli | Permission query condition |

### 2.2 Tasarim Kararlari

#### Neden Ayri DocType?

Category Proposal, dogrudan Category veya Product Category uzerinde bir alan olarak eklenmiyor cunku:

1. **Onay sureci**: Teklif, nihai kategori olusturulmadan once gozden gecirilmelidir
2. **Izlenebilirlik**: Tekliflerin gecmisi (kim, ne zaman, neden reddedildi) kayit altinda tutulmalidir
3. **Izolasyon**: Teklif kayitlari kategori agacini kirletmemelidir
4. **Workflow uyumlulugu**: Frappe Workflow ayri bir DocType uzerinde calisir

#### Hedef Kategori DocType: Product Category

Onaylanan teklifler **Product Category** DocType'inda gercek kategori olarak olusturulacaktir. Sebepler:

- Multi-tenant desteği (`tenant`, `is_global` alanlari)
- Daha yeni ve aktif olarak gelistirilen DocType
- Seller ve Buyer rolleri icin izin yapilandirmasi mevcut
- `is_group`, `level`, `full_path` alanlari gibi zengin metadata

#### Workflow Yaklasimi: Frappe Yerlesik Workflow

Ozel Python state machine yerine Frappe'nin yerlesik Workflow modulu tercih ediliyor:

- Duzenleme kolayligi (Desk UI'dan workflow duzenlenebilir)
- Workflow Action bildirimleri otomatik
- Dashboard'da bekleyen islemler goruntulenir
- Audit trail (durum gecis gecmisi) yerlesik

```json
  "autoname": "CAT-PROP-.YYYY.-.#####",
  "naming_rule": "By \"Naming Series\" field",
  "allow_rename": 0,
  "is_submittable": 0,
  "track_changes": 1,
  "quick_entry": 0,
  "sort_field": "creation",
  "sort_order": "DESC",
  "title_field": "proposed_name",
  "search_fields": "proposed_name, seller, status",
  "show_name_in_global_search": 0
}
```

**Aciklamalar:**
- `autoname: "CAT-PROP-.YYYY.-.#####"` — Yil bazli sira numarasi (orn. CAT-PROP-2026-00001)
- `is_submittable: 0` — Workflow ile yonetilecek, submit/cancel gerekmez
- `track_changes: 1` — Tum degisiklikler kayit altina alinir
- `show_name_in_global_search: 0` — Awesome Bar'da gorunmez (yalnizca modul ici erisim)

## 4. Workflow Tanimi (Workflow Definition)

### 4.1 Workflow Durumlari (States)

Category Proposal icin 4 asamali bir is akisi tanimlanir:

| # | State | Doc Status | Style | Izin Verilen Roller | Guncelleme Izni | Aciklama |
|---|-------|-----------|-------|---------------------|-----------------|----------|
| 1 | Draft | 0 | — | Seller, System Manager | Seller, System Manager | Satici teklifi hazirlama asamasinda |
| 2 | Pending Review | 0 | Primary | System Manager | System Manager | Teklif incelemeye gonderildi |
| 3 | Approved | 0 | Success | System Manager | — | Teklif onaylandi, kategori otomatik olusturulacak |
| 4 | Rejected | 0 | Danger | System Manager | — | Teklif reddedildi |

### 4.2 Workflow Gecisleri (Transitions)

| # | State | Action | Next State | Allowed | Condition | Aciklama |
|---|-------|--------|-----------|---------|-----------|----------|
| 1 | Draft | Submit for Review | Pending Review | Seller, System Manager | — | Satici teklifi incelemeye gonderir |
| 2 | Pending Review | Approve | Approved | System Manager | — | Yonetici teklifi onaylar |
| 3 | Pending Review | Reject | Rejected | System Manager | — | Yonetici teklifi reddeder |
| 4 | Rejected | Revise | Draft | Seller | — | Satici reddedilen teklifi revize eder |
| 5 | Draft | Cancel | Cancelled | Seller, System Manager | — | Teklif iptal edilir (opsiyonel) |

> **Not:** Gecis #4 saticilara reddedilen tekliflerini duzenleyip tekrar gonderme imkani tanir. Gecis #5 opsiyonel olarak taslak asamasinda iptal imkani sunar.

## 5. Otomatik Kategori Olusturma (Auto-Creation of Approved Category)

### 5.1 Tetikleme Mekanizmasi

Teklif **Approved** durumuna gectiginde, `on_update` hook'u tetiklenerek otomatik olarak yeni bir **Product Category** kaydi olusturulur.

### 5.2 Python Controller Kodu (category_proposal.py)

```python
# tradehub_catalog/doctype/category_proposal/category_proposal.py

import frappe
from frappe.model.document import Document
from frappe.utils import now_datetime, cint

class CategoryProposal(Document):
    def validate(self):
        self.validate_proposed_name()
        self.validate_justification()
        self.validate_rate_limit()
        self.validate_duplicate_proposal()

    def on_update(self):
        if self.status == "Approved" and not self.created_category:
            self.create_category_from_proposal()
        if self.status in ("Approved", "Rejected") and not self.reviewed_by:
            self.reviewed_by = frappe.session.user
            self.reviewed_at = now_datetime()

    def validate_proposed_name(self):
        """Onerilen kategori adi validasyonu"""
        if not self.proposed_name or len(self.proposed_name.strip()) < 3:
            frappe.throw("Onerilen kategori adi en az 3 karakter olmalidir.")
        if len(self.proposed_name) > 140:
            frappe.throw("Onerilen kategori adi en fazla 140 karakter olabilir.")

        # Mevcut kategorilerde ayni isimde kontrol
        existing = frappe.db.exists("Product Category", {
            "category_name": self.proposed_name.strip()
        })
        if existing:
            frappe.throw(
                f"'{self.proposed_name}' adinda bir kategori zaten mevcut: {existing}"
            )

    def validate_justification(self):
        """Gerekce minimum uzunluk kontrolu"""
        if self.justification:
            # HTML etiketlerini temizle ve uzunluk kontrol et
            plain_text = frappe.utils.strip_html(self.justification)
            if len(plain_text.strip()) < 20:
                frappe.throw(
                    "Gerekce alani en az 20 karakter icermelidir (HTML etiketleri haric)."
                )

    def validate_rate_limit(self):
        """Spam onleme: Satirci basina rate limit kontrolu"""
        if self.is_new() and self.seller:
            # Son 24 saatte ayni saticidan kac teklif gonderilmis
            recent_count = frappe.db.count("Category Proposal", {
                "seller": self.seller,
                "creation": (">=", frappe.utils.add_days(
                    frappe.utils.now_datetime(), -1
                )),
                "name": ("!=", self.name or "")
            })
            max_daily = cint(
                frappe.db.get_single_value(
                    "TradeHub Catalog Settings",
                    "max_daily_category_proposals"
                )
            ) or 5  # Varsayilan: gunluk 5 teklif

            if recent_count >= max_daily:
                frappe.throw(
                    f"Gunluk maksimum kategori teklifi limitine ({max_daily}) ulastiniz. "
                    f"Lutfen 24 saat sonra tekrar deneyin."
                )

        # Son 7 gunde toplam teklif kontrolu
        if self.is_new() and self.seller:
            weekly_count = frappe.db.count("Category Proposal", {
                "seller": self.seller,
                "creation": (">=", frappe.utils.add_days(
                    frappe.utils.now_datetime(), -7
                )),
                "name": ("!=", self.name or "")
            })
            max_weekly = cint(
                frappe.db.get_single_value(
                    "TradeHub Catalog Settings",
                    "max_weekly_category_proposals"
                )
            ) or 15  # Varsayilan: haftalik 15 teklif

            if weekly_count >= max_weekly:
                frappe.throw(
                    f"Haftalik maksimum kategori teklifi limitine ({max_weekly}) ulastiniz. "
                    f"Lutfen daha sonra tekrar deneyin."
                )

    def validate_duplicate_proposal(self):
        """Ayni isimde aktif teklif tekrar kontrolu"""
        if self.is_new() or self.has_value_changed("proposed_name"):
            existing_proposal = frappe.db.exists("Category Proposal", {
                "proposed_name": self.proposed_name.strip(),
                "status": ("in", ["Draft", "Pending Review"]),
                "name": ("!=", self.name or "")
            })
            if existing_proposal:
                frappe.throw(
                    f"'{self.proposed_name}' adinda aktif bir teklif zaten mevcut: "
                    f"{existing_proposal}. Lutfen mevcut teklifi takip edin."
                )

    def create_category_from_proposal(self):
        """Onaylanan tekliften otomatik Product Category olustur"""
        try:
            new_category = frappe.get_doc({
                "doctype": "Product Category",
                "category_name": self.proposed_name.strip(),
                "parent_product_category": self.parent_category,
                "is_group": 0,
                "enabled": 1,
                "tenant": self.tenant,
                "is_global": 0,
                "description": frappe.utils.strip_html(
                    self.justification or ""
                )[:500]
            })
            new_category.insert(ignore_permissions=True)

            # Proposal'a olusturulan kategori referansini kaydet
            self.db_set("created_category", new_category.name, update_modified=False)
            self.db_set("created_at", now_datetime(), update_modified=False)

            frappe.msgprint(
                f"Kategori '{new_category.category_name}' basariyla olusturuldu.",
                indicator="green",
                title="Kategori Olusturuldu"
            )

        except frappe.DuplicateEntryError:
            frappe.throw(
                f"'{self.proposed_name}' adinda bir kategori zaten mevcut. "
                f"Otomatik olusturma basarisiz."
            )
        except Exception as e:
            frappe.log_error(
                title="Category Proposal Auto-Create Failed",
                message=f"Proposal: {self.name}\nError: {str(e)}"
            )
            frappe.throw(
                "Kategori olusturma sirasinda bir hata olustu. "
                "Lutfen sistem yoneticisiyle iletisime gecin."
            )
```

### 5.3 Otomatik Olusturma Akisi

```
1. Yonetici "Approve" butonuna tiklar
2. Workflow, status'u "Approved" olarak gunceller
3. on_update hook tetiklenir
4. create_category_from_proposal() cagirilir
5. Yeni Product Category olusturulur:
   - category_name = proposed_name
   - parent_product_category = parent_category
   - is_group = 0 (yaprak dugum)
   - enabled = 1
   - tenant = proposal.tenant
   - is_global = 0
6. created_category ve created_at alanlari guncellenir
7. Satirci bilgilendirilir (Notification)
```

### 5.4 Onemli Hususlar

- `ignore_permissions=True` kullanilir cunku Workflow gecisi System Manager yetkisiyle yapilir, ancak kategori olusturma icin ayri bir izin kontrolu gerekmez
- `DuplicateEntryError` yakalanir: Yarış durumu (race condition) icin koruma
- `frappe.log_error` ile hata kaydi tutulur
- Olusturulan kategori `is_global: 0` ile baslar — yonetici isterse sonra `is_global: 1` yapabilir
- Olusturulan kategori `is_group: 0` ile baslar — alt kategori ihtiyaci olursa yonetici duzenleyebilir

### 7.1 Rate-Limit Yapisi

### 7.2 TradeHub Catalog Settings Alanlari

Mevcut `TradeHub Catalog Settings` (veya yeni olusturulacak Settings DocType) icine eklenecek alanlar:

### 7.3 Ek Spam Onleme Stratejileri

1. **Satirci Durum Kontrolu:** Yalnizca `status == "Active"` olan satircilar teklif olusturabilir
2. **Minimum Satirci Yasi:** Hesap acilisindan itibaren minimum 7 gun gecmis olmali
3. **Tekrar Teklif Kontrolu:** Ayni isimde aktif (Draft veya Pending Review) teklif varsa yeni teklif engellenir
4. **Satirci Tier Bazli Limit:** Premium satircilar daha yuksek limitlere sahip olabilir

```python
def validate_seller_eligibility(self):
    """Satirci uygunluk kontrolu"""
    if not self.seller:
        return

    seller = frappe.get_doc("Seller Profile", self.seller)

    # Aktif satirci kontrolu
    if seller.status != "Active":

        "seller": self.seller,
        "status": ("in", ["Draft", "Pending Review"])
    })

        frappe.db.get_single_value(
            "TradeHub Catalog Settings", "max_pending_proposals"
        )
    ) or 3
```

## 8. Izin Yapisi (Permission Structure)

### 8.1 DocType Izin Matrisi

| Rol | Create | Read | Write | Delete | Export | Print | Report | Share | if_owner |

| System Manager | 1 | 1 | 1 | 1 | 1 | 1 | 1 | 1 | 0 |

### 10.3 Parent Category Secici

`parent_category` alani icin filtrelenmi Link field:

```javascript
// category_proposal.js

frappe.ui.form.on('Category Proposal', {
    setup: function(frm) {
        // Ust kategori filtreleme: yalnizca aktif ve is_group olanlar
        frm.set_query('parent_category', function() {
            return {
                filters: {
                    'enabled': 1,
                    'is_group': 1
                }
            };
        });
    },

    refresh: function(frm) {
        // Satirci icin auto-set
        if (frm.is_new() && !frm.doc.seller) {
            frappe.call({
                method: 'frappe.client.get_value',
                args: {
                    doctype: 'Seller Profile',
                    filters: { user: frappe.session.user },
                    fieldname: 'name'
                },
                callback: function(r) {
                    if (r.message) {
                        frm.set_value('seller', r.message.name);
                    }
                }
            });
        }

        // Satirci icin seller alani read-only
        if (frappe.user.has_role('Seller') &&
            !frappe.user.has_role('System Manager')) {
            frm.set_df_property('seller', 'read_only', 1);
        }
    }
});
```

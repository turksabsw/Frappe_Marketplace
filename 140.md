# Ürün Menşe & Durum Alanları (Product Origin & Condition Fields)

## 1. Bağlam (Context)

### Mevcut Durum

TradeHub platformunda ürün durumu (condition) ve menşe bilgisi (origin) alanları şu anda kanal bazlı olarak dağınık bir yapıdadır. Genel marketplace seviyesinde standart bir `condition` alanı bulunmamaktadır. Bu belge, hem **PIM Product** hem de **Listing** DocType'larına eklenecek standart durum ve menşe alanlarını, validasyon kurallarını, filtre entegrasyonunu ve rozet (badge) gösterim yaklaşımını planlamaktadır.

---

### 1.1 PIM Product DocType — Mevcut Durum

- **Konum:** `tradehub_catalog/doctype/pim_product/pim_product.json`
- **Toplam Alan:** 247 alan, 18 sekme (tab)

#### 1.1.1 Mevcut Compliance Tab (Uyumluluk Sekmesi)

PIM Product'ta `compliance_tab` altında aşağıdaki menşe alanları **zaten mevcut**:

| fieldname | fieldtype | label | options | reqd | default | description |
|-----------|-----------|-------|---------|------|---------|-------------|
| country_of_origin | Link | Country of Origin | Country | 0 | — | Country where product was designed/originated |
| country_of_manufacture | Link | Country of Manufacture | Country | 0 | — | Country where product was manufactured |

**Önemli Not:** Bu alanlar `compliance_section` → "Origin & Manufacturing" başlığı altında yer alır.

#### 1.1.2 Kanal Bazlı Condition Alanları (Mevcut)

| Kanal | fieldname | fieldtype | options | default |
|-------|-----------|-----------|---------|---------|
| Amazon | amazon_condition | Select | New\nRefurbished\nUsed - Like New\nUsed - Very Good\nUsed - Good\nUsed - Acceptable\nCollectible - Like New\nCollectible - Very Good\nCollectible - Good\nCollectible - Acceptable | New |
| eBay | ebay_condition_id | Data | — (1000=New, 1500=New Other, vb.) | 1000 |
| eBay | ebay_condition_description | Small Text | — | — |

**Tespit:** Genel marketplace seviyesinde bir `condition` alanı **mevcut değil**. Her kanal kendi condition formatını kullanıyor.

### 1.2 Listing DocType — Mevcut Durum

- **Konum:** `tradehub_seller/doctype/listing/listing.json`
- **Toplam Alan:** ~141 alan

#### 1.2.1 Mevcut Shipping Section (Kargo Bölümü)

| fieldname | fieldtype | label | options | reqd | default |
|-----------|-----------|-------|---------|------|---------|
| ships_from_country | Link | Ships From Country | Country | 0 | Turkey |
| ships_from_city | Data | Ships From City | — | 0 | — |

**Tespit:**
- `condition` alanı **mevcut değil** — satıcılar ürün durumunu belirleyemiyor
- `country_of_origin` alanı **mevcut değil** — sadece kargo çıkış ülkesi var (`ships_from_country`)
- `city_of_origin`, `manufactured_by` alanları **mevcut değil**

### 1.3 Country DocType (Frappe Yerleşik)

Frappe Framework'te `Country` DocType'ı yerleşik olarak gelir:
- **Adlandırma:** Ülke adı (ör. "Turkey", "China", "Germany")
- **Alanlar:** `country_name`, `code` (ISO 3166-1 alpha-2)
- PIM Product'taki `country_of_origin` ve `country_of_manufacture` alanları zaten bu DocType'a Link olarak bağlıdır
- Listing'deki `ships_from_country` da aynı şekilde Country'ye bağlıdır

## 5. Validasyon Kuralları (Validation Rules)

### 5.1 Alan Validasyonları

| # | Kural | Alan | Koşul | Hata Mesajı (İngilizce) | DocType |
|---|-------|------|-------|--------------------------|---------|
| 1 | Zorunlu alan | condition | Her zaman | "Condition is required" | PIM Product, Listing |
| 2 | Koşullu zorunlu | condition_note | condition ∈ {Used - Like New, Used - Good, Used - Acceptable, Renewed} | "Condition Note is required for Used and Renewed products" | PIM Product, Listing |
| 3 | Minimum uzunluk | condition_note | Zorunlu olduğu durumlarda len ≥ 20 karakter | "Condition Note must be at least 20 characters for Used/Renewed products" | PIM Product, Listing |
| 4 | Geçerli ülke | country_of_origin | Var ise, Country DocType'ında mevcut olmalı | "Invalid Country of Origin" | PIM Product, Listing |
| 5 | Maksimum uzunluk | manufactured_by | len ≤ 200 karakter | "Manufactured By cannot exceed 200 characters" | PIM Product, Listing |
| 6 | Maksimum uzunluk | city_of_origin | len ≤ 140 karakter | "City of Origin cannot exceed 140 characters" | PIM Product, Listing |

### 5.2 İş Kuralları (Business Logic)

| # | Kural | Açıklama | Uygulama |
|---|-------|----------|----------|
| 1 | Condition değişiklik kaydı | Condition değiştiğinde eski değer loglanır | `before_save` hook ile comment oluştur |
| 2 | Used/Renewed uyarı | Status "Active" iken condition "For Parts or Not Working" yapılırsa uyarı | `validate` hook ile frappe.msgprint |
| 3 | Kanal eşleme | Genel condition değiştiğinde kanal bazlı condition alanları güncellenir | `validate` hook ile CONDITION_CHANNEL_MAP kullanarak |
| 4 | condition_note HTML temizliği | XSS önleme — condition_note'tan HTML tag'lerini temizle | `validate` hook ile `frappe.utils.html_utils.clean_html` |

### 5.3 Validasyon Kodu (Planlama Notu)

```python
# pim_product.py ve listing.py'de validate metodu (planlama notu — kod oluşturulmayacak)

def validate(self):
    self.validate_condition_note()
    self.sync_channel_conditions()

def validate_condition_note(self):
    """Used ve Renewed ürünler için condition_note zorunluluğu"""
    requires_note = ["Used - Like New", "Used - Good", "Used - Acceptable", "Renewed"]

    if self.condition in requires_note:
        if not self.condition_note or len(self.condition_note.strip()) < 20:
            frappe.throw(
                _("Condition Note is required and must be at least 20 characters "
                  "for {0} products").format(self.condition),
                title=_("Condition Note Required")
            )

def sync_channel_conditions(self):
    """Genel condition alanından kanal bazlı condition'ları otomatik güncelle"""
    # Sadece PIM Product'ta (kanal alanları olan)
    if self.doctype == "PIM Product" and self.condition:
        mapping = CONDITION_CHANNEL_MAP.get(self.condition, {})
        if mapping.get("amazon_condition"):
            self.amazon_condition = mapping["amazon_condition"]
        if mapping.get("ebay_condition_id"):
            self.ebay_condition_id = mapping["ebay_condition_id"]
```

```javascript
// listing.js / pim_product.js (planlama notu — kod oluşturulmayacak)
frappe.ui.form.on("Listing", {
    condition: function(frm) {
        // condition_note alanını temizle (condition "New" yapılırsa)

        var requires_note = [

                message: __("Please provide a detailed condition note for {0} products",
                    [frm.doc.condition]),
                indicator: "orange"
            });
        }
    }
});
```

```python
# patch: set_default_condition_for_existing_records (planlama notu)
def execute():
    # PIM Product — mevcut kayıtlara "New" ata
    frappe.db.sql("""
        UPDATE `tabPIM Product`
        SET `condition` = 'New'
        WHERE `condition` IS NULL OR `condition` = ''
    """)

```

#### 7.2.2 Kanal Condition Senkronizasyonu

Amazon/eBay kanal condition alanlarından genel condition alanına ters eşleme:

```python
# patch: sync_channel_conditions_to_general (planlama notu)
REVERSE_AMAZON_MAP = {
    "New": "New",
    "Refurbished": "Refurbished",
    "Used - Like New": "Used - Like New",
    "Used - Very Good": "Used - Good",
    "Used - Good": "Used - Good",
    "Used - Acceptable": "Used - Acceptable",
}

def execute():
    # Amazon condition değeri olan PIM Product'lar için genel condition ata
    products = frappe.get_all("PIM Product",
        filters={"amazon_condition": ("is", "set"), "condition": ("is", "not set")},
        fields=["name", "amazon_condition"]
    )
    for product in products:
        general_condition = REVERSE_AMAZON_MAP.get(product.amazon_condition, "New")
        frappe.db.set_value("PIM Product", product.name, "condition", general_condition)

    frappe.db.commit()
```

### 7.3 Index Oluşturma

Filtre performansı için veritabanı index'leri:

```sql
-- PIM Product condition index
ALTER TABLE `tabPIM Product` ADD INDEX `idx_condition` (`condition`);

-- Listing condition index
ALTER TABLE `tabListing` ADD INDEX `idx_condition` (`condition`);

-- Listing country_of_origin index
ALTER TABLE `tabListing` ADD INDEX `idx_country_of_origin` (`country_of_origin`);

-- Composite index (condition + country)
ALTER TABLE `tabListing` ADD INDEX `idx_condition_country` (`condition`, `country_of_origin`);
```

**Not:** Frappe, `in_standard_filter: 1` olan alanlar için index'leri migration sırasında otomatik oluşturur. Yukarıdaki SQL'ler sadece referans amaçlıdır.

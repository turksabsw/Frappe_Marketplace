## 1. Executive Summary

This document defines the **Brand Authorization System** for the TradeHub B2B Marketplace. It introduces a new `Brand Authorization Request` DocType as an **application/request workflow** through which sellers apply for permission to sell a specific brand. When approved, the request creates or updates a record in the existing **Brand Gating** DocType — which remains the single source of truth for seller-brand authorization status.

### Key Deliverables

1. **Brand DocType `owner_seller` Field** — New Link→Seller Profile field (defined in Task 156) and its relationship to the existing `registered_by` field
2. **Brand Authorization Request DocType** — A new APPLICATION workflow DocType (naming_series `BAR-.YYYY.-.#####`) for sellers to formally request brand authorization with supporting documents
3. **Approval → Brand Gating Integration** — On approval, automatically creates or updates a Brand Gating record with the appropriate authorization_type, validity period, and supporting documents
4. **Product Creation Brand Enforcement** — Server-side `validate` hook on SKU Product + client-side `get_query` filter to prevent unauthorized sellers from selecting gated brands
5. **Brand Owner Authorization Panel** — Brand owners (via `owner_seller`) can manage incoming authorization requests, approve/reject, and view authorized sellers
6. **Multi-Tenant Isolation** — All data scoped via tenant field inherited from Seller Profile

### Critical Design Principle

> **Brand Gating = the AUTHORIZATION RECORD (who is authorized, what they can do)**
> **Brand Authorization Request = the APPLICATION WORKFLOW (how they apply for authorization)**
>
> The Brand Authorization Request DocType does NOT replace Brand Gating. It feeds INTO Brand Gating.
> A seller can also be authorized via direct Brand Gating creation (e.g., by admin or brand owner)
> without going through the request workflow.

### Cross-Task Dependencies

```
Task 156 (Brand Ownership)    Task 151 (this doc)              Task 152 (Variant Request)
  owner_seller field     --->   Brand Authorization Request --->  Only authorized sellers
  Brand Ownership Claim         Feeds into Brand Gating           can create variant requests
  Owner control panel           Product creation enforcement      via Brand Gating check
```

- **Task 156** defines `owner_seller` on Brand — this document uses it to determine who reviews authorization requests
- **Task 152** depends on Brand Gating (with status=Approved) as the prerequisite for variant request creation
- The Brand Authorization Request is the **formal application path** — not the only path to Brand Gating authorization

### 2.2 `owner_seller` vs `registered_by` — Key Distinction

| Aspect | `registered_by` (existing) | `owner_seller` (planned, Task 156) |
|--------|---------------------------|-----------------------------------|
| **Links to** | User | Seller Profile |
| **Set by** | System (auto-set on insert) | Admin (on Brand Ownership Claim approval) |
| **Meaning** | The user who first created this brand record | The seller business entity that legally owns the brand |
| **Mutability** | Set once, never changes | Can be revoked and reassigned |
| **Authorization role** | None (informational only) | Can review Brand Authorization Requests, manage Brand Gating |

A brand can be `registered_by` a marketplace admin but `owner_seller` is the verified legal owner. Many brands will have `registered_by` set but `owner_seller` = null (no ownership claimed yet). When `owner_seller` is null, only System Manager can manage Brand Gating records for that brand.

## 3. Brand Authorization Request DocType

### 3.1 Overview

The `Brand Authorization Request` is a **new standalone DocType** that provides a formal application workflow for sellers to request brand authorization. Unlike the existing `request_authorization()` API which directly creates a Brand Gating record in Pending status, this DocType:

1. **Requires supporting documents** (authorization letter, distribution agreement)
2. **Has its own status workflow** independent of Brand Gating
3. **Routes to brand owner** (if `owner_seller` exists) for review, not just System Manager
4. **Creates/updates Brand Gating** only upon approval
5. **Supports justification and review notes** for audit trail

### 3.2 Relationship to Brand Gating — Explicitly Defined

```
┌────────────────────────────────┐         ┌──────────────────────┐
│ Brand Authorization Request    │         │ Brand Gating         │
│ (Application Workflow)         │         │ (Authorization Record)│
│                                │         │                      │
│ • Seller submits request       │         │ • Stores final auth  │
│ • Attaches documents           │  ───>   │ • Controls product   │
│ • Brand owner/admin reviews    │ creates │   listing ability    │
│ • Status: Pending → Approved   │  or     │ • Integrates with    │
│                                │ updates │   Buy Box algorithm  │
│ Has: justification, documents, │         │ • Has: pricing rules,│
│   review_notes, review trail   │         │   product scope,     │
│                                │         │   validity period    │
└────────────────────────────────┘         └──────────────────────┘
```

**Key points:**
- Brand Authorization Request is **one path** to getting a Brand Gating record
- Admin/brand owner can ALSO create Brand Gating records directly (e.g., for known partners)
- Brand Authorization Request has a back-reference (`brand_gating` field) to the created Brand Gating record
- Deleting a Brand Authorization Request does NOT delete the associated Brand Gating record

```
│ Pending  │ ──────────────> │ Under Review │

     ↑                             │
     │ resubmit                   ├──── approve ──→ ┌──────────┐   creates   ┌──────────────┐
     │                             │                  │ Approved │ ─────────> │ Brand Gating │
     └─────── reject ◄────────────┘                  └──────────┘            └──────────────┘
                │
                ↓
          ┌──────────┐
          │ Rejected │
          └──────────┘
```

**Valid Transitions:**

```python
valid_transitions = {
    "Pending": ["Under Review"],
    "Under Review": ["Approved", "Rejected", "Pending"],
    "Approved": [],         # Terminal state (cannot be changed; revoke via Brand Gating instead)
    "Rejected": ["Pending"]  # Seller can resubmit with updated documents
}
```

**Transition Side Effects:**

| Transition | Actor | Side Effect |
|------------|-------|-------------|
| Pending → Under Review | System/Admin/Brand Owner | Set request as under active review; notify admin queue |
| Under Review → Approved | Admin/Brand Owner | Set `reviewed_by`, `review_date`; create/update Brand Gating record; set `brand_gating` back-reference; notify seller |
| Under Review → Rejected | Admin/Brand Owner | Set `reviewed_by`, `review_date`; `rejection_reason` required; notify seller |
| Under Review → Pending | Admin/Brand Owner | Return to seller for revisions; notify seller |
| Rejected → Pending | Seller | Seller resubmits with updated documents/justification; clear review fields |

### 3.8 Controller Logic (brand_authorization_request.py)

```python
# Copyright (c) 2026, Trade Hub and contributors
# For license information, please see license.txt

"""
Brand Authorization Request DocType for Trade Hub B2B Marketplace.

This module implements the Brand Authorization Request — an APPLICATION workflow
for sellers to formally request brand authorization. On approval, it creates or
updates a Brand Gating record. It does NOT replace Brand Gating.

Key Features:
- Document-based application with supporting files
- Routing to brand owner (if exists) or System Manager for review
- Automatic Brand Gating creation/update on approval
- Duplicate request prevention
- Multi-tenant isolation via Seller Profile's tenant
"""

import frappe
from frappe import _
from frappe.model.document import Document
from frappe.utils import today, getdate


class BrandAuthorizationRequest(Document):
    """
    Brand Authorization Request — application workflow feeding into Brand Gating.

    Each request represents a seller's formal application to be authorized to sell
    a specific brand. Approval creates/updates a Brand Gating record.
    """

    def before_insert(self):
        """Set defaults before inserting a new request."""
        self.request_date = today()
        self.set_tenant_from_seller()
        self.check_duplicate_request()
        self.check_existing_gating()

    def validate(self):
        """Validate request data before saving."""
        self.validate_seller_eligibility()
        self.validate_brand_eligibility()
        self.validate_status_transition()
        self.validate_documents_for_type()
        self.validate_rejection_reason()
        self.validate_tenant_isolation()

    def on_update(self):
        """Actions after request is updated."""
        if self.has_value_changed("status"):
            self.handle_status_change()

    # =========================================================================
    # INITIALIZATION METHODS
    # =========================================================================

    def set_tenant_from_seller(self):
        """Auto-set tenant from requesting seller."""
        if self.requesting_seller and not self.tenant:
            self.tenant = frappe.db.get_value(
                "Seller Profile", self.requesting_seller, "tenant"
            )

    def check_duplicate_request(self):
        """Prevent duplicate active requests for same brand-seller."""
        existing = frappe.db.exists(
            "Brand Authorization Request",
            {
                "requesting_seller": self.requesting_seller,
                "brand": self.brand,
                "status": ["in", ["Pending", "Under Review"]],
                "name": ["!=", self.name or ""]
            }
        )
        if existing:
            frappe.throw(
                _("An active authorization request already exists for this brand-seller "
                  "combination: {0}").format(existing)
            )

    def check_existing_gating(self):
        """Check if a Brand Gating record already exists."""
        existing_gating = frappe.db.exists(
            "Brand Gating",
            {
                "brand": self.brand,
                "seller": self.requesting_seller,
                "authorization_status": ["in", ["Pending", "Under Review", "Approved"]]
            }
        )
        if existing_gating:
            self.has_existing_gating = 1
            frappe.msgprint(
                _("Note: A Brand Gating record ({0}) already exists for this "
                  "brand-seller combination. Approval will update it.").format(
                    existing_gating
                ),
                indicator="orange"
            )

    # =========================================================================
    # VALIDATION METHODS
    # =========================================================================

    def validate_seller_eligibility(self):
        """Ensure requesting seller is active and verified."""
        if not self.requesting_seller:
            frappe.throw(_("Requesting Seller is required"))

        seller = frappe.db.get_value(
            "Seller Profile",
            self.requesting_seller,
            ["status", "verification_status"],
            as_dict=True
        )
        if not seller:
            frappe.throw(_("Seller Profile not found"))
        if seller.status == "Deactivated":
            frappe.throw(_("Deactivated sellers cannot request brand authorization"))
        if seller.verification_status != "Verified":
            frappe.msgprint(
                _("Warning: Seller is not yet verified. "
                  "Request may be delayed until verification is complete."),
                indicator="orange",
                alert=True
            )

    def validate_brand_eligibility(self):
        """Ensure brand is enabled and preferably verified."""
        if not self.brand:
            frappe.throw(_("Brand is required"))

        brand = frappe.db.get_value(
            "Brand",
            self.brand,
            ["enabled", "verification_status"],
            as_dict=True
        )
        if not brand:
            frappe.throw(_("Brand not found"))
        if not brand.enabled:
            frappe.throw(_("Cannot request authorization for a disabled brand"))
        if brand.verification_status != "Verified":
            frappe.msgprint(
                _("Warning: Brand '{0}' is not yet verified.").format(self.brand),
                indicator="orange",
                alert=True
            )

    def validate_status_transition(self):
        """Validate allowed status transitions."""
        if self.has_value_changed("status"):
            old_doc = self.get_doc_before_save()
            old_status = old_doc.status if old_doc else "Pending"
            new_status = self.status

            valid_transitions = {
                "Pending": ["Under Review"],
                "Under Review": ["Approved", "Rejected", "Pending"],
                "Approved": [],
                "Rejected": ["Pending"]
            }

            if new_status not in valid_transitions.get(old_status, []):
                # System Manager can override
                if "System Manager" not in frappe.get_roles():
                    frappe.throw(
                        _("Invalid status transition from {0} to {1}").format(
                            old_status, new_status
                        )
                    )

    def validate_documents_for_type(self):
        """Validate document requirements based on authorization type."""
        if self.status in ["Under Review", "Approved"]:
            if self.authorization_type == "Authorized Distributor":
                if not self.authorization_letter:
                    frappe.throw(
                        _("Authorization letter is required for "
                          "Authorized Distributor requests")
                    )
            elif self.authorization_type == "Exclusive Distributor":
                if not self.authorization_letter:
                    frappe.throw(
                        _("Authorization letter is required for "
                          "Exclusive Distributor requests")
                    )
                if not self.distribution_agreement:
                    frappe.throw(
                        _("Distribution agreement is required for "
                          "Exclusive Distributor requests")
                    )
            elif self.authorization_type == "Official Partner":
                if not self.authorization_letter:
                    frappe.throw(
                        _("Authorization letter is required for "
                          "Official Partner requests")
                    )

    def validate_rejection_reason(self):
        """Ensure rejection reason is provided when rejecting."""
        if self.status == "Rejected" and not self.rejection_reason:
            frappe.throw(_("Rejection reason is required when rejecting a request"))

    def validate_tenant_isolation(self):
        """Validate multi-tenant data isolation."""
        if not self.tenant:
            return

        if "System Manager" in frappe.get_roles():
            return

        try:
            from tradehub_core.tradehub_core.utils.tenant import get_current_tenant
            current_tenant = get_current_tenant()
            if current_tenant and self.tenant != current_tenant:
                frappe.throw(
                    _("Access denied: You can only access requests in your tenant")
                )
        except ImportError:
            pass

    # =========================================================================
    # STATUS CHANGE HANDLERS
    # =========================================================================

    def handle_status_change(self):
        """Handle side effects of status changes."""
        if self.status == "Approved":
            self._on_approve()
        elif self.status == "Rejected":
            self._on_reject()
        elif self.status == "Pending":
            self._on_return_to_pending()
        elif self.status == "Under Review":
            self._on_submit_for_review()

    def _on_approve(self):
        """Create or update Brand Gating record on approval."""
        self.reviewed_by = frappe.session.user
        self.review_date = today()

        # Check for existing Brand Gating record
        existing_gating = frappe.db.get_value(
            "Brand Gating",
            {
                "brand": self.brand,
                "seller": self.requesting_seller
            },
            "name"
        )

        if existing_gating:
            # Update existing Brand Gating record
            gating = frappe.get_doc("Brand Gating", existing_gating)
            gating.authorization_status = "Approved"
            gating.authorization_type = self.authorization_type
            if self.requested_product_scope:
                gating.product_scope = self.requested_product_scope
            if self.requested_categories:
                gating.allowed_categories = self.requested_categories
            if self.authorization_letter:
                gating.authorization_document = self.authorization_letter
            if self.distribution_agreement:
                gating.authorization_certificate = self.distribution_agreement
            gating.authorization_notes = (
                f"Updated via Brand Authorization Request {self.name}"
            )
            gating.save(ignore_permissions=True)
            self.brand_gating = gating.name
        else:
            # Create new Brand Gating record
            gating = frappe.new_doc("Brand Gating")
            gating.brand = self.brand
            gating.seller = self.requesting_seller
            gating.authorization_status = "Approved"
            gating.authorization_type = self.authorization_type
            if self.requested_product_scope:
                gating.product_scope = self.requested_product_scope
            if self.requested_categories:
                gating.allowed_categories = self.requested_categories
            if self.authorization_letter:
                gating.authorization_document = self.authorization_letter
            if self.distribution_agreement:
                gating.authorization_certificate = self.distribution_agreement
            gating.authorization_notes = (
                f"Created via Brand Authorization Request {self.name}"
            )
            gating.insert(ignore_permissions=True)
            self.brand_gating = gating.name

        # Notify seller
        frappe.publish_realtime(
            "brand_authorization_approved",
            {
                "brand": self.brand,
                "seller": self.requesting_seller,
                "brand_gating": self.brand_gating
            }
        )

    def _on_reject(self):
        """Set rejection info and notify seller."""
        self.reviewed_by = frappe.session.user
        self.review_date = today()

        frappe.publish_realtime(
            "brand_authorization_rejected",
            {
                "brand": self.brand,
                "seller": self.requesting_seller,
                "reason": self.rejection_reason
            }
        )

    def _on_return_to_pending(self):
        """Clear review fields when returned to pending."""
        # Only clear if going from Rejected → Pending (resubmission)
        old_doc = self.get_doc_before_save()
        if old_doc and old_doc.status == "Rejected":
            self.reviewed_by = None
            self.review_date = None
            self.rejection_reason = None
            self.review_notes = None

    def _on_submit_for_review(self):
        """Notify reviewers when request enters review."""
        # Determine reviewer: brand owner or System Manager
        brand_owner = frappe.db.get_value(
            "Brand", self.brand, "owner_seller"
        )

        if brand_owner:
            # Get the user linked to the brand owner's Seller Profile
            owner_user = frappe.db.get_value(
                "Seller Profile", brand_owner, "user"
            )
            if owner_user:
                frappe.publish_realtime(
                    "brand_authorization_request_submitted",
                    {
                        "request_name": self.name,
                        "brand": self.brand,
                        "seller": self.requesting_seller,
                        "seller_name": self.seller_name
                    },
                    user=owner_user
                )
```

### 4.3 Handling Existing Brand Gating Records

When a Brand Authorization Request is approved:

| Existing Brand Gating Status | Action |
|------------------------------|--------|
| None (no record) | Create new Brand Gating record |
| Pending | Update to Approved |
| Under Review | Update to Approved |
| Approved | Update authorization_type and documents (effectively a re-authorization) |
| Rejected | Update to Approved (reinstatement) |
| Suspended | Update to Approved (reinstatement) |
| Expired | Update to Approved (renewal) |
| Revoked | Update to Approved (reinstatement; new valid_from = today) |

### 4.4 Immutability of Approved Requests

Once a Brand Authorization Request is Approved:
- The request status cannot be changed (terminal state)
- The Brand Gating record can be independently modified (suspended, revoked, etc.)
- The `brand_gating_status` field on the request auto-reflects the current gating status via `fetch_from`
- To revoke authorization, use Brand Gating directly — not the request
- A seller can submit a NEW request if they need to change authorization_type or submit updated documents

## 5. Product Creation Brand Enforcement

### 5.1 Overview

Currently, any seller can select any brand when creating an SKU Product. This section defines how to enforce brand authorization checks so that:
1. **Unauthorized sellers cannot create products** for brands that have active Brand Gating records (i.e., gated brands)
2. **Non-gated brands remain freely selectable** (no Brand Gating records = open brand)
3. **Brand owners are automatically authorized** for their own brand

### 5.2 Determining if a Brand is Gated

A brand is considered "gated" if ANY Brand Gating record exists for that brand (regardless of the seller). This means at least one seller has been explicitly authorized, implying the brand requires authorization.

```python
def is_brand_gated(brand):
    """
    Check if a brand requires authorization (has any Brand Gating records).

    Returns:
        bool: True if brand has at least one Brand Gating record
    """
    return frappe.db.exists(
        "Brand Gating",
        {
            "brand": brand,
            "authorization_status": ["in", ["Approved", "Pending", "Under Review"]]
        }
    )
```

### 5.3 Server-Side Enforcement: `validate` Hook on SKU Product

Add to `sku_product.py` (in `tradehub_seller`):

```python
def validate(self):
    """Extended validation with brand authorization check."""
    # ... existing validations ...
    self.validate_brand_authorization()

def validate_brand_authorization(self):
    """
    Validate that the seller is authorized to list products for this brand.

    Rules:
    1. If brand is null/empty → skip (brand is optional on SKU Product)
    2. If brand has no Brand Gating records → skip (open brand)
    3. If seller is the brand owner (Brand.owner_seller) → allow
    4. If seller has Approved Brand Gating for this brand → allow
    5. Otherwise → throw error
    """
    if not self.brand or not self.seller:
        return

    # Check if brand is gated (has any Brand Gating records)
    gated = frappe.db.exists(
        "Brand Gating",
        {
            "brand": self.brand,
            "authorization_status": ["in", [
                "Approved", "Pending", "Under Review",
                "Suspended", "Expired", "Revoked"
            ]]
        }
    )

    if not gated:
        # Open brand — no restrictions
        return

    # Check if seller is the brand owner
    brand_owner = frappe.db.get_value("Brand", self.brand, "owner_seller")
    if brand_owner and brand_owner == self.seller:
        return  # Brand owner is always authorized

    # Check if seller has approved Brand Gating
    authorization = frappe.db.get_value(
        "Brand Gating",
        {
            "brand": self.brand,
            "seller": self.seller,
            "authorization_status": "Approved",
            "is_active": 1
        },
        "name"
    )

    if not authorization:
        frappe.throw(
            _("You are not authorized to list products for brand '{0}'. "
              "Please submit a Brand Authorization Request or contact the "
              "brand owner for authorization.").format(self.brand),
            title=_("Brand Authorization Required")
        )

    # Check product count limits
    gating = frappe.get_doc("Brand Gating", authorization)
    if not gating.can_add_product():
        if gating.max_products > 0:
            frappe.throw(
                _("Product limit reached for brand '{0}'. "
                  "Maximum {1} products allowed, currently {2}.").format(
                    self.brand, gating.max_products, gating.current_product_count
                ),
                title=_("Product Limit Exceeded")
            )
```

```python
@frappe.whitelist()
@frappe.validate_and_sanitize_search_inputs
def get_authorized_brands_for_seller(doctype, txt, searchfield, start, page_len, filters):
    """
    Return brands that a seller is authorized to sell.

    Logic:
    1. Include all open brands (no Brand Gating records)
    2. Include brands where seller has Approved Brand Gating
    3. Include brands where seller is the brand owner (owner_seller)
    4. Exclude gated brands where seller is NOT authorized

    Args:
        filters: Must include 'seller' key

    Returns:
        list: Matching brand names for the Link field dropdown
    """
    seller = filters.get("seller")
    if not seller:
        # No seller → show all enabled brands
        return frappe.db.sql("""
            SELECT name, brand_name
            FROM `tabBrand`
            WHERE enabled = 1
              AND (brand_name LIKE %(txt)s OR name LIKE %(txt)s)
            ORDER BY brand_name
            LIMIT %(start)s, %(page_len)s
        """, {
            "txt": f"%{txt}%",
            "start": start,
            "page_len": page_len
        })

    return frappe.db.sql("""
        SELECT b.name, b.brand_name
        FROM `tabBrand` b
        WHERE b.enabled = 1
          AND (b.brand_name LIKE %(txt)s OR b.name LIKE %(txt)s)
          AND (
            -- Open brands (no gating records at all)
            NOT EXISTS (
                SELECT 1 FROM `tabBrand Gating` bg
                WHERE bg.brand = b.name
                  AND bg.authorization_status IN (
                    'Approved', 'Pending', 'Under Review',
                    'Suspended', 'Expired', 'Revoked'
                  )
            )
            -- OR seller is the brand owner
            OR b.owner_seller = %(seller)s
            -- OR seller has approved Brand Gating
            OR EXISTS (
                SELECT 1 FROM `tabBrand Gating` bg2
                WHERE bg2.brand = b.name
                  AND bg2.seller = %(seller)s
                  AND bg2.authorization_status = 'Approved'
                  AND bg2.is_active = 1
            )
          )
        ORDER BY b.brand_name
        LIMIT %(start)s, %(page_len)s
    """, {
        "txt": f"%{txt}%",
        "seller": seller,
        "start": start,
        "page_len": page_len
    })
```

```python
def has_permission(doc, ptype, user=None):
    """
    Custom permission check for Brand Authorization Request.

    Brand owners get read+write access to requests for their brand.
    """
    if not user:
        user = frappe.session.user

    # System Manager always has access
    if "System Manager" in frappe.get_roles(user):
        return True

    # Check if user is the brand owner
    if doc.brand:
        brand_owner_seller = frappe.db.get_value("Brand", doc.brand, "owner_seller")
        if brand_owner_seller:
            # Get the user linked to this Seller Profile
            seller_user = frappe.db.get_value(
                "Seller Profile", brand_owner_seller, "user"
            )
            if seller_user == user:
                return True  # Brand owner can read/write requests for their brand

    # Default: use standard permission (if_owner for Seller role)
    return None  # Return None to fall through to standard permission checks


def get_permission_query_conditions(user=None):
    """
    Add permission conditions for list views.

    Shows:
    - Own requests (standard if_owner)
    - Requests for brands the user owns
    """
    if not user:
        user = frappe.session.user

    if "System Manager" in frappe.get_roles(user):
        return ""

    # Get seller profile for user
    seller = frappe.db.get_value(
        "Seller Profile", {"user": user}, "name"
    )

    if not seller:
        return "1=0"  # No access

    # Get brands owned by this seller
    owned_brands = frappe.get_all(
        "Brand",
        filters={"owner_seller": seller},
        pluck="name"
    )

    conditions = [f"`tabBrand Authorization Request`.requesting_seller = '{seller}'"]

    if owned_brands:
        brand_list = "', '".join(owned_brands)
        conditions.append(
            f"`tabBrand Authorization Request`.brand IN ('{brand_list}')"
        )

    return "({})".format(" OR ".join(conditions))
```

```python
@frappe.whitelist()
def review_authorization_request(request_name, action, review_notes=None,
                                  internal_notes=None, rejection_reason=None):
    """
    Brand owner or admin reviews an authorization request.

    Args:
        request_name: Brand Authorization Request name (e.g., BAR-2026-00001)
        action: "approve", "reject", or "return" (return to Pending)
        review_notes: Notes visible to seller
        internal_notes: Internal notes (not visible to seller)
        rejection_reason: Required if action is "reject"

    Returns:
        dict: Updated request with new status
    """
    doc = frappe.get_doc("Brand Authorization Request", request_name)

    # Verify caller is brand owner or System Manager
    _verify_brand_owner_or_admin(doc.brand)

    action_map = {
        "approve": "Approved",
        "reject": "Rejected",
        "return": "Pending"
    }

    new_status = action_map.get(action)
    if not new_status:
        frappe.throw(_("Invalid action: {0}. Use 'approve', 'reject', or 'return'.").format(action))

    doc.status = new_status
    if review_notes:
        doc.review_notes = review_notes
    if internal_notes:
        doc.internal_review_notes = internal_notes
    if rejection_reason:
        doc.rejection_reason = rejection_reason

    doc.save(ignore_permissions=True)

    return {
        "success": True,
        "name": doc.name,
        "status": doc.status,
        "brand_gating": doc.brand_gating,
        "message": _("Request {0}").format(new_status.lower())
    }
```

```python
def get_permission_query_conditions(user=None):
    """
    Multi-tenant aware permission conditions.

    Shows:
    - Own requests (if Seller role with if_owner)
    - Requests for brands the user owns (brand owner)

        return "1=0"
```

### 7.4 Cross-Tenant Brand Authorization

```python
def validate_tenant_for_brand(self):
```

## 8. Relationship Summary: Brand Gating vs Brand Authorization Request

### 8.1 Comparison Table

| Aspect | Brand Gating | Brand Authorization Request |
|--------|-------------|---------------------------|

| **Status states** | 7 (Pending, Under Review, Approved, Rejected, Suspended, Expired, Revoked) | 4 (Pending, Under Review, Approved, Rejected) |

## 11. Edge Cases & Error Handling

### 11.1 Edge Case Matrix

| # | Edge Case | Expected Behavior |
|---|-----------|-------------------|
| 1 | Seller submits request for brand with no owner | Request routes to System Manager queue (no brand owner to review) |
| 2 | Brand owner deactivated after request submitted | System Manager takes over review; brand owner loses has_permission access |
| 3 | Duplicate request (same seller + brand, both Pending) | Prevented by `check_duplicate_request()` in before_insert |
| 4 | Request approved but Brand Gating creation fails | Transaction rollback; request stays in previous state; error logged |
| 5 | Brand Gating revoked after request was approved | Request.brand_gating_status reflects "Revoked" via fetch_from; request remains Approved (historical record) |
| 6 | Brand becomes disabled after request submitted | `validate_brand_eligibility()` blocks transition to Under Review/Approved |
| 7 | Seller deactivated after request submitted | `validate_seller_eligibility()` blocks transition to Under Review/Approved |
| 8 | Brand gated after seller already has products | Existing products are NOT retroactively blocked; only new product creation is enforced |
| 9 | Authorization type changed from Exclusive to Standard | Updates Brand Gating record; existing products remain unaffected |
| 10 | System Manager approves request for brand they don't own | Allowed — System Manager has full access regardless of ownership |
| 11 | Seller resubmits after rejection | New request OR Rejected→Pending transition; review fields cleared |
| 12 | Multiple requests from different sellers for same brand | Each request is independent; no aggregation (unlike variant requests in Task 152) |

### 11.2 Error Messages

| Error | Message | When |
|-------|---------|------|
| Duplicate request | "An active authorization request already exists for this brand-seller combination: {name}" | before_insert |
| Seller deactivated | "Deactivated sellers cannot request brand authorization" | validate |
| Brand disabled | "Cannot request authorization for a disabled brand" | validate |
| Invalid transition | "Invalid status transition from {old} to {new}" | validate |
| Missing documents | "Authorization letter is required for {type} requests" | validate (on review/approve) |
| Missing rejection reason | "Rejection reason is required when rejecting a request" | validate |
| Brand not authorized | "You are not authorized to list products for brand '{brand}'" | SKU Product validate |
| Product limit exceeded | "Product limit reached for brand '{brand}'. Maximum {max} products allowed, currently {count}." | SKU Product validate |
| Not brand owner | "Only the brand owner or System Manager can perform this action." | API call |

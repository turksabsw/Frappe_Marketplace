## 1. Executive Summary

This document defines the **Brand Profile & Ownership System** for the TradeHub B2B Marketplace. It extends the existing `Brand` DocType in `tradehub_catalog` with enhanced profile fields and introduces a new `Brand Ownership Claim` DocType to support formal brand ownership through an admin-reviewed claim workflow.

### Key Deliverables

1. **Brand DocType Extension** — Add 7 new fields (cover_image, social_media_links, founding_story, brand_values, owner_seller, owner_seller_name, ownership_date) to the existing Brand DocType while clearly preserving all existing fields
2. **Brand Social Media Link Child DocType** — Structured child table for social media links (platform, url, is_active)
3. **Brand Ownership Claim DocType** — Full claim workflow: a seller submits trademark/trade registry documents → admin reviews → one seller per brand is approved as owner
4. **Brand Owner Control Panel** — Approved brand owners gain seller authorization management, catalog management, profile editing, and content control capabilities
5. **Buyer-Facing Brand Page** — Public brand profile with verified badge, product catalog, authorized seller list, and brand story
6. **Competing Claims Resolution** — Admin workflow for handling multiple sellers claiming the same brand
7. **Ownership Revocation Cascade** — Defined effects when brand ownership is revoked

### Cross-Task Dependencies

```
Task 156 (this doc)           Task 151                    Task 152
Brand Ownership        -->    Brand Authorization   -->   Variant Request
  owner_seller field          Request workflow             Brand owner reviews
  Brand Ownership Claim       Feeds into Brand Gating      Only authorized sellers
  Owner control panel         Product creation enforcement  Demand aggregation
```

- **Task 151** depends on the `owner_seller` field defined here to determine who can manage Brand Gating records
- **Task 152** depends on Task 151's authorization model to restrict variant request creation to authorized sellers
- The Brand Ownership Claim DocType defined here is the prerequisite for the entire brand authority chain

### 3.4 `owner_seller` vs `registered_by` — Key Distinction

| Aspect | `registered_by` (existing) | `owner_seller` (new) |
|--------|---------------------------|---------------------|
| **Links to** | User | Seller Profile |
| **Set by** | System (auto-set on insert) | Admin (on Brand Ownership Claim approval) |
| **Meaning** | The user who first created this brand record in the system | The seller business entity that legally owns the brand |
| **Mutability** | Set once, never changes | Can be revoked and reassigned |
| **Permission grant** | None (informational only) | Grants brand owner control capabilities |

A brand can be `registered_by` a marketplace admin but `owner_seller` is the verified legal owner. Many brands will have `registered_by` set but `owner_seller` as null (no ownership claimed yet).

### 3.6 Validation Additions to brand.py

New methods to add to `Brand` class:

```python
def validate(self):
    """Extended validation."""
    # ... existing validations ...
    self.validate_brand_values_length()
    self.validate_owner_seller()

def validate_brand_values_length(self):
    """Validate brand values max length."""
    if self.brand_values and len(self.brand_values) > 500:
        frappe.throw(_("Brand Values cannot exceed 500 characters"))

def validate_owner_seller(self):
    """Validate brand owner seller is active and verified."""
    if self.owner_seller:
        seller = frappe.db.get_value(
            "Seller Profile",
            self.owner_seller,
            ["status", "verification_status"],
            as_dict=True
        )
        if not seller:
            frappe.throw(_("Brand Owner seller not found"))
        if seller.status != "Active":
            frappe.throw(_("Brand Owner seller must have Active status"))
        if seller.verification_status != "Verified":
            frappe.throw(
                _("Brand Owner seller must be verified")
            )
```

### 4.5 Validation (in Brand.validate)

```python
def validate_social_media_links(self):
    """Validate social media links."""
    if not self.social_media_links:
        return

    seen_platforms = {}
    for idx, link in enumerate(self.social_media_links):
        # Validate URL format
        if link.profile_url and not link.profile_url.startswith("http"):
            frappe.throw(
                _("Row {0}: Profile URL must start with http:// or https://").format(idx + 1)
            )

        # Warn on duplicate platforms (allow but warn)
        if link.platform in seen_platforms:
            frappe.msgprint(
                _("Row {0}: Duplicate platform '{1}' (also in row {2})").format(
                    idx + 1, link.platform, seen_platforms[link.platform]
                ),
                indicator="orange"
            )
        seen_platforms[link.platform] = idx + 1
```

## 5. Brand Ownership Claim DocType

### 5.1 Overview

The `Brand Ownership Claim` DocType is a **request/application workflow** that allows a seller to claim legal ownership of a brand. When approved by an admin, the Brand's `owner_seller` field is set to the claiming seller.

**Key principle:** Only ONE seller can be the approved owner of a brand at any time. Multiple claims can coexist in Pending/Under Review states, but only one can reach Approved.

### 5.4 Status Workflow

```
┌─────────┐     submit      ┌──────────────┐
│ Pending  │ ──────────────> │ Under Review │
└─────────┘                  └──────┬───────┘
     ↑                             │
     │ reopen                      ├──── approve ──→ ┌──────────┐
     │                             │                  │ Approved │
     └─────── reject ◄────────────┘                  └──────────┘
                │
                ↓
          ┌──────────┐
          │ Rejected │
          └──────────┘
```

**Valid Transitions:**

```python
valid_transitions = {
    "Pending": ["Under Review"],
    "Under Review": ["Approved", "Rejected", "Pending"],
    "Approved": ["Rejected"],  # Revocation
    "Rejected": ["Pending"]    # Resubmission
}
```

**Transition side effects:**

| Transition | Side Effect |
|------------|-------------|
| → Under Review | Set `reviewed_by`, notify admin queue |
| → Approved | Set `review_date`, set Brand.`owner_seller` = `claiming_seller`, set Brand.`ownership_date` = today, reject all other pending/under review claims for same brand |
| → Rejected | Set `review_date`, `rejection_reason` required, notify seller |
| Approved → Rejected | **Revocation cascade** (see Section 8) |

| Seller | 1 | 1 (if_owner) | 1 (if_owner, status=Pending only) | 0 | 0 |
| Buyer | 0 | 0 | 0 | 0 | 0 |

**Note:** Sellers can only edit their OWN claims, and only while status is Pending. Once submitted for review, the claim becomes read-only for the seller.

### 5.7 Controller Logic (brand_ownership_claim.py)

```python
class BrandOwnershipClaim(Document):
    """Brand Ownership Claim with admin review workflow."""

    def before_insert(self):
        self.claim_date = today()
        self.set_tenant_from_seller()
        self.check_existing_ownership()
        self.check_competing_claims()

    def validate(self):
        self.validate_seller_eligibility()
        self.validate_status_transition()
        self.validate_documents()
        self.validate_unique_active_claim()

    def on_update(self):
        if self.has_value_changed("status"):
            self.handle_status_change()

    def set_tenant_from_seller(self):
        """Auto-set tenant from claiming seller."""
        if self.claiming_seller and not self.tenant:
            self.tenant = frappe.db.get_value(
                "Seller Profile", self.claiming_seller, "tenant"
            )

    def check_existing_ownership(self):
        """Warn if brand already has an owner."""
        owner = frappe.db.get_value("Brand", self.brand, "owner_seller")
        if owner:
            frappe.msgprint(
                _("Brand '{0}' already has owner '{1}'. "
                  "Your claim will be reviewed as a competing claim.").format(
                    self.brand_name, owner
                ),
                indicator="orange"
            )

    def check_competing_claims(self):
        """Check and flag competing claims."""
        existing = frappe.db.count(
            "Brand Ownership Claim",
            filters={
                "brand": self.brand,
                "status": ["in", ["Pending", "Under Review"]],
                "name": ["!=", self.name or ""]
            }
        )
        if existing > 0:
            self.is_competing_claim = 1

    def validate_seller_eligibility(self):
        """Ensure claiming seller is active and verified."""
        seller = frappe.db.get_value(
            "Seller Profile",
            self.claiming_seller,
            ["status", "verification_status"],
            as_dict=True
        )
        if not seller:
            frappe.throw(_("Seller not found"))
        if seller.status != "Active":
            frappe.throw(_("Only active sellers can claim brand ownership"))
        if seller.verification_status != "Verified":
            frappe.throw(_("Only verified sellers can claim brand ownership"))

    def validate_status_transition(self):
        """Validate allowed status transitions."""
        if self.has_value_changed("status"):
            old_doc = self.get_doc_before_save()
            old_status = old_doc.status if old_doc else "Pending"
            # ... transition validation (see Section 5.4)

    def validate_documents(self):
        """Ensure required documents are uploaded."""
        if self.status in ["Under Review", "Approved"]:
            if not self.trademark_certificate:
                frappe.throw(
                    _("Trademark certificate is required for review")
                )
            if not self.trade_registry_document:
                frappe.throw(
                    _("Trade registry document is required for review")
                )

    def validate_unique_active_claim(self):
        """One seller can have at most one active claim per brand."""
        existing = frappe.db.exists(
            "Brand Ownership Claim",
            {
                "claiming_seller": self.claiming_seller,
                "brand": self.brand,
                "status": ["in", ["Pending", "Under Review", "Approved"]],
                "name": ["!=", self.name or ""]
            }
        )
        if existing:
            frappe.throw(
                _("You already have an active ownership claim for this brand")
            )

    def handle_status_change(self):
        """Handle side effects of status changes."""
        if self.status == "Approved":
            self._approve_claim()
        elif self.status == "Rejected":
            self._reject_claim()

    def _approve_claim(self):
        """Grant brand ownership to claiming seller."""
        self.reviewed_by = frappe.session.user
        self.review_date = today()

        # Set brand owner
        brand = frappe.get_doc("Brand", self.brand)
        brand.owner_seller = self.claiming_seller
        brand.ownership_date = today()
        brand.save(ignore_permissions=True)

        # Reject all other claims for this brand
        competing_claims = frappe.get_all(
            "Brand Ownership Claim",
            filters={
                "brand": self.brand,
                "status": ["in", ["Pending", "Under Review"]],
                "name": ["!=", self.name]
            },
            pluck="name"
        )
        for claim_name in competing_claims:
            claim = frappe.get_doc("Brand Ownership Claim", claim_name)
            claim.status = "Rejected"
            claim.rejection_reason = "Competing Claim Approved"
            claim.rejection_details = _(
                "Another ownership claim for this brand has been approved."
            )
            claim.reviewed_by = frappe.session.user
            claim.review_date = today()
            claim.save(ignore_permissions=True)

        # Notify seller
        frappe.publish_realtime(
            "brand_ownership_approved",
            {"brand": self.brand, "seller": self.claiming_seller}
        )

    def _reject_claim(self):
        """Reject claim and notify seller."""
        self.reviewed_by = frappe.session.user
        self.review_date = today()

        if not self.rejection_reason:
            frappe.throw(_("Rejection reason is required"))
```

## 6. Competing Claims Handling

### 6.1 Scenario

Two or more sellers submit Brand Ownership Claims for the same brand. This is a common real-world scenario (e.g., a manufacturer and an exclusive distributor both claim "ownership").

### 6.2 Detection

On `before_insert` of Brand Ownership Claim:
1. Check if any other claim exists with `status IN (Pending, Under Review, Approved)` for the same brand
2. If yes, set `is_competing_claim = 1` on the new claim
3. Also retroactively update existing pending/under review claims to `is_competing_claim = 1`

```python
def flag_all_competing_claims(brand):
    """Flag all pending/under review claims for a brand as competing."""
    claims = frappe.get_all(
        "Brand Ownership Claim",
        filters={
            "brand": brand,
            "status": ["in", ["Pending", "Under Review"]]
        },
        pluck="name"
    )
    if len(claims) > 1:
        for claim_name in claims:
            frappe.db.set_value(
                "Brand Ownership Claim", claim_name,
                "is_competing_claim", 1,
                update_modified=False
            )
```

### 6.4 Resolution Rules

| Priority | Ownership Type | Evidence Weight |
|----------|---------------|-----------------|
| 1 (Highest) | Manufacturer | Strongest — direct production evidence |
| 2 | Trademark Holder | Strong — legal registration |
| 3 | Parent Company | Strong — corporate ownership documents |
| 4 (Lowest) | Exclusive Licensee | Moderate — contractual, time-limited |

**Note:** These are guidance rules for admins, not hard-coded logic. The admin makes the final decision and documents reasoning in `admin_notes`.

### 6.5 Auto-Rejection on Approval

When one claim is approved:
- All other Pending/Under Review claims for the same brand are automatically rejected
- Rejection reason is set to "Competing Claim Approved"
- Each affected seller receives a notification explaining that another seller's claim was approved
- Rejected sellers can appeal by submitting a new claim with additional documentation

def validate_owner_editable_fields(self):
    """Ensure brand owner only edits allowed fields."""
    owner_editable = {

### 8.1 Role Hierarchy

## 10. Brand Ownership Revocation Cascade Effects

### 10.1 Revocation Triggers

Brand ownership can be revoked via:
1. **Admin action**: System Manager changes Brand Ownership Claim status from Approved → Rejected
2. **Seller deactivation**: Seller Profile status changes to Inactive/Suspended
3. **Seller verification revoked**: Seller Profile verification_status changes from Verified

### 10.2 Cascade Effects

When `Brand.owner_seller` is cleared (revocation):

```
Brand Ownership Revoked
│
├─── 1. Clear Brand.owner_seller
│    Brand.owner_seller = None
│    Brand.ownership_date = None
│
├─── 2. Update Brand Ownership Claim
│    Claim.status → Rejected
│    Claim.rejection_reason = "Ownership Revoked"
│    Claim.rejection_details = admin-provided reason
│
├─── 3. Brand Gating Records (CRITICAL)
│    ├── Records CREATED BY the former owner:
│    │   → Mark authorized_by as "Revoked Owner"
│    │   → Keep authorization_status UNCHANGED
│    │   → Admin decides whether to revoke seller authorizations
│    │
│    └── Records for the former owner AS SELLER:
│        → No automatic change (owner was also a seller)
│        → Separate Brand Gating record governs their selling rights
│
├─── 4. SKU Products
│    → No automatic changes to products
│    → Products remain listed under the brand
│    → Former owner loses catalog management view
│
├─── 5. Variant Requests (Task 152 — future)
│    → Pending variant requests remain pending
│    → Admin or new owner must review them
│
└─── 6. Notifications
     ├── Notify former owner: "Your brand ownership has been revoked"
     ├── Notify all authorized sellers: "Brand ownership has changed"
     └── Notify admins: "Brand {X} ownership revoked from {seller}"
```

### 10.3 Revocation API

```python
@frappe.whitelist()
def revoke_brand_ownership(brand_name, reason, revoke_seller_authorizations=False):
    """
    Admin revokes brand ownership.

    Args:
        brand_name: Brand to revoke ownership from
        reason: Reason for revocation
        revoke_seller_authorizations: If True, also revoke all Brand Gating
            records created by the former owner

    Returns:
        dict: {
            "brand": brand_name,
            "former_owner": seller_name,
            "gating_records_affected": int,
            "notifications_sent": int
        }
    """
```

### 10.4 Important Design Decision: Gating Records Survive Revocation

**Rationale:** Brand Gating records represent business relationships between sellers and the brand. Revoking ownership should NOT automatically invalidate existing seller authorizations because:
- Authorized sellers may have inventory, active orders, and customer commitments
- Mass revocation could disrupt the marketplace
- The admin should make a conscious decision per-seller

**Default behavior:** Keep all Brand Gating records as-is. Provide an opt-in flag `revoke_seller_authorizations` for cases where the admin wants to clean up.

## 11. Multi-Tenant Isolation

### 11.1 Existing Pattern (from Brand DocType)

The Brand DocType already implements multi-tenant isolation:
- `tenant` (Link → Tenant): Set on brand record
- `is_global` (Check): Global brands visible to all tenants
- `set_tenant_from_user()`: Auto-sets tenant from current user's tenant
- `validate_tenant_consistency()`: Global brands clear tenant, non-global brands require tenant

### 11.2 Brand Ownership Claim — Tenant Isolation

The Brand Ownership Claim inherits tenant from the claiming seller:

```json
{
  "fieldname": "tenant",
  "fieldtype": "Link",
  "options": "Tenant",
  "fetch_from": "claiming_seller.tenant",
  "read_only": 1,
  "in_standard_filter": 1,
  "description": "Auto-filled from Seller - Multi-tenant isolation field"
}
```

**Rules:**
1. Tenant is auto-set from `claiming_seller.tenant` via `fetch_from`
2. Sellers can only claim brands within their tenant OR global brands
3. A seller in Tenant A cannot claim a brand that belongs to Tenant B
4. Global brands (`is_global = 1`) can be claimed by sellers from any tenant

**Validation:**
```python
def validate_tenant_consistency(self):
    """Ensure seller and brand are in the same tenant."""
    brand_tenant = frappe.db.get_value("Brand", self.brand, "tenant")
    brand_is_global = frappe.db.get_value("Brand", self.brand, "is_global")

    if not brand_is_global and brand_tenant and self.tenant:
        if brand_tenant != self.tenant:
            frappe.throw(
                _("Cannot claim brand from a different tenant. "
                  "Brand belongs to tenant '{0}' but your tenant is '{1}'.").format(
                    brand_tenant, self.tenant
                )
            )
```

### 11.3 Brand Social Media Link — Tenant Isolation

As a child table, Brand Social Media Link inherits tenant from its parent Brand document. No additional tenant field needed.

### 11.4 Permission Query Conditions

All list views must respect tenant isolation:

```python
# In permissions.py
def brand_ownership_claim_conditions(user):
    """Filter Brand Ownership Claims by tenant."""
    if "System Manager" in frappe.get_roles(user):
        return ""

    user_tenant = frappe.db.get_value("User", user, "tenant")
    if user_tenant:
        return f"`tabBrand Ownership Claim`.tenant = '{user_tenant}'"

    return "1=0"
```

### 11.5 Cross-Tenant Brand Ownership Scenarios

| Scenario | Brand Tenant | Seller Tenant | Allowed? |
|----------|-------------|---------------|----------|
| Same tenant | Tenant A | Tenant A | Yes |
| Global brand | None (global) | Tenant A | Yes |
| Cross-tenant | Tenant A | Tenant B | No — blocked by validation |
| No tenant brand | None (non-global) | Tenant A | Yes — admin must resolve |
